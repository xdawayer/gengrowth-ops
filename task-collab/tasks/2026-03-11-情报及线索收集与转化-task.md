# 情报及线索收集与转化任务

- [ ] [情报系统] 建立 GenGrowth 情报收集、人物调研、价值转化与周更沉淀机制，作为后续智能体向下执行的统一任务入口 #task #owner/lynne

## 目的

本任务的目的，不是简单“看资讯”或“做资料汇总”，而是建立一套面向 GenGrowth 的持续情报系统，把分散在社交媒体、社区、平台规则、潜在合作资源中的信息，转化为可复用、可追踪、可被智能体继续执行的结构化资产。

这套情报系统要直接服务 GenGrowth 当前阶段的四个核心目标：

1. 服务商业判断：持续理解 AI 增长行业、独立开发者生态、平台变化和竞品动态，减少战略盲区。
2. 服务获客与品牌：识别值得跟进的人、内容、案例和趋势，为内容营销、Build in Public 和官网转化提供输入。
3. 服务合作与招聘：从高价值账号、社区和内容中提炼商务合作线索、潜在招聘对象线索和生态资源线索。
4. 服务自动化执行：把“看见的信息”沉淀为标准化字段、固定目录、固定表格、固定节奏和固定文档入口，便于后续智能体自动产出日报、周报、线索池和内容参考。

## 背景前提

本任务必须以以下文档为前提进行理解和执行，不能脱离 GenGrowth 当前的商业方向和营销策略单独做泛信息收集：

1. `docs/01-company/business-plan/GenGrowth商业计划书.md`
2. `docs/03-marketing/01-strategy/GenGrowth整体营销框架.md`
3. `docs/03-marketing/02-social-media/social-media-operations.md`
4. `docs/02-product/01-prd/2026-03-04-gengrowth-agent-sop-prd-v1.3.md`
5. `docs/04-programs/planning/项目规划.md`

当前默认判断口径：

1. GenGrowth 的核心目标用户是独立开发者、超级个体、小团队出海产品创建者。
2. 当前最重要的内容方向不是泛 AI 新闻，而是“增长、分发、验证、自动化执行、平台规则、真实案例、合作资源”。
3. 当前最需要沉淀的不是大而全百科，而是能帮助 GenGrowth 更快获客、建立信任、找到合作、找到合适人才、提炼自动化策略的高信号信息。

## 范围

本任务包含四部分，缺一不可：

1. 日报需求：沉淀近 24 小时高价值内容摘要，并做个性化推荐优化。
2. 调研需求：围绕已关注人物做结构化画像、分组研究、平台策略总结和相关人群扩展。
3. 转化需求：把调研结果转成商务沟通、招聘沟通、内容营销和自动化流程可直接使用的线索资产。
4. 执行边界：当前阶段只做研究、沉淀、结构化输出和参考文档映射，不启动实际执行动作。

## 日报需求

### 任务目标

将个人账号在以下平台“已关注 + 平台推荐”的近 24 小时内容进行汇总，按热度排序，并基于当前已知的个人关注习惯与 GenGrowth 业务重点进行推荐优化：

1. X
2. 微信公众号
3. LinkedIn
4. Discord
5. Product Hunt

### 日报输出要求

每份日报至少包含以下内容：

1. 近 24 小时高热内容摘要，按热度排序。
2. 每条内容的来源平台、作者、发布时间、主题、热度依据、原始链接。
3. 每条内容与 GenGrowth 的相关性说明。
4. 每条内容的建议动作：继续观察、加入周研报、沉淀到内容参考、提炼为商务线索、提炼为招聘线索、暂时忽略。
5. 一段“推荐优化建议”，说明接下来哪些类型内容应增加关注，哪些类型应降低权重。

### 推荐优化的默认逻辑

在没有额外个人偏好输入前，默认按以下优先级进行推荐优化：

1. 高优先级：独立开发者增长、AI 产品分发、Build in Public、平台规则变化、社媒内容方法、真实转化案例、Product Hunt 发布经验、Discord 社区玩法、合作资源、招聘信号。
2. 中优先级：竞品动态、AI 工具链变化、创始人叙事、社媒风格案例、跨平台内容复用方法。
3. 低优先级：纯行业八卦、泛融资新闻、无明确执行价值的宏观观点、与独立开发者生态弱相关的热点。

### 日报固定输出目录

日报统一沉淀到：

1. `每日日报/`

建议文件命名：

1. `YYYY-MM-DD-0900.md`
2. `YYYY-MM-DD-2000.md`

如后续需要更细分，可在同一文档中增加“社媒情报”固定章节，不另建分散目录。

### 日报建议表格字段

| 平台 | 作者/账号 | 是否已关注 | 发布时间 | 主题 | 热度依据 | 相关性等级 | 价值判断 | 建议动作 | 原始链接 |
|---|---|---|---|---|---|---|---|---|---|

## 调研需求

### 任务目标

在明确 GenGrowth 商业计划书、营销规划、PRD 和内容策略的前提下，对本人已关注的人群进行平台化、分类化、结构化研究，形成可复用的人物库和平台打法参考。

### 人群分类要求

至少按以下类别分类，不限于以下类别：

1. AI 行业大佬 / AI 基础设施 / AI 应用方向意见领袖
2. 营销 / 增长 / SEO / GEO / 内容营销从业者
3. 独立开发者 / 超级个体 / Build in Public 创作者
4. 竞品 / 替代产品 / 相邻赛道创始人
5. 平台官方 / 社区组织者 / 规则制定者 / 活跃版主
6. 投资人 / 生态资源方 / 合作渠道方
7. 高信号内容创作者 / 媒体号 / 案例号

### 单个人物画像要求

每个人物至少要输出以下内容：

1. 所属分类
2. 平台
3. 账号名称 / 昵称 / Handle
4. 基本介绍
5. 主要产品或主要项目及简介
6. 粉丝量
7. 关注量
8. 发帖数量或可观测内容规模
9. 平均互动情况
10. 发帖频率
11. 常见话题
12. 发帖风格
13. 受众特征
14. 对 GenGrowth 的价值点
15. 可借鉴的内容或增长思路
16. 潜在合作思路
17. 风险或注意事项
18. 原始主页链接
19. 关键代表内容链接
20. 数据抓取时间

### 人物调研建议表格字段

| 类别 | 平台 | 账号/Handle | 人物介绍 | 主要产品/项目 | 粉丝量 | 关注量 | 发帖量 | 平均互动 | 发帖频率 | 发帖风格 | 价值点 | 可借鉴思路 | 可合作思路 | 风险点 | 主页链接 | 数据时间 |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|

### 平台策略总结要求

除单个人物画像外，每周调研报告还必须单独总结不同平台的内容发布策略。至少回答以下问题：

1. 这个平台当前什么内容更容易获得曝光？
2. 这个平台适合 GenGrowth 哪类账号去发？
3. 这个平台更适合发观点、过程、案例、教程、互动问答还是产品发布？
4. 这个平台有哪些显性的规则约束、隐性的社区习惯或常见踩坑点？
5. 当前平台上有哪些值得持续观察的头部人物、活跃社群、栏目、榜单或话题标签？

### 相关人群扩展要求

基于已关注的人，继续探索相关高价值人群。扩展逻辑至少包括：

1. 同类账号的关注/被关注关系
2. 高频互动对象
3. 被转发、被引用、被提名的账号
4. Product Hunt 的 Maker / Hunter / 评论者网络
5. Discord 的活跃管理员、版主、核心贡献者
6. Reddit 的高信号发帖者、版主、常见高赞评论者

扩展时必须做筛选，优先保留与以下主题强相关的人群：

1. 独立开发者增长
2. AI 产品冷启动
3. SEO / GEO / 社媒分发
4. Build in Public
5. 工具类 SaaS / 小团队产品营销
6. 潜在合作资源与潜在招聘对象

### 调研报告固定输出目录

周度调研报告统一沉淀到：

1. `参考资料/社媒人物调研/`

建议文件命名：

1. `YYYY-MM-DD-gengrowth-社媒人物调研周报.md`

如后续需要拆分子主题，仍优先在该目录下统一维护，不分散到多个无规则文件夹。

### 调研更新频率

1. 每周至少更新一次完整调研报告。
2. 遇到平台规则重大变化、关键人物爆发增长、竞品重要动作、Product Hunt 上新高相关产品时，可额外插入专题调研。

## 转化需求

### 任务目标

基于对以下基础知识文档的理解，把“信息”转成“可用于沟通、内容和自动化执行的资产”：

1. GenGrowth 商业计划书
2. PRD
3. 营销策略
4. 营销内容相关文档

### 转化方向

调研结果必须至少转成以下三类线索：

1. 商务沟通线索：适合合作、资源互换、渠道联动、内容共创、社区联动的人和机会点。
2. 招聘对象沟通线索：潜在增长运营、内容运营、技术合伙人、社媒执行等候选对象的切入点与吸引点。
3. 内容营销参考：值得模仿的发帖风格、开头钩子、案例表达方式、截图方式、线程结构、评论互动策略。

### 转化输出要求

每次转化不能只停留在“这人不错”或“这篇值得看”，而要明确产出：

1. 可复用的话术切入点
2. 可复用的价值主张
3. 适合的沟通对象
4. 适合的场景
5. 对应的风险提醒
6. 是否适合纳入后续自动化流程

### 转化后的固定沉淀位置

1. 商务沟通线索：`商务线索/`
2. 招聘沟通线索：建议沉淀到 `参考资料/招聘与人才线索/`
3. 内容风格参考：建议沉淀到 `内容资产/`

建议形成以下长期文件：

1. `商务线索/GenGrowth商务沟通线索池.md`
2. `参考资料/招聘与人才线索/GenGrowth招聘沟通线索池.md`
3. `内容资产/GenGrowth社媒发帖风格参考.md`

## 自动化流程参考文档映射

本任务不是孤立研究任务，后续必须把高价值结论映射进 GenGrowth 自动化流程参考文档，供智能体向下执行。

优先映射到以下文档：

1. `docs/03-marketing/02-social-media/social-media-operations.md`
   - 适合承接平台规则、内容节奏、社媒协同、风险控制、复盘字段。
2. `docs/02-product/01-prd/2026-03-04-gengrowth-agent-sop-prd-v1.3.md`
   - 适合承接需要进入 Agent 输入项、输出物、自动化流程或字段设计的内容。
3. `docs/04-programs/planning/项目规划.md`
   - 适合承接需要落到人力分工、周节奏、岗位补位和项目里程碑的结论。

重点转化方向包括：

1. 哪类人物和内容值得系统持续监控
2. 哪类帖子适合做 X / Reddit / LinkedIn / Product Hunt / Discord 的模板变体
3. 哪类商务沟通对象应进入线索池
4. 哪类人才画像值得进入招聘优先级
5. 哪类平台规则应作为自动化执行的护栏条件

## 执行边界

当前阶段执行需求暂缓。

这句话的具体含义必须明确为：

1. 当前先做调研、结构化沉淀、模板定义、固定目录规划和文档映射。
2. 当前不启动实际发帖、评论、私信、加好友、社群运营、商务触达和招聘触达。
3. 当前不启动自动抓取脚本开发、不启动自动发布能力接入、不启动账号批量操作。
4. 当前允许产出“建议动作”和“后续可执行动作”，但不实际执行。

## 验收标准

满足以下标准，才算该任务定义完整、后续可交给智能体执行：

1. 任务文档明确写清楚“目的”，并与 GenGrowth 商业计划书、营销规划和 PRD 对齐。
2. 日报要求完整覆盖 X、微信公众号、LinkedIn、Discord、Product Hunt 五个平台。
3. 日报明确要求按热度排序，并给出基于当前习惯与业务重点的推荐优化逻辑。
4. 调研要求明确“人物分类、人物画像字段、平台策略总结、相关人群扩展、周更频率、固定目录”。
5. 转化要求明确“商务沟通、招聘沟通、内容营销参考、自动化流程映射”四种下游用途。
6. 固定输出目录和建议文件命名已写明，便于后续智能体持续产出。
7. 已明确执行边界，避免任务在当前阶段被误解为“立刻开始运营执行”。
8. 文档中的表格字段和说明足够结构化，后续可直接转成模板或自动化输入。
