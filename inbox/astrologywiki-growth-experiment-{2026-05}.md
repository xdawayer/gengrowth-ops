# Day 0 诊断执行 SOP

> **适用场景**：每个种子客户实验启动前，在 Week 1 第一天执行。  
> **总时长**：约 5 小时，建议分两段执行（上午 3h + 下午 2h）。  
> **执行角色**：SEO 同事主导，Lynne 审核"产品初步判断"部分。  
> **配套文件**：填写 `{产品名}-growth-experiment-{YYYY-MM}.md`（Section I：Day 0 诊断基线）

---

## GenGrowth 协同框架

> **核心原则**：GenGrowth 提供结构化起点，人工负责判断层。  
> 只有经过 2-3 个种子客户验证的稳定工作流，才反哺进产品。**不把不确定的流程塞进产品再反复修改。**

### 能力映射与职责边界

| 步骤                | GenGrowth 负责                           | 人工负责                                             | 数据流向                     | 产品化时机                   |
| ----------------- | -------------------------------------- | ------------------------------------------------ | ------------------------ | ----------------------- |
| Step 0 产品理解       | 输出产品档案草稿（分类 / 核心功能 / 描述 / 定价 / 竞品初步列表） | 亲自使用产品，验证/修正草稿，补充主观判断                            | 修正结果 → 回写 GenGrowth 产品档案 | **已可用**，当前作为起点而非空白      |
| Step 1 技术审计       | 执行审计，输出技术健康报告                          | 解读报告，标注优先级，做 go / no-go                          | 审计报告 → 模板技术健康部分          | **已可用**                 |
| Step 2 流量基线（自有）   | GSC / GA4 接入后自动拉取数据（当前数据量过小）           | 品牌/非品牌词拆分判断，解读战略含义                               | → 模板流量现状                 | **已规划**，接入 GSC/GA4 后可用  |
| Step 2 外链基线       | 提供竞品 URL 列表                            | Ahrefs 拉取 DR / 引用域名，标记快速机会                       | → 模板流量现状 / 外链基线          | 需要 Ahrefs API，暂不规划      |
| Step 3 竞品分析（产品层）  | **功能对比 / 定价 / 定位分析**（GenGrowth 现有功能）   | 验证 GenGrowth 输出，补充主观判断                           | → 统一竞品档案                 | **已可用**                 |
| Step 3 竞品分析（SEO层） | 暂不涉及                                   | **Top Pages / Content Gap / 关键词映射**（Ahrefs 人工操作） | → 统一竞品档案 + 关键词主表         | 需要 Ahrefs API，待方法论稳定后规划 |
| Step 4 种子词 + 社区挖掘 | 暂不涉及                                   | 全流程（按关键词研究 SOP）                                  | → 关键词主表                  | 需 2-3 客户验证方法论后再考虑集成     |
| Step 5 关键词分桶      | 分桶逻辑已验证（Apps Script），可作为近期集成目标         | 关键词数据录入；CPC=0 词意图判断                              | → 分桶结果 → 执行计划            | **逻辑稳定**，可规划集成          |
| Step 6 可增长性判断     | 数据驱动部分可系统化（快速胜利词数量 / DR 差距 / 技术阻塞）     | 核心假设制定（需要业务判断，不能自动化）                             | → 模板核心目标设定               | 判断框架规则清晰，中期集成目标         |

### 竞品分析的明确切分

两种竞品分析**定位不同，不重叠，最终汇入同一竞品档案**：

| 维度 | GenGrowth 产品竞品分析 | 人工 SEO 竞品分析 |
|------|---------------------|----------------|
| 目的 | 理解市场定位、功能差距、定价策略 | 找到可执行的关键词机会和内容缺口 |
| 关注点 | 功能列表 / 定价 / 目标用户 / 独特卖点 | Top Pages / 排名关键词 / Content Gap |
| 工具 | GenGrowth 内置 | Ahrefs Site Explorer / Content Gap |
| 产出 | 竞品功能对比表 + 定位差异 | 关键词候选池（竞品映射来源） |
| 执行者 | GenGrowth 自动生成，人工验证 | SEO 同事手动操作 |

> **注意**：GenGrowth 的竞品分析不替代 SEO 竞品分析，两者回答的问题不同。

### 反哺规则

满足以下任一条件，才将工作流写入 GenGrowth 产品：

1. **稳定性**：在 2 个以上种子客户中执行过，没有出现需要修改流程的情况
2. **可复制性**：非 Lynne 本人可以按照文档独立执行，不依赖个人判断
3. **可量化**：有明确的输入 → 输出定义，不是"根据经验判断"

当前**明确不产品化**的项目：
- 社区挖掘（原话识别依赖人工语义判断）
- 核心假设制定（业务判断，无法标准化）
- 内容角度选题（需要创始人直接参与）

## Step 0 — 产品理解（15 分钟，先于任何工具操作）

**目标**：以 GenGrowth 产品档案为起点，通过亲自使用产品验证并修正，不从空白开始。

执行动作：
1. **从 GenGrowth 拉取产品档案草稿**：分类 / 核心功能描述 / 定价 / 竞品初步列表
2. **亲自使用产品 5 分钟**（注册 → 核心功能走一遍），带着以下问题：
   - GenGrowth 的产品描述是否准确？哪里有偏差？
   -产品描述准确，无偏差
   - 核心价值主张的用户感受与描述一致吗？
   -一致
   - 转化路径（CTA → 注册 → 付费）有没有明显摩擦？
   -没有明显摩擦，但付费入口不明显
1. **修正产品档案**，补充以下主观判断（GenGrowth 无法自动生成）：
   - 最可能付费的用户画像（一句话）
   -具有进取心，渴望了解自我改变自身情况的年轻人、渴望获得专业占星术知识的爱好者、迷信占星学生活方式的主妇类群体
   - 用户为什么选它而不选竞品（差异化核心）
   - [www.astro.com/cgi/ade.cgi](http://www.astro.com/cgi/ade.cgi) :注册流程更加简洁 [www.astrology.com](http://www.astrology.com/) ：购买计划更加便宜，且有7天免费体验 ==（这一部分相当的主观，没有的对比标准，例如页面美观程度、网站加载速度、商品定价、提供服务差异等）==
1. **将修正结果回写 GenGrowth 产品档案**
==目前用时大概30min，主要用时在体验产品初期流程，以及查看竞品链接是否正确，对比与竞品所提供的价格以及服务== 

> 如果修正后三个核心问题仍回答不清楚，先和产品负责人确认，不要带着模糊假设启动分析。  
> 目标是把 30 分钟缩短到 15 分钟——GenGrowth 提供的草稿质量越高，这一步越快。

## Step 1 — GenGrowth 技术审计（30 分钟）

**目标**：发现阻塞性技术问题（索引问题、Core Web Vitals 严重超标、移动可用性故障）。

| 工具                        | 检查项           | 判断标准                         |
| ------------------------- | ------------- | ---------------------------- |
| GenGrowth 审计工具            | 技术健康整体扫描      | 导出主要问题清单                     |
| Google PageSpeed Insights | INP（交互到下一次绘制） | INP ≤ 200ms 为良好；> 500ms 为阻塞级 |
| Google PageSpeed Insights | LCP（最大内容渲染）   | ≤ 2.5s 为良好                   |
| Google PageSpeed Insights | CLS（累积布局偏移）   | ≤ 0.1 为良好                    |
| GSC → Indexing → Pages    | 索引覆盖率         | 「已索引」比例，记录「已排除」的主要原因         |
| GSC → Experience          | 移动可用性         | 是否有错误页面                      |

填入模板：技术健康部分  
标记优先级：**阻塞级**（必须 Week 1 修复才能继续 SEO）/ **影响级**（影响效果但不阻塞）/ **可选**

## Step 4 — 种子词拓展 + 社区挖掘（60 分钟）

**目标**：补齐竞品分析遗漏的长尾需求和用户真实用语。

### 4.1 种子词多维度拓展

操作路径：Ahrefs → Keywords Explorer

基于产品功能设计 15-20 个种子词，按以下维度拓展：

| 维度 | 示例模式 | 预期产出 |
|-----|---------|---------|
| 问题型 | `how to [功能]`、`[问题] fix` | 教程/指南类 |
| 比较型 | `[产品类] vs`、`[产品类] alternative` | 商业意图高 |
| 工具型 | `best [工具类型]`、`[功能] tool` | 工具聚合页机会 |
| 价格型 | `[产品类] pricing`、`[竞品] cost` | 底层漏斗 |
| 集成型 | `[产品] + [相关工具]` | 长尾，低竞争 |

每个维度拿 20-50 词，过滤 KD > 60，导入关键词主表（来源：种子词拓展）。

### 4.2 社区快速挖掘（20 分钟）

平台优先级：Reddit → Indie Hackers → Hacker News → 产品相关 Discord/Slack

操作步骤：
1. 在 Reddit 搜索产品类型关键词，找 upvote ≥ 100 的帖子
2. 读评论区，提取用户描述问题的原话（不是平台用语）
3. 这些原话 = 长尾关键词候选
4. 记录原话 + 帖子链接，填入关键词主表（来源：社区挖掘）

> 社区挖掘的核心价值：找到工具不会建议的词，通常是最真实的搜索意图。

### 4.3 趋势词检查（10 分钟）

操作路径：Google Trends（trends.google.com）

- 对比 3-5 个核心种子词的近 12 个月趋势
- 找到明显上升但竞品尚未大量覆盖的话题
- Exploding Topics 补充检查
- 有趋势信号的词，在关键词主表中 F 列填入 Trends 比值估算

==对4.2和4.3进行整合修改==
 _Step 4.2— 战略信号审计（15 分钟）_

  **目标**：不产出词表，只验证品类的“天花板”和“活力”，为 Go/No-Go 提供主观依据。

   1. **社区信号探测**：

       * 在 Reddit / Indie Hackers 搜索 3-5 个产品核心痛点。

       * **审计标准**：是否存在近 6 个月内、评论数 > 10 的活跃讨论？

       * **结论**：若有，证明“流量可挖掘”；若无，SEO 难度增加。

   2. **趋势信号探测**：

       * 在 Google Trends 输入 3 个种子词。

       * **审计标准**：近 12 个月曲线是向上、持平还是暴跌？

       * **结论**：若持平或向上，证明“品类安全”。


  > **注意**：Day 0 不需要将任何社区原话填入主表。这一步只写在诊断结论的“核心假设”中。
  
   _【新增章节】Week 1+ 日常维护：动态关键词供血_

  **执行节奏**：每周一上午，30-60 分钟。

  **1. 社区原话捕捉 (Weekly)**

   * **动作**：深入上周探测到的活跃帖子，提取 3-5 个用户的真实提问原话。

   * **录入**：直接填入关键词主表 A 列，**来源 B 列选“社区挖掘”**。

   * **特权**：即使你查不到 Volume 和 KD，脚本也会自动将其标为 **💎社区高潜**，排入本周写作计划。

  **2. 趋势捕捉 (Weekly)**

   * **工具**：Exploding Topics 或 Google Trends。

   * **录入**：发现飙升话题，填入主表，**来源选“趋势词”**。

   * **特权**：脚本自动将其标为 **🚀趋势词**，实现即时插队。

## Step 5 — 关键词分桶（30 分钟）

**目标**：将 300-500 个关键词候选自动分类为 4 个执行桶。

操作步骤：

1. 所有来源的关键词导入 Google Sheets（`keyword-sheet-setup.gs` 生成的模板）
2. 填写 A-H 列数据：关键词 / 来源 / 月搜索量 / KD / CPC / Trends 比值 / DR差距
==以上几个数据，每个表导出的会有不同，比如有的表没有cpc，有的表没有DR差距，没法整合为一个表，我的建议是== 
3. I / J / K 列公式自动计算（意图词检测 / DR 过滤 / 分桶）
4. 对 CPC = 0 的词：复制词列表，让 Claude/GPT 批量判断是否有商业意图（`I2` 列可手动覆盖）
**Step 5 — 关键词分桶（精简高效版）**

  **核心逻辑：先定性标签，后统一补全指标，最后利用 VLOOKUP 自动化合龙。**

  _5.1 原始数据“打标” (Phase: Pre-processing)_

  在将 Ahrefs 的各处导出的 CSV 汇总前，先给它们盖章：

   1. **分头处理**：打开 Content Gap.csv、Competitor.csv 等原始文件。

   2. **新增 Source 列**：在最后一列手动加上表头 Source。

   3. **批量填充**：根据来源填入“内容缺口”、“竞品映射”等标签。

   4. **汇总**：将所有文件的“关键词”和“Source”两列内容粘贴到一个 Google Sheet 中（命名为 [标签表]）。

  _5.2 汇总去重 (Phase: Deduplication)_

   1. **全选** **[标签表]**。

   2. **执行去重**：数据 -> 数据清理 -> 移除重复项。

   3. **关键设置**：只勾选“关键词”列。这样你会得到一个唯一的关键词清单，且每个词都保留了它第一次出现的来源标签。

  _5.3 批量数据补全 (Phase: Batch Enrichment)_

   4. **复制**去重后的纯关键词清单。

   5. **Ahrefs 操作**：打开 **Keywords Explorer**，粘贴清单，点击查询。

   6. **标准化导出**：点击 **Export**。你将得到一个包含统一 Volume、Difficulty (KD) 和 CPC 的 CSV 文件。

   7. **创建** **[指标表]**：将导出的内容贴入 Google Sheet 的另一个工作表。

  _5.4 标签与指标“合龙” (Phase: Integration)_


  **第一步：将两个表放在同一个 Google Sheet 文件里**

   * 把 Ahrefs 导出的 CSV 内容全部粘贴到 指标表 中。

   * 确认 指标表 的 A 列是 **Keyword**。

  **第二步：使用 VLOOKUP 函数接回标签**

  在 指标表 的数据末尾（假设是 F 列），我们新增一列叫 **“Source”**。


   1. 在 **F2** 单元格输入以下公式：
   =VLOOKUP(A2, '标签表'!A:B, 2, FALSE)

   **公式解释**：

   A2：拿着当前行的关键词去寻找。

           * '标签表'!A:B：去“标签表”的 A 到 B 列范围里找。

           * 找到了之后，把“标签表”第 2 列（即 Source 列）的内容抓回来。

           * FALSE：要求精确匹配，一个字母都不能错。

   2. **向下填充**：双击 F2 单元格右下角的小方块，公式会自动填满整列。

       * _你会发现：无论 Ahrefs 导出的顺序怎么乱，标签都准确无误地回到了每个词身边。_

   8. **固化数据**：选中该列，Command + C 复制，然后 右键 -> 选择性粘贴 -> 仅粘贴值（消除公式）。

  _5.5 填入 GenGrowth 模板 (Phase: Final Setup)_

   9. **对齐列顺序**：调整 [指标表]，确保前五列顺序为：

       * A：关键词 (Keyword)

       * B：来源 (Source)

       * C：月搜索量 (Volume) —— _即 CSV 中的原始 Volume 字段_

       * D：KD (Difficulty)

       * E：CPC (CPC)

   2. **粘贴入主表**：将这五列粘贴到由 keyword-sheet-setup.gs 生成的“关键词主表”中。

   3. **忽略项**：Trends 比值 和 DR 差距 留空，除非遇到极其看重的核心词再手动抽查。

  _5.6 二段式权重审计_
  该步骤旨在过滤掉那些“KD 虽低但大厂霸屏”的死胡同。

   * **第一段（机器分桶）**：数据贴入后，观察 ⚡快速胜利 桶产出的名单。

   * **第二段（人工核准）**：

       * **对象**：只针对 ⚡快速胜利 和 🚀趋势词 中流量最高的前 20 个词。

       * **动作**：在 Ahrefs 中查看 **SERP Overview**。如果发现前 10 名被 DR 70+ 的大站垄断，且你没有明显的意图缝隙可钻。

       * **操作**：在主表 **G 列 (SERP平均DR)** 直接填入该词的平均 DR（如 75）。

       * **反馈**：脚本会自动将其从快速胜利降级为 **🎯战略词，保护内容产能不被浪费。

  ##### 5.7 杠杆调节（选填项说明）

   * **Trends 比值 (F 列)**：通常留空。若遇到来源不是“趋势词”但你判定其必火的词，填入 1.5 强制插队。

   * **SERP 平均 DR (G 列)**：通常留空。仅在上述“人工核准”阶段用于规避低KD，但实际竞争激烈的情况。

|---|---|---|

针对“快速胜利”桶里的词，增加一个人工审计的步骤：

  使用 Ahrefs 的 SERP 概览，看 Top 10 的第 3 到第 8 名：

   1. **看“外链数” (Backlinks)**：如果这些页面的外链都是 0，且 KD 低，说明**技术门槛确实低**。

   2. **看“站点类型”**：如果是垂直小站（DR < 40）能排进去，说明**权重门槛不存在**。

   3. **看“内容契合度”**：如果大厂排在那里，但文章标题跟你想做的词“只是沾边”，说明**意图门槛有缝隙**。
我们可以把 **DR 差距 (G列)** 的地位从“可选”更提升一步。

  **优化逻辑：**

   * 如果一个词 **KD < 10**，但 **Top 10 平均 DR 超过我们的站 50 分以上**。

   * 脚本自动将其从 ⚡快速胜利 降级为 🎯战略词（因为它不再是“快速”能拿下的了）。

   * 只有当 **KD 低 且 DR 差距合理（< 30）** 时，才是真正的 ⚡快速胜利。

## 常见问题

**Q：竞品主要靠品牌流量，Content Gap 没什么结果怎么办？**  
A：说明竞品在内容营销上也没做好，市场机会更大。直接从种子词拓展和社区挖掘入手，不要过度依赖竞品分析。

**Q：产品处于早期，GSC / GA4 数据不足 28 天怎么办？**  
A：用现有数据（哪怕只有 3 天）建立基线，并在模板备注里标注数据时间跨度。不要等到有足够数据才开始。

**Q：Ahrefs 付费功能用不到，有替代方案吗？**  
A：SEMrush 提供类似功能；免费替代方案：Google Search Console 提供流量基线，Ubersuggest / Keyword Surfer 浏览器插件可快速看 KD 估算，竞品 Top Pages 可用 SimilarWeb Free 粗估。精度会降低，但 Day 0 基线仍然可以建立。

**Q：Day 0 必须在一天内完成吗？**  
A：不是。建议上午完成 Step 0-3（产品理解 + 技术审计 + 流量基线 + 竞品分析），下午完成 Step 4-6（种子词 + 社区 + 分桶 + 目标设定）。关键是在 Week 1 Day 1 结束前产出"可增长性判断"，这个判断决定是否继续推进内容生产。
