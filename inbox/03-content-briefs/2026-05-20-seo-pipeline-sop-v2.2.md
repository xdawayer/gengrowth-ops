# 🚀 SEO 内容生产流水线 (v2.2 · 完整版)

> **核心哲学**：去机械化、数据闭环。通过“脚本闸门+意图折叠+结构化信息增益”，在保护关键词底层数据的同时，确保生产的高效与目标的精准对齐。
> **执行准则**：本规范内的所有操作步骤为**强约束**，未按步骤执行的文章在质检环节将直接拒收。

---

## 📊 第一部分：基础设施 - 选题登记表与数据保护

采用**三层数据模型**：数据层 (Ahrefs原表) -> 架构层 (主题集群表) -> 表现层 (选题登记表)。

### 1. 《选题登记表 v2.2》标准表头 (Schema)
> **注意**：列号以 `keyword-sheet-setup.gs` v3.0 为准。若有冲突，以字段名称为准。

| 列 | 字段名称 (Header) | 填充性质 | 功能说明 |
| :--- | :--- | :--- | :--- |
| **A** | **`page_id`** | 人工/递增 | 唯一 ID（如 `P-001`），全系统追踪主键。 |
| **B** | **`cluster_id`** | 下拉菜单 | 关联“主题集群表”，决定内链网归属。 |
| **C** | **`Track`** | 下拉菜单 | 线别：量产线 (Mass) / 精修线 (Refine)。 |
| **D** | **`Target Keyword`** | 人工输入 | A列主排名词。命中黑名单时该行自动标红。 |
| **E** | **`Associated Keywords`** | AI/人工 | 1+N 折叠词包。包含所有合并的长尾词。 |
| **F** | **`MSV (US)`** | **公式自动** | VLOOKUP 关键词主表，获取美国月搜索量。 |
| **G** | **`KD`** | **公式自动** | VLOOKUP 关键词主表，获取竞争难度。 |
| **H** | **`Intent`** | **公式自动** | 四层漏斗判定：Utility / Compare / Tutorial / Info。 |
| **I** | **`Tier`** | 下拉菜单 | 产能定级：T1 (重装) / T2 (标准) / T3 (占位)。 |
| **J** | **`page_role`** | 下拉菜单 | Pillar / Spoke / Tool / Wiki / Standalone。 |
| **Q** | **`Primary CTA`** | **公式映射** | **[V2.2 强化]** 自动关联工具页或 Newsletter 订阅。 |
| **R** | **`Status`** | 下拉菜单 | 待写 / 写作中 / 审核中 / 已发布 / 已合并(Archived)。 |
| **U** | **`us_share`** | **公式自动** | **[V2.2 新增]** 从集群表透传美国流量占比（高/中/低）。 |

### 2. 意图拆分与 1+N 语义折叠 (数据保护法则)
1. **自动意图判定**：在登记表的 `H列 (Intent)` 中，配置正则公式自动剥离意图（Utility/Compare/Tutorial/Info）。
2. **1+N 逻辑折叠（消灭同类相食）**：
   * 在同一 Intent 下，保留搜索量最大的作为 `Target Keyword (D列)`。
   * 其余变体词存入 `Associated Keywords (E列)`，无数量上限。
   * **数据保护**：在《关键词主表》中将变体行标记为 `merged_into_page_id`。**严禁从登记表中物理删除含有历史数据的行**，应将其状态改为“已合并”。

## 🧠 第二部分：核心判定标准矩阵 (决策去黑箱化)

### 1. 地区闸门：`us_share` 优先准则 (V2.2)
在建卡阶段，运营人员必须执行“美国流量优先”核查：
*   **`us_share = 低`**：禁止排入 P0 产能，原则上仅限 T3 极速处理。
*   **`us_share = 高/中`**：优先分配 T1/T2 精力。

### 2. 四层漏斗意图判定词典 (Intent Decision Matrix)
| 意图层级 | 用户心理 | 典型词根 | 页面使命 | 禁忌 (Red Flags) |
| :--- | :--- | :--- | :--- | :--- |
| **Utility** | “直接给我算出来。” | `test`, `quiz`, `calculator`, `tool` | 提供交互输入。 | ❌ 严禁写成长文。 |
| **Compare** | “帮我做决定。” | `vs`, `best`, `difference` | 权衡利弊，必须有对比表。 | ❌ 严禁只列举不比较。 |
| **Tutorial** | “第一步干什么？” | `how to`, `steps`, `guide` | 清晰的步骤图。 | ❌ 严禁缺乏编号步骤。 |
| **Info** | “解释这个概念。” | `meaning`, `what is` | 建立语义权威。 | ❌ 纯名词需人工核实。 |

### 3. 产能定级与审核标准 (Tiering Matrix)
| 定级 | 触发条件 | 人工深度 | 审核标准 |
| :--- | :--- | :--- | :--- |
| **T1 (重装)** | 核心商业词 / 高危敏感词 / Pillar | **45-60 min** | 极致 EEAT。校验免责声明、真实痛点。 |
| **T2 (主力)** | 正常行业词 / 明确搜索量的 Spoke | **15-20 min** | 中等。检查 AIO 饵块、H2 逻辑、向内链。 |
| **T3 (占位)** | 超长尾 / 边缘词 (占比 $\le 60\%$) | **3-10 min** | **机械扫描+批量审计**。确保表格/FAQ 存在。 |

### 4. 文章角色定义 (Page Role Matrix)
*   **Pillar (支柱)**：统领集群。必定 T1。
*   **Spoke (支撑)**：挂载在 Pillar 下。向上回链。
*   **Standalone (独立卫星)**：无法归簇的高优词（全站 $\le 10\%$）。**准入条件**：意图独立、物理无关联、高流量潜力。

## 🛠 第三部分：六步标准化生产流程 (Execution)

### STEP 1：准入建卡与地区闸门 (耗时: 2 分钟)
1. **核查 `us_share`**：低美国占比页面不得占据 T1 权重。
2. **心理安全红线**：若 `Psych Safety` 为 `Y`，强制加入医疗免责声明。

### STEP 2：实体主权与 EEAT 搜证 (耗时: 5 分钟)
1. **提取 Entity**：从 SERP 前 3 提取 5 个专业术语填入 N 列。
2. **透明度声明**：页脚必含 `Reviewed by AstrologyWiki Team`。

### STEP 3：信息增益注入
*   **【T1/T2】**：去 Reddit 挖掘真实痛点 (Friction) 填入 O 列。
*   **【T3】**：跳过搜证，执行“强制格式化增益”（表格+FAQ）。

### STEP 4：AI 组装 (v2.2 优化版)
```text
# 🟢 开始组装任务
Target Keyword: {D列词} | Associated Keywords: {E列词包}
Intent: {H列} | Tier: {I列} | CTA: {Q列} | Psych Safety: {M列}

[强制要求]
1. AIO 饵块：顶部 `## TL;DR`。第1段 100 字加粗直接回答意图。取消强制悬念句式，以深度摘要为主。
2. CTA 嵌入：根据 {Q列} 在正文合适节点植入转化按钮/链接。
3. T3 增益：强制生成 3x3 对比表及文末 ## FAQ。
```

### STEP 5：双向语义布线 (SOP 分离执行)
1. **日常发布 (向上回链)**：Spoke 文前 30% 指向 Pillar。
2. **Day 30 扫荡**：每月 1 次利用 Obsidian 进行跨簇/横向锚文本编织。

### STEP 6：发布后复盘与资产处置 [V2.2 新增]
发布后按周期核查：
*   **Day 14 (收录)**：未收录则手动提交抓取。
*   **Day 30 (排名)**：核心词未入 Top 100，调整 Entity 密度。
*   **Day 60 (处置)**：PV/美国流量不达标，评估合并或下架。
