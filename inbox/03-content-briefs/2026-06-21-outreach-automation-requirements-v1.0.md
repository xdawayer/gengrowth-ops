# 外联 SOP 自动化需求文档

## 一、自动化目标

**核心问题：** 当前 SOP 每月约需 **40–60 小时人工**（站点发现 + 筛选 + 个性化邮件 + 状态追踪 + 质量核查），高度依赖执行者记忆和手工操作，存在遗漏和不一致性风险。

**自动化目标：**
- 将重复性数据处理环节人工时间压缩 **≥ 70%**
- 关键节点建立自动提醒，消除"没人管"的信息断层
- 保留必须人工判断的环节（个性化内容创作、价格谈判），不过度自动化

## 四、各环节详细自动化需求

### 4.1 【P0】均分DR批量筛选自动化

**痛点：** 每个候选站点需手动查 Ahrefs DR + Dofollow Linked Domains → 手动计算均分DR → 手动对照分层标准判断 → 5 分钟/站点，100 个站点就是 8 小时。

**需求描述：**

输入：候选站点域名列表（CSV 或 Airtable 表格，一列域名）

处理逻辑：
1. 批量调用 Ahrefs API 拉取每个域名的 DR 和 Dofollow Linked Domains
2. 自动计算 均分DR = DR ÷ Dofollow Linked Domains
3. 应用分层判断公式（来自 SOP v1.1 1.5 节）：
   - DR 0–30：均分DR ≥ 0.06 且 LD ≤ 300 → 通过
   - DR 30–50：均分DR ≥ 0.05 且 LD ≤ 600 → 通过
   - DR 50–70：均分DR ≥ 0.03 且 LD ≤ 1500 → 通过
   - DR 70+：均分DR ≥ 0.02 且 LD ≤ 4000 → 通过
4. 输出结果列：通过 / 均分不足 / LD超限

输出：原表格新增三列（DR、均分DR、筛选结果），自动标红排除站点

**工具方案：**
- 方案 A（推荐）：Python 脚本 + Ahrefs API → 输出到 Google Sheets / Airtable
- 方案 B（低代码）：Airtable + Ahrefs API 集成 + Formula 字段自动计算

**验收标准：**
- 100 个站点 ≤ 10 分钟完成筛选
- 筛选结果与手工计算比对误差为零
- 每季度可一键重跑，更新数据

### 4.2 【P0】外展邮件序列 + 自动跟进

**痛点：** 150–250 封邮件全手工发送，跟进靠记忆，漏跟进率高。

**需求描述：**

**序列设计（三步）：**

| 步骤 | 触发时间 | 邮件内容 | 操作 |
|---|---|---|---|
| 邮件 1 | D+0 | Guest Post / Link Insertion 初次联系 | 人工撰写个性化内容（必须保留） |
| 邮件 2（跟进 1）| D+5，无回复自动触发 | "Just checking in..." 简短跟进 | 自动发送，可配置模板 |
| 邮件 3（跟进 2）| D+10，仍无回复 | 最后一次跟进，说明可提供其他合作形式 | 自动发送，可配置模板 |
| 停止条件 | 任意时间 | 对方回复（任意内容） | 自动停止序列，通知人工接管 |

**跟进邮件模板（需内置到工具）：**

跟进 1：
```
Subject: Re: [原始标题]

Hi [姓名],

Just following up on my previous email — wanted to check if 
you had a chance to review my pitch.

Happy to adjust the topic or angle if needed.

Best,
[作者名] | AstrologyWiki.com
```

跟进 2：
```
Subject: Re: [原始标题]

Hi [姓名],

One last follow-up — if guest posting isn't the right fit 
right now, I'd also be open to a link insertion in an 
existing article, or simply connecting for a future 
collaboration.

Either way, no pressure!

Best,
[作者名] | AstrologyWiki.com
```

**工具方案：** Instantly.ai（推荐）或 Lemlist
- 邮件 1 人工撰写后导入
- 跟进序列在工具内配置，全自动执行
- 对方回复后自动暂停序列，在统一收件箱提示人工处理

**验收标准：**
- 对方回复后序列 ≤ 5 分钟内自动停止（不再发后续邮件）
- 跟进邮件发出率 100%（不依赖人工记忆）
- 每封序列邮件可追踪打开率、点击率、回复率

### 4.3 【P0】外展状态自动同步到追踪表

**痛点：** 当前依赖人工在外链追踪表 CSV 里手动更新状态，容易遗漏，且 CSV 无法触发提醒。

**需求描述：**

状态字段：待联系 → 已发送 → 等待回复 → 已回复 → 谈判中 → 已付款 → 已上线 → 已拒绝

自动化规则：
1. **邮件发出 → 自动更新状态为"已发送"**（Instantly webhook → Airtable）
2. **对方回复 → 自动更新状态为"已回复"+ 时间戳**（邮件 webhook → Airtable）
3. **跟进 1 发出 → 自动更新"跟进次数+1"**
4. **D+14 仍无回复 → 自动标记"无响应/暂停"，发飞书/Slack 提醒负责人**
5. **付款确认 → 人工更新"已付款"，自动触发 D+14 验收提醒（检查链接是否上线）**

输出：Airtable 追踪表实时更新，飞书/Slack 每周推送状态汇总

**工具方案：** Instantly + Zapier/Make → Airtable

**验收标准：**
- 状态更新延迟 ≤ 1 小时（非实时但近实时）
- 无需人工更新"已发送"和"已回复"状态
- 每周自动推送各状态站点数量汇总

### 4.4 【P1】候选站点批量发现自动化

**痛点：** Google 搜索指令每月手动跑 5–10 条，复制粘贴 URL，重复劳动。

**需求描述：**

子需求 A：**Google 搜索指令定时执行**
- 预设 SOP 中所有搜索指令（约 15 条）
- 每月自动运行，拉取前 50 条结果 URL
- 去重（与已有追踪表对比，排除已联系站点）
- 输出：新候选站点列表，待人工判断相关性

子需求 B：**Ahrefs Link Intersect 定期执行**
- 每季度自动调用 Ahrefs API，对 3–5 个竞品执行 Link Intersect
- 排除已在追踪表中的域名
- 输出：新发现、链接多个竞品但未链接 astrologywiki 的域名清单

**工具方案：**
- A：SerpApi / ValueSERP + Python 定时任务（每月1号自动跑）
- B：Ahrefs API + Python + Cron

**验收标准：**
- 每月自动产出 ≥ 20 个新候选 URL，无需人工搜索
- 与追踪表自动去重，不产生重复联系

### 4.5 【P1】HARO 相关性过滤 + 分发自动化

**痛点：** Featured.com 每天推送大量记者请求，人工逐条判断相关性耗时 1h/天，且容易错过 2 小时黄金响应窗口。

**需求描述：**

处理逻辑：
1. 订阅 Featured.com / Qwoted 邮件推送
2. 自动解析邮件内容，提取每条请求的：主题、媒体名、截止时间
3. 关键词匹配过滤（相关词库：astrology / zodiac / spiritual / wellness / birth chart / chakra / tarot / horoscope / psychology / self-help / meditation）
4. 匹配命中 → 立即推送到飞书/Slack，附带：媒体名、请求原文、截止时间、建议响应作者（参考 SOP 4.2 四位作者分工）
5. 不匹配 → 静默忽略

输出格式（飞书/Slack 消息）：
```
📰 HARO 相关请求
媒体：Forbes
截止：2026-06-22 17:00 EDT（剩余 4h）
主题：[astrology and mental health trends]
建议作者：Julian Thorne
原文：[...请求内容...]
```

**工具方案：** Zapier / Make → Gmail 解析 → 关键词过滤 → 飞书 Bot

**验收标准：**
- 相关请求 ≤ 5 分钟内推送到飞书
- 误报率（不相关却推送）< 20%
- 漏报率（相关却未推送）< 10%

### 4.6 【P1】品牌提及 → 无链接自动提醒

**痛点：** Google Alerts 已设置，但收到提醒后需人工检查该页面是否有 dofollow 链接，容易遗漏。

**需求描述：**

1. Google Alerts 触发（关键词：AstrologyWiki / astrologywiki.com）
2. 自动抓取目标页面，检查是否存在指向 astrologywiki.com 的链接
3. **有链接** → 静默记录到"已提及+有链接"列表
4. **无链接** → 推送飞书提醒：页面 URL + 提及上下文 + 建议联系邮件草稿

**工具方案：** Zapier（Google Alerts → Webhook → Python 检查链接 → 飞书）

**验收标准：**
- 无链接提及 100% 推送提醒，不依赖人工每天检查

### 4.7 【P2】月度质量核查自动报告

**痛点：** 每月手动执行 SOP 核查清单 4–5 项，整理成报告需 3–4 小时。

**需求描述：**

每月1日自动生成报告，包含：

| 核查项 | 数据来源 | 自动化方式 |
|---|---|---|
| 新增 referring domains 列表 | Ahrefs API | 自动拉取，过滤 DR 0–5 |
| 锚文字分布 | Ahrefs API | 自动统计各类型比例，超 10% 精确匹配自动标红 |
| GSC 手动操作通知 | GSC API | 自动检查，有异常立即推送 |
| 可疑链接列表 | Ahrefs API | DR < 5 或 spam score 高 → 自动列出，待人工决策 disavow |
| 各类状态站点数 | Airtable | 自动统计：已联系/等待回复/已上线/已拒绝 |

输出：飞书文档 / Notion 自动生成月度外链报告，@负责人审阅

**工具方案：** Python + Ahrefs API + GSC API + Airtable API → 输出到飞书文档

**验收标准：**
- 每月1日自动推送，无需人工启动
- 报告包含所有核查项，人工仅需 30 分钟审阅决策

---

### 4.8 【P2】锚文字比例实时统计

**痛点：** 每新增5条外链需手动统计锚文字分布，判断是否超过10%精确匹配红线。

**需求描述：**

- 每当追踪表新增"已上线"链接时，自动重新计算锚文字分布
- 如精确匹配关键词 > 10% → 立即推送警告
- 仪表板实时显示当前锚文字分布饼图

**工具方案：** Airtable Automation + 公式字段 → 飞书 Bot 告警

---

### 4.9 【P3】断链自动监控

**需求描述：**

- Ahrefs Alerts 设置竞品域名的断链监控
- 当竞品有新断链出现时，自动推送飞书提醒，附：断链 URL + 来源页 DR + AstrologyWiki 替代页建议

**工具方案：** Ahrefs Alerts（已有功能，配置即可）→ 邮件 → Zapier → 飞书

## 七、保留人工环节（不建议自动化）

| 环节 | 保留原因 |
|---|---|
| 邮件个性化内容撰写 | AI 撰写质量不稳定，个性化是 Guest Post 成功率核心变量 |
| 候选站点相关性最终判断 | 需要人工结合业务判断，误判代价高 |

| Disavow 最终决策 | 误 disavow 风险大，必须人工审批 |
| HARO 回复内容撰写 | 2小时内响应且内容质量直接影响成功率，不宜纯自动化 |
