# 外联 SOP 自动化需求文档

## 一、自动化目标

**核心问题：** 当前 SOP 每月约需 **30–45 小时人工**（站点发现 + 筛选 + 联系方式查找 + 个性化邮件 + 谈判跟进 + 验收 + 状态追踪），高度依赖执行者记忆和手工操作，存在遗漏和不一致性风险。

**自动化目标：**
- 将重复性数据处理环节人工时间压缩 **≥ 70%**
- 关键节点建立自动提醒，消除"没人管"的信息断层
- 保留必须人工判断的环节（个性化内容创作、价格谈判），不过度自动化

---

## 二、当前人工操作全流程（8步）

> 追踪工具：Google Sheets（外链追踪表），以下简称"追踪表"

| #          | 环节            | 当前人工操作                                                                                                                            | 频率            | 估计耗时     |
| ---------- | ------------- | --------------------------------------------------------------------------------------------------------------------------------- | ------------- | -------- |
| **STEP 1** | 候选站点发现 + 初筛   | Ahrefs → 输入竞品域名 → Referring domains 页面 → 筛选 Dofollow + Dofollow linked domains 0–4K（直接在页面操作，不导出 Excel）→ 逐个点进网站，判断主题相关性、是否含赌博/成人内容 | 每批次           | 4–6h/月   |
| **STEP 1.5** | 数据记录 + 均分DR核查 | 将通过初筛的站点基础数据（DR、Dofollow LD、站点类型等）手动录入 Google Sheets 追踪表；表格公式自动计算均分DR；核查均分DR是否达标，不达标则排除，不进入 STEP 2 | 每个候选站点 | 分散 |
| **STEP 2** | 联系方式 + 投稿要求查找 | 在通过均分DR核查的网站上手动寻找 Contact / Write for Us / Guest Post 页面，记录联系邮箱和投稿要求                                                                  | 每个候选站点        | 1–2h/月   |
| **STEP 3** | 发送客制化邮件       | 根据网站内容和投稿要求，手动撰写个性化外联邮件，逐封发送                                                                                                      | 每封邮件 15–30min | 10–15h/月 |
| **STEP 4** | 跟进无回复站点       | 靠记忆或人工日历提醒，对无回复站点发送跟进邮件                                                                                                           | 随时            | 2–3h/月   |
| **STEP 5** | 回复处理（双路分叉）    | 收到回复后判断路线：**投稿路线** → 撰写并提交文章；**购买路线** → 进行价格谈判/砍价，确认外链细节                                                                          | 随时            | 分散       |
| **STEP 6** | 付款            | 谈妥后付款（需事先书面确认 dofollow 属性 + 上线 TAT）                                                                                               | 每次成交          | 分散       |
| **STEP 7** | 链接验收          | 确认链接上线 + 用 GSC/Ahrefs 核查是否被 Google 收录 + 核查链接是否为 dofollow                                                                          | 每次上线后         | 0.5h/条   |
| **STEP 8** | 追踪表全程记录       | 在 Google Sheets 追踪表中手动更新每个站点的进度状态                                                                                                 | 每次有进展         | 分散       |

## 四、各环节详细自动化需求

---

### 4.1 【P0】候选站点内容自动预判

**对应：** STEP 1 后半段（逐个打开网站判断）

**痛点：** Ahrefs 的 0–4K 过滤已完成数量收口，但之后仍需逐个手动打开网站查看内容，每站点 3–5 分钟，候选站点多时累计 4–6h/月。（均分 DR 已由 Google Sheets 公式自动计算，不在自动化范围内。）

**需求描述：**

输入：Ahrefs 筛选后的候选站点列表（域名列表）

处理逻辑：
1. 自动抓取每个域名的首页 + About 页 + Contact 页
2. 用关键词/分类模型判断：
   - 主题相关性（占星 / 灵性 / 健康 / 心理 / 女性生活 → 相关；电商 / 科技 / 新闻无关内容 → 不相关）
   - 是否含赌博 / 成人内容（命中则直接标记排除）
   - 是否有 "Write for Us" / "Guest Post" / "Contribute" 等标识
   - 是否为明显链接农场（大量外站链接 + 内容质量极低）
3. 输出评分表：相关性 / 黑名单内容 / 投稿入口 / 综合建议（优先联系 / 可联系 / 跳过）

输出：Google Sheets 追踪表新行，自动填入上述字段，人工录入 DR + LD 后表格公式自动计算均分DR，确认达标后决策是否进入 STEP 2

**工具方案：**
- 网页抓取：Firecrawl / Jina Reader API（支持批量抓取）
- 分类判断：GPT-4o mini（成本极低，适合大批量分类）
- 整合串联：Python 脚本 / Make

**验收标准：**
- 100 个候选站点 ≤ 20 分钟完成自动预判
- 主题相关性判断准确率 ≥ 85%（人工抽查 20 个验证）
- 赌博 / 成人内容命中率 ≥ 99%（漏判代价高）
- 人工不再需要逐个打开网站

---

### 4.2 【P0】联系方式自动查找

**对应：** STEP 2（手动找邮箱和投稿要求）

**痛点：** 每个候选站点都需要手动翻 Contact 页或 Write for Us 页，找邮箱耗时且成功率不稳定（有些站点不公开邮箱）。

**需求描述：**

输入：通过初筛的候选站点域名列表

处理逻辑：
1. 优先抓取站点 Contact / Write for Us / Guest Post 页面，提取可见邮箱地址
2. 如页面未暴露邮箱，调用 Hunter.io API 查询该域名的 editorial / info / marketing 邮箱
3. 同步抓取投稿要求页内容（字数要求 / 话题限制 / 稿酬说明），生成摘要
4. 输出：域名 + 联系邮箱 + 投稿要求摘要，写入追踪表对应行

**工具方案：**
- 邮箱查找：Hunter.io API（每月 25 次免费，$49/月 500 次）
- 备用：Apollo.io / Snov.io
- 投稿要求抓取：Firecrawl + GPT-4o mini 总结

**验收标准：**
- 联系方式查找成功率 ≥ 70%（含 Hunter.io 补充后）
- 投稿要求摘要自动生成，人工撰写邮件时直接参考，无需再次打开页面

---

### 4.3 【P0】外展邮件序列 + 自动跟进

**对应：** STEP 3（发送邮件）+ STEP 4（跟进）

**痛点：** 邮件全手工发送，跟进靠记忆，漏跟进率高。

**需求描述：**

**序列设计（三步）：**

| 步骤 | 触发时间 | 邮件内容 | 操作 |
|---|---|---|---|
| 邮件 1 | D+0 | Guest Post / Link Insertion 初次联系 | 人工撰写个性化内容（必须保留） |
| 邮件 2（跟进 1）| D+5，无回复自动触发 | "Just checking in..." 简短跟进 | 自动发送，可配置模板 |
| 邮件 3（跟进 2）| D+10，仍无回复 | 最后一次跟进，说明可提供其他合作形式 | 自动发送，可配置模板 |
| 停止条件 | 任意时间 | 对方回复（任意内容） | 自动停止序列，通知人工接管 |

**跟进邮件模板（需内置到工具）：**

跟进 1：
```
Subject: Re: [原始标题]

Hi [姓名],

Just following up on my previous email — wanted to check if 
you had a chance to review my pitch.

Happy to adjust the topic or angle if needed.

Best,
[作者名] | AstrologyWiki.com
```

跟进 2：
```
Subject: Re: [原始标题]

Hi [姓名],

One last follow-up — if guest posting isn't the right fit 
right now, I'd also be open to a link insertion in an 
existing article, or simply connecting for a future 
collaboration.

Either way, no pressure!

Best,
[作者名] | AstrologyWiki.com
```

**工具方案：** Instantly.ai（推荐）或 Lemlist
- 邮件 1 人工撰写后导入
- 跟进序列在工具内配置，全自动执行
- 对方回复后自动暂停序列，在统一收件箱提示人工处理

**验收标准：**
- 对方回复后序列 ≤ 5 分钟内自动停止（不再发后续邮件）
- 跟进邮件发出率 100%（不依赖人工记忆）
- 每封序列邮件可追踪打开率、点击率、回复率

### 4.4 【P0】外展状态自动同步到追踪表

**对应：** STEP 8（全程追踪表记录）

**痛点：** 当前依赖人工在 Google Sheets 追踪表里手动更新状态，容易遗漏，且表格无法主动触发提醒。

**需求描述：**

状态字段（建议标准化）：
```
待联系 → 已发送 → 跟进中 → 已回复 → 谈判中 → 投稿中 → 已付款 → 已上线 → 待验收 → 验收完成 → 已拒绝
```

自动化规则：
1. **邮件发出 → 自动更新状态为"已发送"**（Instantly webhook → Google Sheets）
2. **对方回复 → 自动更新状态为"已回复"+ 时间戳**
3. **跟进 1/2 发出 → 自动更新"跟进次数+1"**
4. **D+14 仍无回复 → 自动标记"无响应/暂停"，推送飞书提醒负责人**
5. **人工标记"已付款" → 自动触发 D+14 验收提醒（检查链接是否上线）**
6. **链接上线确认 → 自动触发 dofollow 属性检查 + Google 收录检查（见 4.5）**

输出：Google Sheets 追踪表实时更新，飞书每周推送状态汇总

**工具方案：** Instantly + Zapier/Make → Google Sheets

**验收标准：**
- 状态更新延迟 ≤ 1 小时
- 无需人工更新"已发送"和"已回复"状态
- 每周自动推送各状态站点数量汇总

---

### 4.5 【P1】链接验收自动化

**对应：** STEP 7（链接上线后验收）

**痛点：** 付款后需人工等待并主动检查链接是否上线、是否 dofollow、是否被 Google 收录，容易遗漏或推迟检查，增加被动损失风险。

**需求描述：**

触发条件：追踪表中"已付款"状态的站点，进入 D+1 开始监控

处理逻辑：
1. **链接上线检测**：每 24 小时调用 Ahrefs API 检查目标域名是否新增指向 astrologywiki.com 的链接
2. **dofollow 属性验证**：链接检测到后，自动核查 rel 属性，确认为 dofollow
3. **Google 收录检测**：调用 GSC API 的 URL 检查功能，确认链接所在页面已被收录
4. **任一异常 → 立即推送飞书告警**：nofollow 属性 / 超过 TAT 未上线 / 收录延迟超 14 天
5. **全部验收通过 → 自动更新追踪表为"验收完成"**

**工具方案：**
- 链接检测：Ahrefs API Alerts + Make
- GSC 收录检测：GSC API + Python
- 告警：飞书 Bot

**验收标准：**
- 链接上线后 24 小时内自动检测到
- dofollow / nofollow 自动判断，准确率 100%
- 超时未上线自动告警，不依赖人工记忆追催

### 4.6 【P1】竞品反链批量拉取

**对应：** STEP 1 前半段（Ahrefs 翻页筛选）

**痛点：** 在 Ahrefs 页面逐页翻看 Referring domains，手动筛选效率低，无法自动去重（排除已联系站点）。

**需求描述：**

- 每月自动调用 Ahrefs API，对 3–5 个竞品域名执行 Referring domains 批量拉取
- 自动应用筛选条件：Dofollow + LD 0–4K
- 与追踪表已有站点自动去重（排除已联系 / 已排除站点）
- 输出：新候选站点域名列表，进入 4.1 自动预判流程

**工具方案：** Ahrefs API + Python + Cron（每月 1 日自动触发）

**验收标准：**
- 每月自动产出候选站点列表，无需人工翻页
- 与追踪表自动去重，不产生重复联系

---

### 4.7 【P1】品牌提及 → 无链接自动提醒

**痛点：** Google Alerts 已设置，但收到提醒后需人工检查该页面是否有 dofollow 链接，容易遗漏。

**需求描述：**

1. Google Alerts 触发（关键词：AstrologyWiki / astrologywiki.com）
2. 自动抓取目标页面，检查是否存在指向 astrologywiki.com 的链接
3. **有链接** → 静默记录到"已提及+有链接"列表
4. **无链接** → 推送飞书提醒：页面 URL + 提及上下文 + 建议联系邮件草稿

**工具方案：** Zapier（Google Alerts → Webhook → Python 检查链接 → 飞书）

**验收标准：**
- 无链接提及 100% 推送提醒，不依赖人工每天检查

---

### 4.8 【P2】月度质量核查自动报告

**痛点：** 每月手动执行核查清单多项，整理成报告需 3–4 小时。

**需求描述：**

每月 1 日自动生成报告，包含：

| 核查项 | 数据来源 | 自动化方式 |
|---|---|---|
| 新增 referring domains 列表 | Ahrefs API | 自动拉取，过滤 DR 0–5 |
| 锚文字分布 | Ahrefs API | 自动统计各类型比例，超 10% 精确匹配自动标红 |
| GSC 手动操作通知 | GSC API | 自动检查，有异常立即推送 |
| 可疑链接列表 | Ahrefs API | DR < 5 或 spam score 高 → 自动列出，待人工决策 disavow |
| 各类状态站点数 | Google Sheets | 自动统计：已联系 / 等待回复 / 已上线 / 已拒绝 |

输出：飞书文档自动生成月度外链报告，@负责人审阅

**工具方案：** Python + Ahrefs API + GSC API + Google Sheets API → 输出到飞书文档

**验收标准：**
- 每月 1 日自动推送，无需人工启动
- 报告包含所有核查项，人工仅需 30 分钟审阅决策

---

### 4.9 【P2】锚文字比例实时统计

**痛点：** 每新增 5 条外链需手动统计锚文字分布，判断是否超过 10% 精确匹配红线。

**需求描述：**

- 每当追踪表新增"验收完成"链接时，自动重新计算锚文字分布
- 如精确匹配关键词 > 10% → 立即推送飞书警告
- 仪表板实时显示当前锚文字分布

**工具方案：** Google Sheets 公式 + Zapier 触发 → 飞书 Bot 告警

---

### 4.10 【P3】断链自动监控

**需求描述：**

- Ahrefs Alerts 设置竞品域名的断链监控
- 当竞品有新断链出现时，自动推送飞书提醒，附：断链 URL + 来源页 DR + AstrologyWiki 替代页建议

**工具方案：** Ahrefs Alerts（已有功能，配置即可）→ 邮件 → Zapier → 飞书

---

### 4.11 【未来扩展】HARO 相关性过滤 + 分发

> 当前流程未使用 HARO 渠道，此需求在决定开启 HARO 时再实施。

**需求描述：**

处理逻辑：
1. 订阅 Featured.com / Qwoted 邮件推送
2. 自动解析邮件内容，提取每条请求的：主题、媒体名、截止时间
3. 关键词匹配过滤（相关词库：astrology / zodiac / spiritual / wellness / birth chart / chakra / tarot / horoscope / psychology / self-help / meditation）
4. 匹配命中 → 立即推送到飞书，附带：媒体名、请求原文、截止时间
5. 不匹配 → 静默忽略

**工具方案：** Zapier / Make → Gmail 解析 → 关键词过滤 → 飞书 Bot

## 七、保留人工环节（不建议自动化）

| 环节 | 保留原因 |
|---|---|
| 邮件个性化内容撰写 | AI 撰写质量不稳定，个性化是 Guest Post 成功率核心变量 |
| 候选站点相关性最终判断 | 自动预判结果需人工确认，误判代价高 |
| 价格谈判 | 需要实时沟通和灵活决策 |
| 付款最终确认 | 财务动作必须人工执行 |
| Disavow 最终决策 | 误 disavow 风险大，必须人工审批 |

## 九、SOP 现有流程优化建议

> 以下是当前人工 SOP 中发现的风险点和可改进项，与自动化无关，建议在人工操作阶段直接改进。

### 9.1 付款前确认清单（高优先级）

**当前风险：** 目前无标准化的付款前确认清单，存在付款后收到 nofollow 链接或对方迟迟不上线的风险。

**建议：** 每次付款前，邮件往来中必须书面确认以下三项：

```
□ 链接类型：dofollow（明确写入邮件，非口头）
□ 上线 TAT：≤ X 天（具体日期）
□ 链接位置：正文内（非页脚/侧边栏）
```

未收到书面确认，不付款。

---

### 9.2 谈判砍价标准化

**当前风险：** 每次砍价依赖临场判断，没有锚定策略，可能多付。

**建议：** 建立标准谈判规则：

| 对方报价区间 | 第一次反报 | 底线 |
|---|---|---|
| $50 以下 | 直接接受 | — |
| $51–$150 | 报 70% | 90% |
| $151–$300 | 报 60% | 80% |
| $300 以上 | 重新评估是否值得 | 酌情 |

---

### 9.3 已联系站点去重机制

**当前风险：** 没有明确提到联系前检查追踪表，可能对同一站点重复发邮件，损害品牌形象。

**建议：** STEP 2 开始前，在追踪表中搜索该域名，确认未出现在任何非"已排除"状态后，再进入 STEP 3。自动化上线后此步骤由系统处理。

---

### 9.4 投稿后跟进节点

**当前风险：** 投稿后到文章发布有时差（几天到数周），没有明确的跟进机制，文章可能石沉大海。

**建议：** 在追踪表新增"投稿日期"字段，投稿后 D+7 无上线消息，主动发跟进邮件询问状态。

---

### 9.5 Google 收录验证方法规范

**当前风险：** "确认链接被 Google 收录"的具体方法未标准化，`site:` 搜索不可靠（Google 对 site: 的结果不保证完整性）。

**建议：** 标准验收方法：
1. GSC → URL 检查 → 输入链接所在页面 URL → 查看"已收录"状态（最准确）
2. 备用：Ahrefs → 输入目标页面 → 查看 Organic keywords（有排名说明已收录）

---

### 9.6 追踪表字段补全建议

**当前缺失字段（建议新增）：**

| 字段 | 用途 |
|---|---|
| 投稿要求摘要 | 撰写邮件时快速参考，不用重新打开页面 |
| 联系方式来源 | 记录邮箱是从哪里找到的（Contact页/Hunter/公开）|
| 初次联系日期 | 计算跟进 D+5、D+10 的准确时间 |
| 投稿日期 | 触发 D+7 发布跟进 |
| 锚文字（实际使用）| 月度锚文字比例统计的数据来源 |
| 链接 URL（精确到页面）| 验收和 GSC 检查用 |
| 收录确认日期 | 验收完成记录 |
