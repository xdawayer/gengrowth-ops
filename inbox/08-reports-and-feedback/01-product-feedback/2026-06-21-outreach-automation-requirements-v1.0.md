# 外联 SOP 自动化需求文档

## 一、自动化目标

**核心问题：** 当前 SOP 每月约需 **30–45 小时人工**（站点发现 + 筛选 + 联系方式查找 + 个性化邮件 + 谈判跟进 + 验收 + 状态追踪），高度依赖执行者记忆和手工操作，存在遗漏和不一致性风险。

**自动化目标：**
- 将重复性数据处理环节人工时间压缩 **≥ 70%**
- 关键节点建立自动提醒，消除"没人管"的信息断层
- 保留必须人工判断的环节（个性化内容创作、价格谈判），不过度自动化

### 2.0 站点类型 × 合作形式 × 联系方式 分类

当前外联涉及多种站点类型，不同类型在操作路径和自动化范围上存在显著差异：

| 站点类型           | 典型合作形式                | 联系方式        | 是否付费      | 操作特殊点                                                                             |
| -------------- | --------------------- | ----------- | --------- | --------------------------------------------------------------------------------- |
| **Blog（免费投稿）** | Guest Post            | 邮箱 / Web 表单 | 否         | 需提供选题提案 + 完整文章；部分站点有投稿指南须遵守（字数/格式/链接数量限制）                                         |
| **Blog（付费投放）** | Sponsored Post / 赞助文章 | 邮箱 / Web 表单 | 是         | 部分站点有固定价目表；需确认文章是否标注"Sponsored"（影响 SEO 权重）                                        |
| **Blog（链接插入）** | Link Insertion        | 邮箱          | 是         | 需提前指定目标页面 URL + 建议锚文字，对方在已有文章中插入，无需提供全文                                           |
| **工具目录**       | 产品 Listing 提交         | 平台注册 + 提交表单 | 免费 / 付费均有 | 需准备：产品名称 / 一句话简介 / 详细描述 / 截图 / 分类标签 / 定价；部分平台要求在 astrologywiki 首页或页脚添加反向链接或徽章后才审核 |
| **新闻站**        | 投稿 / 付费文章             | 邮箱 / 编辑投递页  | 免费 / 付费   | 话题需具备新闻价值或时效性；通过率低，适合以 PR 角度（数据 / 趋势 / 专家观点）切入                                    |
| **内容社区**       | 账号直接发帖                | 平台注册        | 免费        | 无需联系站主，直接注册发布；需遵守社区规范，不可过度自我推广（否则帖子被删或封号）                                         |

> ⚠️ 同一站点可能同时支持多种合作形式（如 Blog 既接受免费投稿也接受链接插入），STEP 1 判断时优先记录所有可用合作形式，STEP 3 根据策略选择切入角度。

### 2.1 操作全流程（8步）

| #            | 环节                   | 当前人工操作                                                                                                                                                                                                         | 频率            | 估计耗时     |
| ------------ | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------- | -------- |
| **STEP 1**   | 候选站点发现 + 类型初判        | Ahrefs → 输入竞品域名 → Referring domains 页面 → 筛选 Dofollow + LD 0–4K（直接在页面操作）→ 逐个点进网站，判断：① 主题相关性 ② 是否含赌博/成人内容 ③ 站点类型（Blog / 工具目录 / 新闻站 / 内容社区）④ 可用合作形式（免费投稿 / 付费投放 / 链接插入 / 产品提交）                                    | 每批次           | 4–6h/月   |
| **STEP 1.5** | 数据录入 + 均分DR核查        | 将站点基础数据（DR、Dofollow LD、站点类型、合作形式）录入追踪表；表格公式自动计算均分DR；不达标则排除，不进入 STEP 2                                                                                                                                          | 每个候选站点        | 分散       |
| **STEP 2**   | 联系方式 + 合作要求查找（按类型分支） | **Blog / 新闻站**：寻找 Contact / Write for Us 页，判断联系方式类型：(A) 直接邮箱 → 记录邮箱和投稿要求进入 STEP 3；(B) Web 表单 → 记录表单 URL 和必填字段，手动提交表单，无法进入邮件序列 **工具目录**：找到产品提交入口 URL，记录所需素材清单（简介/截图/标签），执行提交子流程（见 2.2）**内容社区**：注册账号，直接发布内容，无需联系 | 每个候选站点        | 1–2h/月   |
| **STEP 3**   | 发送客制化邮件（仅限邮箱联系类站点）   | 根据站点内容、合作形式和投稿要求，手动撰写个性化外联邮件；合作形式不同，邮件模板不同：Guest Post（提供选题思路）/ Link Insertion（指定目标页 + 锚文字）/ Sponsored（询价或回应价目表）/ 新闻站（提供独家数据或专家视角）                                                                              | 每封邮件 15–30min | 10–15h/月 |
| **STEP 4**   | 跟进无回复站点              | 靠记忆或人工日历提醒，对无回复站点发送跟进邮件（仅限邮件外联路径）                                                                                                                                                                              | 随时            | 2–3h/月   |
| **STEP 5**   | 回复处理（多路分叉）           | 收到回复后根据对方意向判断路线：**① 接受免费投稿** → 按投稿要求撰写文章提交；**② 要求付费投放** → 进入价格谈判，确认费用 / dofollow / 标注方式 / TAT；**③ 接受链接插入** → 确认目标页 + 锚文字 + 价格；**④ 拒绝** → 更新追踪表状态为"已拒绝"，结束流程                                                    | 随时            | 分散       |
| **STEP 6**   | 付款                   | 付款前书面确认：dofollow 属性 + 上线 TAT + 链接位置（正文内）+ 是否标注 Sponsored；确认后付款                                                                                                                                                 | 每次成交          | 分散       |
| **STEP 7**   | 链接验收                 | 确认链接上线 + GSC URL 检查是否被 Google 收录 + Ahrefs 核查是否为 dofollow                                                                                                                                                       | 每次上线后         | 0.5h/条   |
| **STEP 8**   | 追踪表全程记录              | 在 Google Sheets 追踪表中手动更新每个站点的进度状态                                                                                                                                                                              | 每次有进展         | 分散       |

|---|---|---|---|
| **P0** | 候选站点内容自动预判（替代 STEP 1 逐个打开） | 节省 4–6h/月 | 中 |

| **P1** | 链接验收自动化（替代 STEP 7） | 消除遗漏，付款后自动监控上线 | 中 |
| **P1** | 竞品反链批量拉取（Ahrefs API 替代手动翻页） | 节省逐页翻看 Ahrefs 的时间，自动去重 | 中 |

| **P3** | 断链自动监控 | 发现被动机会 | 低 |
| **未来扩展** | HARO 相关性过滤 + 分发 | 未来开启 HARO 渠道时启用 | 中 |

## 四、各环节详细自动化需求

---

### 4.1 【P0】候选站点内容自动预判

**对应：** STEP 1 后半段（逐个打开网站判断）

**痛点：** Ahrefs 的 0–4K 过滤已完成数量收口，但之后仍需逐个手动打开网站查看内容，每站点 3–5 分钟，候选站点多时累计 4–6h/月。（均分 DR 已由 Google Sheets 公式自动计算，不在自动化范围内。）

**需求描述：**

输入：Ahrefs 筛选后的候选站点列表（域名列表）

处理逻辑：
1. 自动抓取每个域名的首页 + About 页 + Contact 页 + Write for Us 页（若存在）
2. **站点类型识别**：
   - Blog（内容以文章为主）/ 工具目录（含 Submit / Add a Tool 等入口）/ 新闻站（有编辑署名 + 时效内容）/ 内容社区（有用户账号体系 + 讨论区）
3. **主题相关性判断**：
   - 相关：占星 / 灵性 / 健康 / 心理 / 女性生活 / 自我成长
   - 排除：电商 / 纯科技 / 赌博 / 成人内容（命中黑名单词库则直接标记排除）
4. **可用合作形式识别**：
   - 有"Write for Us"/"Guest Post"页 → 标记"接受投稿"
   - 有"Sponsored"/"Advertise"/"Partner"页 → 标记"付费投放"
   - 有"Submit Tool"/"Add Listing"入口 → 标记"产品提交"
   - 无任何投稿入口但主题相关 → 标记"可尝试链接插入"
5. **联系方式类型识别**：
   - 找到明文邮箱 → 标记"邮箱"+ 记录地址
   - Contact 页为表单（无邮箱暴露）→ 标记"Web 表单"+ 记录表单 URL
   - 工具目录平台（需注册账号提交）→ 标记"平台提交"+ 记录提交入口 URL
6. 是否为明显链接农场（大量外站链接 + 内容质量极低）→ 标记排除
7. 输出综合建议：优先联系 / 可联系 / 跳过，并注明建议切入合作形式

输出：Google Sheets 追踪表新行，自动填入：站点类型 / 主题相关性 / 合作形式 / 联系方式类型 / 邮箱或表单URL / 综合建议；人工录入 DR + LD 后表格公式自动计算均分DR，确认达标后决策是否进入 STEP 2

**工具方案：**
- 网页抓取：Firecrawl / Jina Reader API（支持批量抓取）
- 分类判断：GPT-4o mini（成本极低，适合大批量分类）
- 整合串联：Python 脚本 / Make

**验收标准：**
- 100 个候选站点 ≤ 20 分钟完成自动预判
- 主题相关性判断准确率 ≥ 85%（人工抽查 20 个验证）
- 赌博 / 成人内容命中率 ≥ 99%（漏判代价高）
- 人工不再需要逐个打开网站

### 4.2 【P0】联系方式自动查找

**对应：** STEP 2（手动找邮箱和投稿要求）

**痛点：** 每个候选站点联系方式形态不一（直接邮箱 / Web 表单 / 平台注册提交），需逐个人工判断和记录，耗时且容易遗漏关键字段。

**需求描述：**

输入：通过初筛的候选站点域名列表（已由 4.1 输出站点类型标注）

处理逻辑（按联系方式类型分支）：

**A. 邮箱类站点（Blog / 新闻站）：**
1. 抓取 Contact / Write for Us / Guest Post 页面，提取明文邮箱地址
2. 如页面未暴露邮箱，调用 Hunter.io API 查询该域名的 editorial / outreach / info 邮箱
3. 同步抓取投稿要求页内容，GPT-4o mini 生成摘要（字数要求 / 话题限制 / 链接数量 / 稿酬说明 / 文章独家性要求）
4. 输出：邮箱地址 + 投稿要求摘要 → 写入追踪表，进入 4.3 邮件序列

**B. Web 表单类站点：**
1. 抓取 Contact 页，识别表单并提取必填字段列表（通常包括：姓名 / 邮箱 / 网站 URL / 选题 / 文章简介）
2. 记录表单 URL + 必填字段清单 → 写入追踪表
3. **注意：Web 表单类站点无法接入自动邮件序列，须人工访问页面手动填写提交；提交后无法自动跟进，需人工记录提交日期并设置日历提醒**

**C. 工具目录 / 内容社区类站点：**
1. 提取产品提交入口 URL（如 /submit、/add-tool 等）
2. 识别所需提交素材清单（名称 / 简介 / 截图 / 分类 / URL 等）
3. 若平台要求添加反向链接/徽章，标注该前置条件
4. 输出：提交入口 URL + 素材清单 → 写入追踪表，进入 2.2 工具目录提交子流程（人工执行）

**工具方案：**
- 邮箱查找：Hunter.io API（每月 25 次免费，$49/月 500 次）/ 备用 Apollo.io
- 表单识别 + 字段提取：Firecrawl + GPT-4o mini
- 投稿要求摘要：Firecrawl + GPT-4o mini

**验收标准：**
- 邮箱类站点联系方式查找成功率 ≥ 70%（含 Hunter.io 补充后）
- Web 表单类自动识别表单 URL + 必填字段，人工提交时无需再次分析页面
- 投稿要求摘要自动生成，人工撰写邮件时直接参考
- 工具目录类自动输出提交入口 + 素材清单

### 4.3 【P0】外展邮件序列 + 自动跟进

**对应：** STEP 3（发送邮件）+ STEP 4（跟进）

**适用范围：仅限联系方式为邮箱的站点。** Web 表单类和工具目录类站点不进入此流程。

**痛点：** 邮件全手工发送，跟进靠记忆，漏跟进率高；不同合作形式的初次邮件切入角度完全不同，需要标准化模板框架。

**需求描述：**

**序列设计（三步）：**

| 步骤 | 触发时间 | 邮件内容 | 操作 |
|---|---|---|---|
| 邮件 1 | D+0 | 初次联系（按合作形式选用对应模板框架，见下方） | 人工撰写个性化内容（必须保留） |
| 邮件 2（跟进 1）| D+5，无回复自动触发 | "Just checking in..." 简短跟进 | 自动发送 |
| 邮件 3（跟进 2）| D+10，仍无回复 | 最后一次跟进，提出备选合作形式 | 自动发送 |
| 停止条件 | 任意时间 | 对方回复（任意内容） | 自动停止序列，通知人工接管 |

---

Would this be a good fit? Happy to send a full outline first.

Best,
[作者名] | AstrologyWiki.com

- 工具内可区分不同合作形式的序列（Guest Post / Link Insertion / Sponsored），便于数据分析各类型转化率

### 4.4 【P0】外展状态自动同步到追踪表

**对应：** STEP 8（全程追踪表记录）

**痛点：** 当前依赖人工在 Google Sheets 追踪表里手动更新状态，容易遗漏，且表格无法主动触发提醒。

**需求描述：**

状态字段（建议标准化）：
```
待联系 → 已发送 → 跟进中 → 已回复 → 谈判中 → 投稿中 → 已付款 → 已上线 → 待验收 → 验收完成 → 已拒绝
```

自动化规则：
1. **邮件发出 → 自动更新状态为"已发送"**（Instantly webhook → Google Sheets）
2. **对方回复 → 自动更新状态为"已回复"+ 时间戳**
3. **跟进 1/2 发出 → 自动更新"跟进次数+1"**
4. **D+14 仍无回复 → 自动标记"无响应/暂停"，推送飞书提醒负责人**
5. **人工标记"已付款" → 自动触发 D+14 验收提醒（检查链接是否上线）**
6. **链接上线确认 → 自动触发 dofollow 属性检查 + Google 收录检查（见 4.5）**

输出：Google Sheets 追踪表实时更新，飞书每周推送状态汇总

**工具方案：** Instantly + Zapier/Make → Google Sheets

**验收标准：**
- 状态更新延迟 ≤ 1 小时
- 无需人工更新"已发送"和"已回复"状态
- 每周自动推送各状态站点数量汇总

### 4.5 【P1】链接验收自动化

**对应：** STEP 7（链接上线后验收）

**痛点：** 付款后需人工等待并主动检查链接是否上线、是否 dofollow、是否被 Google 收录，容易遗漏或推迟检查，增加被动损失风险。

**需求描述：**

触发条件：追踪表中"已付款"状态的站点，进入 D+1 开始监控

处理逻辑：
1. **链接上线检测**：每 24 小时调用 Ahrefs API 检查目标域名是否新增指向 astrologywiki.com 的链接
2. **dofollow 属性验证**：链接检测到后，自动核查 rel 属性，确认为 dofollow
3. **Google 收录检测**：调用 GSC API 的 URL 检查功能，确认链接所在页面已被收录
4. **任一异常 → 立即推送飞书告警**：nofollow 属性 / 超过 TAT 未上线 / 收录延迟超 14 天
5. **全部验收通过 → 自动更新追踪表为"验收完成"**

**工具方案：**
- 链接检测：Ahrefs API Alerts + Make
- GSC 收录检测：GSC API + Python
- 告警：飞书 Bot

**验收标准：**
- 链接上线后 24 小时内自动检测到
- dofollow / nofollow 自动判断，准确率 100%
- 超时未上线自动告警，不依赖人工记忆追催

## 九、SOP 现有流程优化建议

> 以下是当前人工 SOP 中发现的风险点和可改进项，与自动化无关，建议在人工操作阶段直接改进。

### 9.1 付款前确认清单（高优先级）

**当前风险：** 目前无标准化的付款前确认清单，存在付款后收到 nofollow 链接或对方迟迟不上线的风险。

**建议：** 每次付款前，邮件往来中必须书面确认以下三项：

```
□ 链接类型：dofollow（明确写入邮件，非口头）
□ 上线 TAT：≤ X 天（具体日期）
□ 链接位置：正文内（非页脚/侧边栏）
```

未收到书面确认，不付款。

---

### 9.2 谈判砍价标准化

**当前风险：** 每次砍价依赖临场判断，没有锚定策略，可能多付。

**建议：** 建立标准谈判规则：

| 对方报价区间 | 第一次反报 | 底线 |
|---|---|---|
| $50 以下 | 直接接受 | — |
| $51–$150 | 报 70% | 90% |
| $151–$300 | 报 60% | 80% |
| $300 以上 | 重新评估是否值得 | 酌情 |

---

### 9.3 已联系站点去重机制

**当前风险：** 没有明确提到联系前检查追踪表，可能对同一站点重复发邮件，损害品牌形象。

**建议：** STEP 2 开始前，在追踪表中搜索该域名，确认未出现在任何非"已排除"状态后，再进入 STEP 3。自动化上线后此步骤由系统处理。

---

### 9.4 投稿后跟进节点

**当前风险：** 投稿后到文章发布有时差（几天到数周），没有明确的跟进机制，文章可能石沉大海。

**建议：** 在追踪表新增"投稿日期"字段，投稿后 D+7 无上线消息，主动发跟进邮件询问状态。

---

### 9.5 Google 收录验证方法规范

**当前风险：** "确认链接被 Google 收录"的具体方法未标准化，`site:` 搜索不可靠（Google 对 site: 的结果不保证完整性）。

**建议：** 标准验收方法：
1. GSC → URL 检查 → 输入链接所在页面 URL → 查看"已收录"状态（最准确）
2. 备用：Ahrefs → 输入目标页面 → 查看 Organic keywords（有排名说明已收录）

---

### 9.6 追踪表字段补全建议

**当前缺失字段（建议新增）：**

| 字段 | 用途 |
|---|---|
| 投稿要求摘要 | 撰写邮件时快速参考，不用重新打开页面 |
| 联系方式来源 | 记录邮箱是从哪里找到的（Contact页/Hunter/公开）|
| 初次联系日期 | 计算跟进 D+5、D+10 的准确时间 |
| 投稿日期 | 触发 D+7 发布跟进 |
| 锚文字（实际使用）| 月度锚文字比例统计的数据来源 |
| 链接 URL（精确到页面）| 验收和 GSC 检查用 |
| 收录确认日期 | 验收完成记录 |

## 📝 审核反馈（Claude Ops / 2026-06-23）

> 4 路 fan-out + Codex 交叉审核。跨文档的战略层与优先级见同目录 `2026-06-23-automation-requirements-cross-review.md`。本节为本文档专属问题。
> **业务拆解（站点类型 ×合作形式分类、四种邮件模板、§九 SOP 优化）质量很高，建议保留；问题在"何时上 / 用什么底座 / 合规"。**

### 🔴 硬伤 1：新域名沙盒期跑自动冷邮件 = 拿主域信誉去赌

§4.3 把邮件序列标"P0 / 难度低"，但全文未提送达率、发信域名、邮箱预热、合规——这恰是冷外联最易翻车处：

- **必须用独立发信域名**，绝不能用 astrologywiki.com 主域发；一旦被标垃圾，正常商务邮件也进垃圾箱。
- **新域名须预热 2–4 周**才能发 → §六路线图"Phase 1 一周内（06-22~06-28）上线邮件序列"**不现实**，应增设 Phase 0 预热期。
- **CAN-SPAM（美国 B2B 不豁免）+ GDPR（欧盟站长）**：模板缺真实物理地址、一键退订、suppression list。
- 发信域名 + 多个 Workspace 邮箱的成本**未进 §五成本表**（实际还要 +域名/邮箱费）。
- §四模板里"I've been a reader of..."若不属实应删——冷邮件里假装熟悉最伤。

### 🔴 硬伤 2（SEO 政策风险）：把"买 dofollow 链接"流程化 = 积累风险证据库

§五/§4.5/§九把"付费投放 + dofollow + 砍价"做成带验收、带告警的标准流程，§9.1 还写"dofollow 未书面确认不付款"。Google 政策明确：付费/赞助链接应标 `rel="sponsored"` 或 `nofollow`。**自动化做得越完整、证据链越完整，风险越集中**；这张外链表会变成"敏感风险资产"。

**修正**：验收字段别只盯 dofollow，改看 `link_live / rel 属性 / 上下文相关性 / referral 潜力`；付费内容默认接受 sponsored/nofollow；§9.1 确认清单改为上线日期 / 正文位置 / 披露标记 / 是否可编辑 / 永久保留 / 退款条款。打开率/点击率受隐私追踪污染，应改看 reply rate / 有效落地 / GSC 后续提升。

### 🟡 成本与可行性

- **Ahrefs API 被低估**（§五）：Lite 虽带 API，但月额度有限、每请求仅返 10 行；§4.6 竞品反链批量拉取会迅速烧光额度或产生超额费。上线前先用 limits-and-usage 实测一个竞品要耗多少 units。
- **Hunter.io**：年付已降至 ~$34/月（成本表可注明）；"成功率≥70%"应限定为"邮箱类站点 + Hunter 补充后，以实测为准"。
- **§4.1 验收指标**："赌博/成人命中≥99%"靠词库会漏软色情/隐晦赌博，建议"词库 + LLM 双判 + 高风险样本人工复核"，口径改为"人工抽查漏判率 ≤1%"；样本 20 太小，置信区间过宽。

### 🟡 逻辑漏洞（状态机）

- **§4.4 状态机是单链表，表达不了 §2.1 STEP5 的四条分叉**（OUT-1）：免费投稿不经"已付款"、链接插入不经"投稿中"、"已拒绝"可发生在任意点——均未定义转移。
- **非邮件三类站点无验收闭环**（OUT-2，死路）：Web 表单类 / 工具目录类不进邮件序列，而 §4.5 验收只由"已付款"触发 → **免费 Web 表单投稿、免费工具目录提交永远进不了验收监控**；内容社区类全程无状态定义。建议 §4.5 触发条件从"仅已付款"扩展为"已付款 OR 投稿中/已提交 → 上线监控"，并为三类各定义状态子流。
- **人工邮件1 → Instantly 衔接未定义**（OUT-4）：谁触发导入、D+5/D+10 从哪一刻起算，直接影响"跟进发出率 100%"。
- **§一"消除人工更新遗漏" vs §4.4 只覆盖邮件态**：谈判中/投稿中/已上线仍需人工手填，承诺与设计不符。

### 🟢 应提前做（与自动化无关，零成本）

**§九 SOP 人工优化**（付款前确认清单、砍价标准化、去重机制、收录验证规范）—— 立刻能落地、直接防损失，建议从"附录"提到首批执行，优先级高于任何自动化。

### 验收标准

"省 70%"无 baseline 快照与测量方法；"dofollow 判断准确率 100%"不可达不可测——每条验收补四要素（baseline / 样本量 / 测法 / 签字人）。
