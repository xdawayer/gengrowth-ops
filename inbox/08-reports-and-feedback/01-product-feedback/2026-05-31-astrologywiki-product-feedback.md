

# SEO 技术需求文档：提升站点 DR 的产品与技术配合需求



---

## 核心逻辑

```
DR 提升 = 外链质量 × 外链数量 × 站点技术健康度

· 外链能不能拿到      → 作者可信度（需求1）+ 可嵌入资产（需求4）
· 外链效果能不能放大   → 社交分享卡片（需求5）
· 内容能不能被收录排名 → 收录机制（需求2）+ 页面速度（需求3）+ 技术健康（需求8）
· 内容权威能不能被识别 → 结构化数据（需求7）+ 内链能力（需求6）
```

**当前站点状态（用于评估紧迫性）：**

- DR：0
- 已收录页面：约 23 个
- sitemap 上次被 Google 读取：约 7 周前（滞后）
- 手动请求索引：受每日约 10 个的配额限制，无法支撑产出速度

## 🔴 P0 — 直接决定外链与收录能否成立

### 需求 1：作者页（Author Page）系统

**需求描述** 为四位虚拟作者（Elena Vane / Julian Thorne / Aditi Sharma / Marcus Orion）各建立独立的作者页，展示资质、经历、专长领域；并在每篇文章页头标注作者署名，链接到对应作者页。

**功能要点**

- 每位作者一个独立 URL（如 `/author/elena-vane`）
- 作者页包含：头像、资质介绍、专长领域、该作者撰写的文章列表
- 文章页头显示作者署名 + 头像，可点击跳转至作者页
- 作者页支持后续添加社交链接（Medium / 外部资料）

**为什么是 P0**

- 外链建设（Guest Post、Digital PR）全部以作者名义进行
- 没有作者页，作者在网络上是"幽灵"，外展邮件可信度大幅降低
- 是 Google E-E-A-T 评估的核心要求，直接影响内容权威性判断

**验收标准**

- 四位作者均有可访问的独立作者页
- 文章页头作者署名可点击跳转
- 作者页含完整资质信息

---

### 需求 2：sitemap 自动更新机制 + Google Indexing API 接入

**需求描述** 确保新文章发布后自动进入 sitemap，并接入 Google Indexing API，实现新文章发布时自动通知 Google，绕开 GSC 手动请求索引每日约 10 个的配额限制。

**功能要点**

- 新文章发布后，sitemap 自动更新并包含新 URL 及正确的 `<lastmod>` 时间
- 接入 Google Indexing API，文章发布/更新时自动推送 URL 通知
- 后台可查看 API 推送状态和结果

**为什么是 P0**

- 当前 sitemap 已 7 周未被 Google 重读，新文章收录持续滞后
- 手动请求索引每天仅约 10 个配额，无法支撑团队的内容产出速度
- Indexing API 是技术团队才能搭建的自动化能力，能从根本上解决收录滞后

**验收标准**

- 新文章发布后 24 小时内出现在 sitemap 中
- Indexing API 成功推送，后台可见推送日志
- 新文章平均收录时间显著缩短

---

### 需求 5：封面图支持 + Open Graph 社交分享卡片配置

**需求描述** 每篇文章支持设置封面图，并正确配置 Open Graph（OG）与 Twitter Card 标签，确保文章链接被分享到任何平台时，显示完整的图文预览卡片。

**功能要点**

- 文章编辑界面支持上传/设置封面图
- 自动生成并配置 OG 标签（og:title / og:description / og:image / og:url）
- 配置 Twitter Card 标签
- 封面图支持设置 Alt 文字和自定义文件名
- 图片自动压缩 + WebP 格式输出

**为什么是 P0**

- 外链分享时，有图文卡片的点击率是纯文字的 2-3 倍
- 占星/灵性内容在 Pinterest 流量巨大，封面图是进入该渠道的入场券
- Google Discover 几乎只推送带封面图的文章
- 直接放大所有外链战术的效果（外链建设是当前重点工作）

**验收标准**

- 文章可设置封面图，含 Alt 文字
- 在 Facebook / Twitter / Slack 等平台分享链接时正确显示图文卡片
- 图片自动压缩至合理大小（建议 ≤200KB）

## 🟠 P1 — 影响排名表现与被动外链获取



### 需求 4：可嵌入 / 可分享的工具资产

**需求描述** 将"星盘计算器 / 气场测试工具"做成可被外部网站嵌入（embed）、可独立分享的形式，并在嵌入代码中自带回链到主站。

**功能要点**

- 工具提供可嵌入的 iframe / embed 代码
- 嵌入代码中自带 dofollow 回链指向主站
- 工具有独立的可分享 URL
- （可选）嵌入时显示 "Powered by AstrologyWiki" 品牌标识 + 链接

**为什么是 P1**

- 免费工具是最强的 Link Bait（外链诱饵）之一
- 其他网站嵌入工具时自然产生回链，是被动获取高质量外链的最佳方式
- 需要产品做"可嵌入"的工程设计，SEO 无法自行实现

**验收标准**

- 工具可通过 embed 代码嵌入第三方网站
- 嵌入后包含指向主站的 dofollow 链接
- 工具有独立可分享 URL

---

### 需求 6：内链管理能力

**需求描述** 让编辑能在正文中方便地插入内链，提供"相关文章推荐"的半自动机制，并支持自定义锚文字。

**功能要点**

- 编辑器支持快速搜索站内文章并插入内链
- 支持自定义锚文字
- （可选）基于主题相关性的"相关文章"推荐
- 后台可查看每篇文章的内链入站/出站情况

**为什么是 P1**

- 内链是新站传递主题权威、提升整站收录的核心机制
- 当前内链靠手动且不规范（存在链接堆在文末、锚文字不统一等问题）
- 好的内链工具让编辑高效、规范地建立站内链接网络

**验收标准**

- 编辑器内可搜索并插入站内文章链接
- 支持自定义锚文字
- 内链插入流程顺畅，无需手动复制 URL

## 给 PM 的排期建议

若资源有限，建议按以下顺序推进——这三个是"不做就会卡住其他 SEO 工作"的瓶颈型需求：

1. **需求 1（作者页系统）** —— 外链建设的前提，不做则外展寸步难行
2. **需求 2（sitemap 自动更新 + Indexing API）** —— 解决当前收录滞后的核心问题
3. **需求 5（封面图 + Open Graph）** —— 低成本，但放大所有外链战术效果
