#### 自动化需求交叉审核（6/23）

4路并行审核（架构对齐/技术核查/成本风险/逻辑一致性）+ Codex 独立第二意见，识别出**三个硬伤**：

| 硬伤 | 问题描述 | 影响 |
| --- | --- | --- |
| 🔴 **索引文档核心假设不成立** | `urlInspection.index.inspect` API **只读**，不能"自动请求 Google 收录"；Google Indexing API 仅限招聘/直播两类，普通文章使用违反政策；IndexNow 对 Google 无效；sitemap ping 端点已于 2023 年底废弃 | 整条"自动提交收录"逻辑建在假起点上，需回炉 |
| 🔴 **新域名冷邮件 = 拿主域信誉做赌注** | 外联文档未设独立发信域名 + 2-4 周预热，"一周上线"不现实 | 直接发可能导致主域名进黑名单 |
| 🔴 **买 dofollow 链接留痕 = 积累 SEO 风险证据库** | 将付费 dofollow 采购流程化并留纸质记录，违反 Google 政策且有损失排名风险 | 应改用 rel=sponsored/nofollow 或不留明文记录 |

**审核建议**：

> **需求拆解质量高，底座需要重选。** 约17个子环节中，只有"邮件发送/送达"（Instantly）一项真正需要 SaaS；其余全部可改造成 flow-mvp 形态——本地 `.mjs` 脚本 + Claude Code subagent + git 里的 CSV/markdown。月成本从 $122–167 降至约 **$37**（仅 Instantly）。

**6月前真正该做的（优先级重排）：**
1. **索引收录监控 + 超期告警**（去掉假动作，只做 sitemap 校验 + GSC 只读监控 + 人工点按清单）——≈$0，纯只读
2. **外联文档 SOP 人工优化**（付款前确认清单 / 去重 / 收录验证规范）——$0，当天落地
3. **外联轻量跟进提醒**（本地脚本，不碰 Instantly 全栈）——近$0

**本次研究的一个重要方法论补充：上升趋势词 vs 平稳趋势词**

哥飞原模型偏好上升词（时间窗口竞争），但我们找到的达标词均为长期平稳词——这不是缺陷，而是**结构性盲区**：工具/娱乐类词 CPC 偏低导致主流 SEO 从业者长期不覆盖；大站靠域名权重溢出排名，页面级 KD 因此维持低位。平稳词的核心优势是需求确定性高、无时间窗口压力、竞争结构可预测。
