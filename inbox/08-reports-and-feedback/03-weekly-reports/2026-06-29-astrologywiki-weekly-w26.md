### 模块二：工具页深度审计 — 发现 P-1 渲染Bug（6/23-6/24）

**这是本周最重要的技术发现，优先级高于一切。**

**问题确认（2026-06-24，双重验证）：**
用 curl 抓到的450-650字说明文字 + FAQ schema，**只存在于服务端预渲染的初始 HTML 里**。JS 加载完成后，React 组件会把这部分内容整体替换为计算器表单 UI，最终渲染页面只剩 H1 → 副标题 → 表单 → 嵌入代码框 → 页脚，**说明段落和 FAQ 文字完全不出现**。

验证路径：① 浏览器截图（JS 执行后）vs curl 原始 HTML 对比；② GSC「网址检查」→「已抓取页面」截图——**Google 渲染引擎看到的结果与浏览器一致，同样无内容**。

**影响范围：** 目前只截图验证了 `birth-chart-calculator` 一个页面，但 16 个工具页大概率共用同一套 React 模板，同样的 bug 很可能全部存在。

**对已规划工作的影响：**
> P-1 修复之前，下面所有 P0-P4 的工作即使做了，对 SEO 和用户都是零效果：
- **P0**（修 FAQ schema 与可见内容不匹配）——内容都被替换了，schema 匹不匹配无意义
- **P1**（建工具间互链矩阵）——外链权重传进来之后，工具页排名还是没有内容支撑
- **P2**（补面包屑）、**P3**（内容生产线反链）、**P4**（补最薄内容页面）——同理

**修复需求（给前端工程团队）：** 让说明文字 + FAQ 成为 React 组件渲染输出的一部分，跟计算器表单同屏持久显示，不能依赖服务端预渲染快照。

**工具页审计其他发现（P-1修复后依次处理）：**

| 问题 | 范围 | 优先级 |
| --- | --- | --- |
| FAQ schema 已植入但内容与可见 H2 逐字不匹配 | 15/16 页面（saturn-return-calculator 是唯一正确参照）| P0 |
| 工具页之间跨工具互链几乎为零 | 15/16 页面零互链 | P1（**有7月中截止，见下**）|
| 面包屑 BreadcrumbList 全部缺失 | 16/16 页面 | P2 |
| Slug 优化窗口期（零/低成本）正在关闭 | 16个工具页 | 尽快确认，窗口2-4周 |

**P1 的截止时间约束：** 外链冲刺计划的链接全部指向首页，权重要传到工具页的唯一路径是内部链接。**P1（互链矩阵）必须在7月中旬前完成**，否则已投入的外链预算在工具页层面会延迟兑现，错过7-9月排名压制释放窗口。

### 模块四：竞品分析 — Arcaniva & Askaseer（6/24）

**核心结论（直接影响 Astrologywiki 优先级判断）：**

- **Arcaniva**：Dec2025→Apr2026 从约1,000到4,000+访问/月（4倍增长）。驱动是：低摩擦旗舰工具（`/en/yes-or-no-tarot/`，零输入门槛）+ 逐张塔罗牌单实体内容矩阵 + 持续外链建设（5.9K 反向链接/302 RD）。品牌词0%，完全依赖泛词排名——结构脆弱。最近28天流量结构骤变为印度94%+停留3秒，疑似被低质量流量污染。
- **Askaseer**：增长几乎纯靠短时间集中获得204个 RD，自然关键词仅20个，内容深度很小。
- **共同验证**：两站增长都主要靠外链/反向链接，不靠内容深度 → **进一步支持 P4（内容深度）降级、P1（互链矩阵）升级** 的判断。

### 模块五：运营自动化需求文档（6/22-6/23）

产出两份需求文档并完成交叉审核，发现**三个致命硬伤**：

| 硬伤 | 问题描述 |
| --- | --- |
| 🔴 **索引文档核心假设不成立** | `urlInspection.index.inspect` API 只读，无法"自动请求收录"；Google Indexing API 仅限招聘/直播；IndexNow 对 Google 无效；sitemap ping 端点2023年已废弃返回404 |
| 🔴 **新域名冷邮件 = 主域信誉风险** | 外联文档未设独立发信域名 + 2-4周预热，"一周上线"不现实 |
| 🔴 **dofollow 采购留痕 = SEO 风险证据库** | 将付费 dofollow 采购流程化并留纸质记录，违反 Google 政策 |

**审核建议（已明确）：** 需求拆解质量高，底座重选。约17个子环节中，只有"邮件发送/送达"（Instantly）一项真正需要 SaaS；其余改造成本地脚本 + git CSV，月成本从 $122-167 降至约 **$37**。

**6/29 前真正该做的（优先级重排）：**
1. 索引收录监控 + 超期告警（去掉假动作，只做 GSC 只读监控 + 人工点按清单）
2. 外联 SOP 人工优化（付款前确认清单 / 去重 / 收录验证规范）
3. 外联轻量跟进提醒（本地脚本，不碰 Instantly 全栈）
