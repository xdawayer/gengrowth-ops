# Aura 系列 4 篇 v8 标准质量审计

## TL;DR

把 `inbox/05-blog/` 下 4 篇 2026-05-18 的 aura 文章（blue / purple / yellow / aura-colors pillar）放进 `gengrowth-flow-mvp` v8 pipeline 的质量门槛跑了一遍。结论：**4/4 needs-rewrite**。

原因不是单点 bug，是这一批稿子写的时候 v8 标准还不存在，所以同步存在 5 个共性问题：

1. **AI 痕迹密集**（"operational reality" / "operational trade-off" 单 batch 出现 7+ 次，自动识别为单 prompt 指纹）
2. **RL1 红线大面积失守**（电磁场、脉轮活化、"thyroid-area activity"、"energy field will literally collapse" 等被当机理性事实写）
3. **科技隐喻 SaaS 化**（"high-bandwidth"、"antenna"、"energy battery"、"system error"、"physical avatar"）
4. **v8 结构缺失**（缺 Quick Reference Table / 编号 Reflection Prompts / Related Reading / Take Action 等 4 个 H2，全员 720-1108 词 vs 标准 1500-1800）
5. **唯一 PASS 的轴**：SEO 关键词密度（exact match 全员 ≤ 6 / 8，没触 RL5）

**建议路径**：4 篇全部走 v8 prompt 完整重生成，**原文保留不动**（per wzb 规则）。新版本写到平行文件（如 `2026-05-21-blue-aura-meaning-v8.md`），人工对比后由 wzb 决定替换时机。Pillar（aura-colors）需要先在 flow-mvp 起 Pillar template 变体——目前只有 Definition template。

## 方法学

每篇文章过两层闸：

**Layer 1 — Binary 结构检查**（本地 node 跑）
- H1/H2/H3 计数
- Word count
- `target_keyword` whole-word 计数（v8 RL5 上限 8）
- Loose entity match 计数（无 "meaning" 后缀，比如 `\bblue aura\b`）
- AI tells 计数（`operational reality` / `operational trade-off` / `delve` / `navigate the landscape` / `crucial`）

**Layer 2 — 质量 subagent 评分**（4 篇并行 fan-out）

5 个维度 1-10 打分（统一 rubric）：
- `natural_english`：AI tells / em-dash-as-comma / 空心 superlatives → 低分；紧致人写 → 高分
- `friction_landing`：是否回应搜索摩擦，还是绕进神秘话术
- `seo_density`：keyword 数量是否在 [3, 8] 区间
- `claim_safety`：RL1 临床、empirical_claim、invented_term、authority_overreach、named_author hallucination 计数
- `v8_schema_gap`：与 v8 Definition template（H1 + 7 H2 + 4-row QRT + numbered prompts + 1500-1800w）的距离

**Verdict 规则**：
- `needs-rewrite`：claim_safety < 5 **或** hallucinations > 3 **或** seo_density < 4
- `needs-revision`：任一分 < 7
- `acceptable`：全分 ≥ 7 **且** hallucinations ≤ 3

## 修复路径

### 推荐：v8 prompt 完整重生成（不动原文）

按 wzb 的 [[workflow-content-via-prompt-only]] 规则——所有 fix 走 prompt 改 → LLM 重生成，不在原文上手工改字。每篇执行：

| 步骤 | 工具 | Cost |
|---|---|---|
| 1. Entity passport RAG | `gg-entity-passport.mjs --emit-rag` | 5 min/篇 |
| 2. Friction mine | synth fallback / 等 [[b-prime-2-reddit-oauth]] 上线 | 0 / 1h |
| 3. Obsidian RAG | `gg-obsidian-rag.mjs --vault-dir ...` | 2 min/篇 |
| 4. SERP cache | `gg-serp-snapshot.mjs` 或 WebSearch+WebFetch | 5 min/篇 |
| 5. 渲染 v8 Phase 1 prompt | `gg-content-draft.mjs --page-id ... --phase 1` | 1 min/篇 |
| 6. 粘 Claude + GPT 双 LLM | 手动（wzb） | 10 min/篇 × 2 |
| 7. Phase 2 validate | `_phase2-validate.mjs` | 1 min/篇 × 2 |
| 8. 写到平行文件 | `2026-05-21-<entity>-v8.md`（原文保留） | 自动 |

预期产出：3 篇 v8 zero-waiver definition 文章（blue / purple / yellow）。

### Pillar（aura-colors）单独处理

当前 v8 只有 Definition template，没有 Pillar template。两条路：

- **A）先 ship 3 个 definition 重生成，pillar 延后**（推荐）。Pillar 重生成需要先做：
  - 起 `pillar.prompt.md` template（H2 = "What is X" / "Common Children Overview" / "How They Relate" / "Quick Reference Map" / "Navigation: Which One to Read First" / "Take Action"）
  - 起 `_phase2-validate.mjs --template pillar` 分支
  - 起 pillar 专属 RAG（child entity index）
- **B）pillar 暂走通用 Definition 标准**（不推荐——会强行扭曲 pillar 调性）

### 风险 / 留坑

1. **Loose kw saturation 没人查**：yellow 篇 `\byellow aura\b`（无 meaning）= 17，但 v8 RL5 只查 `yellow aura meaning`。可能需要 **P-9-001 patch**：RL5 加 "loose entity name" 二级阈值（如 ≤ 15）。
2. **pillar template 缺失**：见上。
3. **图片资产**：原文有 `![[Blue Aura Meaning01.webp]]` 等 Obsidian 嵌入。v8 prompt 不生成图片引用，重生成的版本要么不带图，要么在 Phase 2 之后人工补 obsidian-embed。建议加 P-9-002：v8 prompt 加可选 "image placeholder slots"。
4. **内部链接**：原文用 `[Link to aura colors]` 占位，v8 prompt 输出 `[[<TBD-internal-link: ...>]]`——这两种占位风格不一样，需要人工对齐。
5. **`operational ___` 自动检测**：iterate-prompt-checks.mjs 当前不查 `operational` 词族。考虑加 P-9-003：把 `operational (reality|trade-off|sense|necessity|protocol|strategy)` 加入 AI tells lite 检测列。

## 决策待 wzb

| 选项 | 含义 | 风险 |
|---|---|---|
| A | 3 个 definition 重生成（blue/purple/yellow），pillar 延后 | 量产线 3 篇 zero-waiver 进 prod 候选 |
| B | 先做 pillar template + 4 篇一起重生成 | 周期长，但批量交付完整 |
| C | 暂不动，把这份 audit 作为"已知技术债"归档 | 量产线被 4 篇低质量稿子定锚 |

推荐 **A**，理由：
- 3 篇 definition 走 v8 已验证管线，cost 低
- Pillar 是更大工作（要起新 template），值得另起 task block
- 不重生成 = ops repo `inbox/05-blog/` 里这 4 篇会成为后续批量 ship 的"反例锚"，新作者会学到 v8 之前的范式
