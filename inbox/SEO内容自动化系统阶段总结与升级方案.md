# SEO内容自动化系统阶段总结与升级方案

## 1. 现有SOP流程说明
目前已在 Make.com 环境中跑通全链路自动化流程，具体节点如下：
* **触发源**：监控 Google Sheets 关键词列。
* **搜索检索**：调用 SerpApi 获取 Google 搜索排名前列的竞品链接。
* **数据抓取**：利用 Jina AI 提取竞品网页正文。
* **容错处理**：部署 Ignore 模块，确保死链或反爬网页不中断流程。
* **内容缝合**：将多篇竞品资料聚合为参考包。
* **AI生成**：由 Gemini 3.1 根据参考资料生成包含 H2/H3 的 SEO 大纲。
* **数据回填**：大纲自动写回 Google Sheets 指定列。

## 2. 核心成果与运行状态
* **技术闭环**：解决了 API 404 报错、JSON 格式转义（换行符/引号导致的崩溃）等核心技术阻塞。
* **运行效能**：测试批次运行成功。已确认系统具备处理批量关键词的能力。
* **资源监控**：目前已知单次运行的操作数消耗量，已针对高频触发场景进行了防爆设置。

## 3. 下一步功能扩展建议
计划在现有系统基础上，接入 Ahrefs 数据与 Obsidian 知识库，形成“选词-分析-沉淀”的闭环：

### Ahrefs 接入（关键词初筛）
* **利用现有套餐**：Lite版本（每月10万API积分）。
* **自动化逻辑**：输入核心词后，API 自动筛选 KD <= 20 且月流量 >= 500 的相关长尾词。
* **输出增益**：将筛选出的长尾词直接喂给 Gemini，强制要求在生成的 H2/H3 中覆盖这些高价值词汇。

### Obsidian 接入（文档自动化同步）
* **路径**：通过 GitHub 模块将生成的 Markdown 文件推送至仓库。
* **同步逻辑**：本地 Obsidian Git 插件自动拉取远程仓库。
* **预期效果**：系统生成的深度 SEO 大纲将自动出现在本地笔记库的 Inbox 文件夹中，无需手动复制粘贴。

## 4. 待决策/沟通事项
1.  **资源扩容**：根据后续关键词量级，评估是否需额外购买 Make.com 的 Operation 额度。
2.  **选词标准**：确认 Ahrefs API 的过滤参数（KD 与 Volume 的具体阈值）。
3.  **结构锁定**：确认当前 Gemini 输出的大纲格式是否符合团队标准。
