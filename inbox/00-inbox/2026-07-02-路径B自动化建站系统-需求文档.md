# 路径B自动化建站系统 PRD

## 一、背景与目标

### 1.1 问题

路径B（新词新站）的核心竞争力是**速度**——从发现趋势词到网站上线必须在48小时内完成，过了窗口期竞争者涌入，机会消失。

当前全人工流程的瓶颈：
- 需要人工盯 Google Trends，容易漏词
- 每次建站需要2-4小时手动操作
- 验证流程依赖人工逐步查询，容易跳步骤（ai story generator教训：漏掉手动SERP核查）
- 没有流水线化，单人难以同时跑多个候选词

### 1.2 目标

**从「人工发现→人工验证→人工建站」变为「AI自动监控→自动验证→人工审核1次→自动建站」**

- 趋势词发现时间：从"碰运气盯"缩短到实时（≤2小时延迟）
- 人工介入时间：每个候选词 ≤15分钟（只做go/no-go判断）
- 建站到上线时间：人工审核通过后 ≤30分钟自动完成

---

## 二、完整流程与自动化分层

```
[Module 1] 趋势监控（全自动）
      ↓ 发现候选词，自动触发
[Module 2] 自动验证（全自动）
      ↓ 生成验证报告
[Module 3] 人工审核（人工，≤15分钟）
      ↓ 点击「确认上线」
[Module 4] 自动建站（全自动）
      ↓ 网站上线后持续
[Module 5] 排名追踪（全自动）
```

---

## 三、各模块功能需求

### Module 1：趋势监控

**触发方式**：每2小时抓取一次 Google Trends「Trending Now」列表。

> 注：Google Trends 当前界面分三个入口：Home / Explore / Trending Now。Trending Now 是 Google 主动展示当前搜索量爆发词的模块，无需预先输入词，每个词条自带估算搜索量（如"500K+ searches"），是路径B词发现的核心数据源。

**输入**：
- 监控地区：美国（英语站目标市场）
- 监控频率：每2小时抓取一次 Trending Now 列表

**触发条件**（同时满足）：
- Trending Now 出现新词条
- 该词条估算搜索量 ≥ 50,000（过滤掉量级太小的词）
- 该词条过去7天内未在 Trending Now 出现过（排除反复上榜的周期性热词，如每年固定节日）

**输出**：
- 推送通知（微信/Slack），包含：
  - 词条名称
  - 当前趋势量级（对标哪个锚词的百分比）
  - Google Trends相关查询列表（自动抓取）
  - 趋势曲线截图
  - 自动触发 Module 2

---

### Module 2：自动验证

收到 Module 1 触发后，自动依次执行：

**Step 1：数据查询**
- 调用 DataForSEO API 或 Ahrefs API：查 Volume、KD、CPC
- ⚠️ KD=0 且 Volume>10,000：自动标记红色警报，强制进入 Step 3 SERP核查

**Step 2：KGR计算**
- 自动执行 Google 搜索 `allintitle:"候选词"`
- 抓取结果数量
- 计算 KGR = allintitle结果数 ÷ Volume
- 标注：KGR < 0.25 → 绿色；0.25-1 → 黄色；>1 → 红色

**Step 3：SERP前10构成分析**
- 自动抓取 Google 前10名 URL
- 查询每个 URL 的域名 DR（via Ahrefs/Moz API）
- 识别超级平台特征（ChatGPT、Google自身、Amazon、Wikipedia、Reddit等）
- 判断规则：
  - 前3名有超级平台 → ❌ 直接否决，不进入人工审核
  - 前10名平均 DR > 70 → ❌ 否决
  - 前10名有内页排名 → ✅ 加分项

**Step 4：Google Trends相关查询意图分析**
- 自动抓取相关查询列表
- AI分类：每条相关查询标注「工具需求」/「信息消费」/「其他」
- 计算工具需求占比：< 30% → ❌ 否决

**输出：验证报告**（自动发送给人工审核者）

```
═══════════════════════════════
词条：[候选词]
发现时间：[时间]
═══════════════════════════════
【量级】趋势热度相当于 chatgpt 的 X%
【数据】Volume: X  |  KD: X  |  CPC: $X
【KGR】X（状态：绿/黄/红）
【SERP】前10平均DR: X | 超级平台占位: 有/无 | 内页比例: X%
【意图】工具需求相关查询占比: X%
【结论】建议：✅ 值得做 / ⚠️ 待确认 / ❌ 否决
═══════════════════════════════
[点击确认上线] [点击跳过]
```

---

### Module 3：人工审核

**人工只需判断两件事（≤15分钟）**：

1. **意图核实**：手动搜索这个词，看前5条搜索结果——用户真的在找工具/内容吗？
2. **Go/No-Go**：点击「确认上线」或「跳过」

**辅助判断参考**（Module 2已自动提供）：
- SERP截图
- 相关查询意图分类

> 注意：Module 2已过滤掉明显的超级平台垄断词和意图错误词，人工审核阶段不需要重新核查这些——只需要对"⚠️待确认"的情况做最终判断。

---

### Module 4：自动建站

人工点击「确认上线」后，自动执行：

**Step 1：域名注册**
- 调用 Namecheap/GoDaddy API
- 自动注册 `{候选词去空格}.com`（或.net/.io按优先级）
- 如已被注册：自动尝试次选域名并通知人工确认

**Step 2：内容生成**
- 调用 Claude/GPT-4 API，基于以下模板生成：
  - 首页 H1/H2/H3 结构
  - 工具描述文案（300-500字）
  - FAQ（5条，问答格式）
  - Meta title / Meta description
  - README.md（用于 GitHub）
- 输入参数：候选词 + Google Trends相关查询（提供语义场）

**Step 3：建站部署**
- 从预设模板仓库 fork 新仓库（GitHub API）
- 将生成内容写入对应 md/yml 文件
- 触发 Cloudflare Pages 自动部署（已配置 devops）
- 等待部署成功（通常2-5分钟）

**Step 4：外链基础设置**
- 自动创建 GitHub Profile 仓库，写入 README（含网站链接）
- 自动提交 Google Search Console 收录请求（GSC API）
- 推送通知：网站已上线，附URL

**整体耗时目标：≤30分钟**

---

### Module 5：排名追踪

**自动执行（每日）**：
- 拉取 GSC 数据：点击量、曝光量、平均排名
- 监控候选词排名变化
- 触发条件报警：
  - 首次出现 GSC 数据 → 通知
  - 日点击量突破 100 → 通知
  - 排名进入前10 → 通知
  - 连续7天无点击（且已被索引）→ 通知（可能需要人工介入）

## 四、技术栈要求

| 模块 | 推荐方案 | 说明 |
|---|---|---|
| 趋势监控 | Google Trends API（非官方）/ SerpAPI Trends | 官方API限制较多，可用第三方封装 |
| 关键词数据 | DataForSEO API | 性价比高，Volume/KD/CPC一次调用 |
| SERP抓取 | SerpAPI / DataForSEO SERP | 抓取前10名URL+DR |
| 域名注册 | Namecheap API | 支持自动化注册 |
| 内容生成 | Claude API（claude-sonnet-4-6） | 按词义生成结构化内容 |
| 建站部署 | GitHub API + Cloudflare Pages | 已有devops流程，直接复用 |
| GSC提交 | Google Search Console API | 需OAuth授权 |
| 通知推送 | 微信企业机器人 / Slack Webhook | 推送验证报告和上线通知 |
| 调度系统 | GitHub Actions（已禁用）→ 改本地 cron / Railway cron | 见团队已知问题：GHA账单禁用 |

## 五、人工介入节点（共3处）

| 节点 | 触发条件 | 人工操作 | 耗时 |
|---|---|---|---|
| **审核** | Module 2完成验证报告 | Go/No-Go判断 | ≤15分钟 |
| **域名确认** | 首选域名已被注册 | 从候选列表选一个 | ≤5分钟 |
| **异常处理** | 建站失败/部署报错 | 排查并重试 | 视情况 |

---

## 六、不在本系统范围内的事项

- **路径A（astrologywiki等长青词站点）**：本系统专为路径B趋势词设计，路径A的关键词研究、内容生产、外链建设流程不纳入本系统
- **变现接入**：Google AdSense申请需人工操作（需3天以上流量积累后才能申请），不在自动化范围
- **内容更新**：初版建站后，如需追加内容、优化排名，属于后期运营，不在本系统范围

---

## 七、验收标准

| 指标 | 目标 |
|---|---|
| 趋势词发现延迟 | ≤2小时（从Google Trends出现到收到通知） |
| 验证报告生成时间 | ≤10分钟 |
| 人工审核时间 | ≤15分钟/词 |
| 建站上线时间（审核通过后） | ≤30分钟 |
| 误报率（否决掉的好词） | ≤20%（允许漏掉部分机会，但不能把明显好词过滤掉） |
| 漏报率（放过去的坏词） | ≤10%（超级平台垄断词、意图错误词不能通过自动验证） |

---

## 八、待决问题

1. **Google Trends监控API**：官方API对自动化查询有限制，需确认第三方方案（SerpAPI/PyTrends）的稳定性和成本
2. **模板仓库**：需要提前准备好可复用的建站模板（Nuxt.js或其他框架），这是整个系统的前置依赖
3. **DataForSEO账号**：需申请并充值，按查询量计费，估算每月成本
4. **执行人**：Module 3人工审核由谁负责？需要7×24小时响应能力（趋势词随时出现）
5. **路径B启动决策**：本系统建设需要2-4周开发时间，是否现在启动，取决于路径B策略是否已拍板
