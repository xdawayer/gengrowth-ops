AI Builder 晚报｜2026-06-23

1. 今日值得沉淀

1）评测能力正在成为 Agent 落地的核心门槛
内容：Aaron Levie 提出，模型进步、垂直领域后训练、应用层 Agent 改进，以及企业自动化真正能不能落地，底层都越来越依赖 evals。关键不再只是“会不会接模型”，而是能不能把真实工作流拆成可验证、可迭代的评测体系。
沉淀价值：这是企业 Agent 从 demo 走向交付的判断框架，适合拿来定义 GenGrowth 未来的 AI 服务方法论。
后续用途：可整理成“企业做 Agent 前先补 evals 能力”的方法文章，或包装成工作流诊断服务。
来源：https://x.com/levie/status/2069228335255949775

2）Builder 教程正在从“提示词”转向“完整技能封装”
内容：Zara Zhang 发布 Frontend Slides skill 完整 walkthrough，不只演示如何用 Claude Code 做漂亮 HTML slides，还拆解了 skill 的构建过程、如何复用成自己的 skill，以及插图、视频、发布等具体细节。
沉淀价值：高价值点不在做一套 slides，而在“把个人最佳实践封装成可复用技能资产”。
后续用途：GenGrowth 可跟进“技能产品化”“个人工作流封装”“AI 辅助内容生产 SOP”三个方向。
来源：https://x.com/zarazhangrui/status/2069311440692072481

3）大模型使用门槛正在转移到工作流重构
内容：AI & I 访谈里，Mike Krieger 讨论 Fable 5 时强调，真正的变化不是首日惊艳，而是连续几周使用后，用户会发现自己需要重新学习任务拆解、交互节奏和更长时间跨度的协作方式；尤其是做 agent orchestration 的人，体感最明显。
沉淀价值：这说明新模型的竞争点，正在从“回答更强”变成“是否迫使团队升级工作流”。
后续用途：适合做“为什么很多团队上了强模型却没提效”的复盘选题。
来源：https://www.youtube.com/playlist?list=PLuMcoKK9mKgHtW_o9h5sGO2vXrffKHwJL

3. GenGrowth 选题

1）标题方向：企业做 Agent，先别急着上模型，先把 evals 补上
核心观点：Agent 价值不在回答能力，而在真实工作流里的可验证表现。
适合平台：公众号、即刻、知识星球

2）标题方向：为什么真正的 AI Builder 开始把经验封装成 skill，而不是继续堆提示词
核心观点：可复用技能资产，才是个人生产力的放大器。
适合平台：公众号、X 线程、视频号

3）标题方向：强模型上线后，团队为什么反而更不会用了
核心观点：问题不在模型，而在旧工作流没有升级到长时程协作和任务拆解。
适合平台：公众号、播客提纲、内训分享
