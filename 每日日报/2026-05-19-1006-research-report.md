今天读什么｜别把记忆都塞进上下文，要给 Agent 设计“分层记忆系统”

**1. 适合谁读 / 预计阅读时间**  
适合正在做 AI Builder、智能体工作流、内容生产、知识库、增长自动化、内部 Bot 协作的人读。  
预计阅读时间：8–10 分钟。

**2. 为什么值得读**  
很多团队现在已经不缺“会回答”的模型，真正卡住落地的，反而是另一个更朴素的问题：**Agent 做过一次的事，第二次还能不能更快、更稳、更少走弯路？**

过去大家习惯把“记忆”理解成更长的上下文窗口，好像把更多聊天记录塞进去，Agent 就会更聪明。但最近公开产品和研究都在往同一个方向收敛：**记忆不是把历史全带着走，而是把重要信息提炼出来，在需要时再取回来。**

Cloudflare 在 Agents Week 推出 Agent Memory，强调的是“记住重要的、忘掉不重要的”；Browser Use 则把网页操作经验变成可复用的 skill，让后来的 agent 不用每次重新试错；一些架构文章也开始明确区分 session memory 和 persistent memory。  
这说明一个现实：**真正能上线的 Agent，不是上下文越来越长，而是记忆越来越有结构。**

对 GenGrowth 来说，这很重要。因为我们的很多流程都不是一次性问答，而是反复发生：研究选题、跟踪 builder、生成日报、沉淀洞察、复用模板、避免重复踩坑。没有分层记忆，Agent 每天都像“新来的实习生”；有了分层记忆，它才可能慢慢变成“熟悉业务的老同事”。

**3. 核心概念**  
第一，**工作记忆**。  
这是当前一次任务里的临时状态：目标、步骤、中间结果、待办、当前判断。它解决的是“这次任务怎么继续跑下去”，不是“以后永远记住什么”。

第二，**长期事实记忆**。  
这是跨任务也有价值的稳定信息，比如用户偏好、渠道规则、项目定义、常用术语。它解决的是“下次别再让我重复讲一遍”。

第三，**技能记忆**。  
这类记忆不是“知道什么”，而是“怎么做”。Browser Use 举的例子就很典型：第一个 agent 处理某个 Duo 2FA 页面时，多花了很多步才摸清按钮规律；一旦把这个经验写成 skill，后面的 agent 就可以直接复用，不必每次重学。

第四，**不是所有记忆都该自动晋升**。  
InfoQ 转述 Cloudflare 的一个提醒很实用：关键事实最好显式写入，不要完全依赖自动抽取。因为自动总结容易把噪音、误判、甚至临时情绪也记进去。  
换句话说，**聊天记录不等于事实库，事实库也不等于技能库。**

第五，**选择性记忆比全量携带更重要**。  
一些 2026 年的公开 benchmark 已经不只看“记住多少”，而是看多跳推理、时间变化、检索成本和 token 消耗。方向很清楚：不是越长越好，而是**越准、越省、越能支持任务继续执行越好**。

**4. 可复用方法**  
如果你现在要给 Agent 做一层实用记忆，可以直接用这个四步法：

1）先把信息分四类：**事实、事件、指令、任务**。  
事实进长期记忆；事件进日志；指令进规则；任务进任务板。别混在一起。

2）把本次 run 的中间状态外置。  
计划、草稿、结果、失败原因写到文件、任务卡或数据库里，不要只留在对话窗口里。

3）只把“稳定且未来还会用”的内容升级为长期记忆。  
例如用户偏好、固定流程、常见边界条件；不要把一次性产物、临时结论、当天待办全塞进去。

4）把重复踩坑总结成技能，而不是记成碎片事实。  
比如“某类网页登录怎么过”“某频道消息格式怎么写”“某流程失败后怎么回退”，这些更适合变成 playbook 或 skill。

再补一个容易忽略的点：**记忆需要遗忘和复核机制**。  
如果旧事实永不清理，Agent 很快会被过期信息污染。对高风险记忆，最好有人审一下再晋升。

**5. GenGrowth 可以怎么用**  
第一，用在研究与内容流程。  
把“长期关注的 builder、重复出现的主题、已经写过的角度、频道格式偏好”做成长期事实记忆；把每天的研究草稿、候选链接、摘要过程留在工作记忆里。这样既能减少重复，又不会把临时噪音沉淀成系统事实。

第二，用在多 Bot 协作。  
Hermes、PM、Ops 这类角色最怕上下文串味。更稳的做法不是共享整段聊天，而是共享最小必要事实、任务状态和可复用技能。

第三，用在增长自动化。  
比如内容分发、线索整理、客户画像更新，适合把“稳定画像”和“本次动作”拆开。否则 Agent 很容易把一次活动数据误当成长期判断。

**6. 今日行动**  
如果今天就想开始，不用先搭大系统，先做三件事：

- 先列出你现在流程里哪些信息属于：事实、事件、指令、任务。  
- 再选一个高频流程，给它补一层“任务中间状态外置”。  
- 最后把最近一个重复踩过的坑，写成一张可复用 skill，而不是继续靠聊天记忆。

一句话总结：**别再追求“Agent 什么都记得”，而要追求“它记对的、记该记的、并且能在下一次工作里真正用上”。**
