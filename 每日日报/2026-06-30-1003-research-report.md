今天读什么｜别让研究型 Agent 先写结论，先搭一层“证据工作流”

1. 适合谁读 / 预计阅读时间  
适合正在做 AI Builder、研究助手、内容生产、知识库问答、行业情报整理的人读。尤其适合已经发现：AI 很会“写得像对”，但团队很难快速判断它到底有没有根据。预计阅读时间 8—10 分钟。

2. 为什么值得读  
过去很多团队做研究型 Agent，默认流程是“先搜一点，再写一版”。问题在于，模型一旦先进入表达状态，就容易把“找到过一些资料”误当成“已经足够支撑结论”。这样产出的文章、报告、客户建议，表面流畅，实则很难复核。

最近几条公开信号很值得放在一起看。OpenAI 在 deep research 更新里，已经把“限制到可信站点”“实时查看步骤和来源”做成正式能力；Anthropic 在研究型 Agent 评测文章里，把 groundedness（论断是否被来源支撑）、coverage（关键事实是否覆盖到）、source quality（引用来源是否权威）并列成核心检查项；Vercel 则进一步强调，要把 Agent 从“依赖预训练记忆回答”推向“依赖检索到的证据推理”。

一句话：下一阶段更稳的研究型 Agent，不是谁写得更像分析师，而是谁先把证据链搭出来，再在证据之上写结论。

3. 核心概念  
第一，证据层要独立于表达层。先决定“这次结论允许引用哪些来源、缺哪些证据、还要不要继续搜”，再决定怎么写成自然语言。

第二，来源数量不等于来源质量。十篇转载和一篇官方文档，不是同一种证据。财务、产品、政策、学术问题，最好优先回到原始发布方、官方文档或一手材料。

第三，可引用比可总结更重要。真正有用的不是 Agent 说“我查过了”，而是它能指出：这个判断来自哪篇文档、哪类站点、还存在哪个不确定点。

第四，检索驱动推理，比记忆驱动推理更适合高风险任务。Vercel 把这种方向叫 retrieval-led reasoning，本质就是少靠模型脑补，多靠外部证据推进下一步。

第五，研究质量要单独评。Anthropic 的做法很实用：不要只看文章顺不顺，还要分别检查“有没有依据”“有没有漏关键点”“引用的是不是可靠来源”。

4. 可复用方法  
如果你想给现有研究或写作流程补一层最小“证据工作流”，可以直接用这五步：  
1）先定义可信来源白名单。比如官网、开发者文档、论文原文、监管机构、公司财报。  
2）要求 Agent 先交“证据清单”，再交正文。清单至少包含：来源、核心发现、可支持的判断、仍未解决的问题。  
3）把“继续搜索”变成显式判断。如果证据冲突、来源太弱、关键事实缺失，就先补搜，不急着成稿。  
4）把最终输出分成两层：上层是可读结论，下层是可追溯引用与不确定点。  
5）上线前单独做三项验收：是否有依据、是否覆盖关键问题、是否优先使用高质量来源。

5. GenGrowth 可以怎么用  
第一，用在 AI 学习/研究报告。不要让系统直接把链接改写成文章，而是先抽出“今天最值得信的 3—5 个证据点”，再组织成 Slack 可读文。

第二，用在内容生产。写行业解读、工具测评、Agent 方法论时，把“观点”和“证据出处”一起沉淀，后面改短视频脚本、长文或销售材料时更稳。

第三，用在增长自动化与客户方案。以后做客户诊断、竞品扫描、行业 briefing，可以先给出“证据包 + 初步判断”，把需要人工拍板的地方暴露出来，而不是直接端出一个像结论的结论。

6. 今日行动  
- 挑一条现有研究流程，补一个“证据清单先于正文”的中间产物。  
- 给常见任务列一个最小可信来源白名单，不再默认全网结果一视同仁。  
- 下次评审 AI 稿件时，除看文风外，额外问三句：依据在哪、漏了什么、引用够不够硬。
