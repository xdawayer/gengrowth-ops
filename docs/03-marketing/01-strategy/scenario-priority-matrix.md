# 增长场景优先级矩阵（内部决策草稿）

> **本文档定位**：基于 `keyword-research-overview.md` v0.4 的 4 决策维度，全维度发散场景组合 → 三维加权评分 + GenGrowth 商业模式契合度过滤 → 选出 Top 1-3 作为业务开展重点。
>
> **状态**：v0.3 草稿。本版核心修订是引入"GenGrowth 服务模式契合度过滤"——纠正 v0.2 把休闲游戏放在 Top 1 的错误。仍需团队基于内部销售线索与实际客户成本进一步修正。
>
> **使用方式**：先读评分方法论（含 2.4 商业模式契合度过滤）→ 看主矩阵（28 主组合）→ 看 Top 6 子矩阵（阶段 × 地区展开）→ 看 Top 3 一段话概括 → 团队讨论 → 定稿后驱动 SOP 优先级建设。

---

## 一、为什么需要这份矩阵

`keyword-research-overview.md` v0.4 给出了 4 维决策框架，但没有回答："**全 4 维交叉的所有可行组合中，GenGrowth 应该把资源投到哪 1-3 个赛道？**"。

本文档做三件事：
1. **发散**：4 维交叉，列出全部有效组合
2. **评分**：复合三维加权（商业可行性 + 能力契合度 + 客户画像清晰度）+ **GenGrowth 服务模式契合度过滤**，每维 0-3 分总分 0-9
3. **聚焦**：选出 Top 3，给出 100-200 字业务开展思路 + 后续 SOP 优先级建议

---

## 二、评分方法论

### 2.1 三个评分维度（每维 0-3 分，总分 0-9）

#### 维度 A：商业可行性（市场 × 转化 × 壁垒）

| 子维度  | 0 分            | 1 分           | 2 分             | 3 分             |
| ---- | -------------- | ------------- | --------------- | --------------- |
| 市场规模 | 极小（< $10M/年）   | 小（$10M–100M）  | 中（$100M–1B）     | 大（> $1B/年）      |
| 转化效率 | 难转化（CAC > LTV） | 中等（CAC ≈ LTV） | 良好（LTV/CAC > 2） | 极佳（LTV/CAC > 5） |
| 竞争壁垒 | 完全红海           | 中等竞争          | 有差异化空间          | 低竞争 + 易立壁垒      |

**计算方式**：三子维度均值，四舍五入到整数。

#### 维度 B：能力契合度（团队 × 工具 × SOP 复用）

| 子维度 | 0 分 | 1 分 | 2 分 | 3 分 |
|---|---|---|---|---|
| 团队基因 | 完全无经验 | 有部分经验 | 团队熟悉 | 核心强项 |
| 工具链覆盖 | 完全空白 | 部分覆盖 | 大部分覆盖 | 完全覆盖 |
| 现有 SOP 复用度 | 需全新建 | 大部分新建 | 部分复用 | 完全复用 |

**GenGrowth 当前能力锚点**：

- **团队**：核心 2 人（王玲 / 王志彪）+ 当前产品负责人（中国区大型游戏研发经验）
- **业务经验**：以**美国 + 巴西为主**的**休闲游戏、工具研发与发行**
- **客户合作资源**：工具、内容、C 端小游戏、电商均有合作或自建能力；SaaS 接触面较窄；**不具备** YMYL、本地服务相关合作资源
- **工具链**：Ahrefs / DataForSEO / GSC / Google Trends / Sheets（英文 SEO 全套）；葡 / 西语支持已具备；**不做百度系**
- **现有 SOP**：`keyword-research-sop.md` v2.3（内容/工具站 + 新到中等 DR + 英文谷歌 + 内容驱动）
- **战略意图**：商业模式以订阅付费为主（少量广告），**付费驱动是下一阶段战略重点**

#### 维度 C：客户画像清晰度（识别 × 付费 × 焦虑）

| 子维度 | 0 分 | 1 分 | 2 分 | 3 分 |
|---|---|---|---|---|
| 识别度 | 客户极难找 | 需筛查 | 有清晰目标库 | 直接可触达 |
| 付费意愿 | 不愿付费 | 需教育 | 有付费习惯 | 愿意为增长付费 |
| 增长焦虑度 | 无焦虑 | 偶尔关注 | 持续焦虑 | 紧急刚需 |

### 2.2 不可行 / 暂缓 / 不规划组合

**N/A（业务模式不匹配，物理上不存在）**：
- B2C 工具 / B2B SaaS / 媒体 / 本地服务 / YMYL + 直接交易（不通过电商售卖）
- 媒体 + 关系驱动（媒体本身是关系运营）
- C 端小游戏 + 直接交易（游戏发行通过应用商店属付费驱动）
- 新站 + 头部期（自相矛盾）
- 多语言多国 + 新站起步（罕见）

**0 分（团队/资源不具备，明确暂缓 / 不规划）**：
- **YMYL 全部组合** → 团队**缺合作资源**，能力 0–1 分
- **本地服务全部组合** → 团队**缺合作资源**，能力 0–1 分
- **中文百度系全部组合** → 团队**明确不规划**，子矩阵不展开
- **头部期组合**（DR > 60）→ GenGrowth 难帮到头部站

### 2.3 评分注意事项

- **AI 评分仍需团队修正**：以下评分基于补充背景知识 + 公开常识，不掌握具体客户合同金额、销售线索、长期数据
- **相对评分原则**：评分是各组合之间的相对比较，不是绝对量化
- **总分相同时**：按"能力契合度 + 商业模式契合度优先"排序

### 2.4 GenGrowth 服务模式契合度过滤（v0.3 关键修订）

> 本轮加入的关键过滤维度。GenGrowth 当前服务模式以**订阅付费业务**的关键词研究 / 增长 SOP 咨询为主。客户的商业变现模式与 GenGrowth 服务模式的契合度，影响"客户画像清晰度（C）"评分中"付费意愿"子分。

| 客户业务的变现模式 | 与 GenGrowth 契合度 | 影响 |
|---|---|---|
| 订阅付费（B2C 工具 / B2B SaaS）| 高 | 付费意愿子分 +1（满分 3）|
| 直接交易持续付费（电商）| 中高 | 付费意愿子分 维持（2）|
| 服务付费（本地服务 / YMYL）| 中 | 维持（2）|
| 广告变现 / IAP（休闲游戏 / 媒体）| 中低 | 付费意愿子分 -1（休闲游戏发行公司多有自己的投放团队，对外部咨询服务付费意愿中等）|

**为什么这个过滤重要**：广告变现客户（休闲游戏发行公司、媒体内容站）的核心 KPI 是 ARPDAU / RPM / 留存，与 GenGrowth 的"60-90 天 SEO 增长"产出物匹配度低；他们更需要的是付费投放素材优化、留存机制设计等，不是 GenGrowth 当前主力服务。即便团队有相关业务的能力基因（如团队休闲游戏发行经验），客户对外部咨询服务的付费意愿仍然中等。

**v0.2 → v0.3 主要影响行**（重新评分）：
- C 端小游戏 + 内容驱动：**8 → 5**（A 从 2→1：休闲游戏不靠内容获客 + 转化效率低 + 红海；B 从 3→2：团队基因偏付费而非内容；C 从 3→2：开发者对内容驱动需求弱）
- C 端小游戏 + 付费驱动：**7 → 6**（C 从 3→2：发行公司多有自己的投放团队，外部咨询付费意愿中等）
- C 端小游戏 + 关系驱动：**5 → 4**（C 从 2→1）
- B2C 工具 + 付费驱动：**6 → 7**（C 从 2→3：订阅付费 = GenGrowth 服务模式契合度满分）

## 五、第三层：Top 3 最终组合（一段话概括，100-200 字/Top）

> 综合主矩阵 + 子矩阵的最高分组合，选出 Top 3。

### Top 1：B2C 工具 / 早期 SaaS + 内容驱动 + 新站到增长期 + 英文谷歌（美国）（8/9）

GenGrowth 现有 SOP v2.3 完全覆盖（六源四桶 + 60–90 天种子实验 + DR 闸门），团队基因匹配，工具链 Ahrefs / DataForSEO / GSC 完整。客户画像满分：indie hacker、独立开发者、早期订阅付费工具，对增长有强烈焦虑且愿意付费 $200–500/月获取增长服务。商业可行性中等（市场不算最大，但能立"60–90 天可见效果"的差异化壁垒），能力契合度满分，**订阅付费 = GenGrowth 服务模式契合度满分**。**业务开展建议**：保持当前默认场景，做 5–10 个种子客户的 60–90 天实验，跑通"6 周首流量、3 个月可观流量"的样本数据后扩展至巴西葡语市场。这是 GenGrowth 当前最稳的赛道，也是 SOP 立刻可用的赛道。

### Top 2：B2C 工具 / 早期 SaaS + 付费驱动 + 增长期 + 英文谷歌（美国）（7/9）

与 Top 1 同客户类型的扩量协同——已通过 Top 1 跑通内容驱动起步流量的客户，进入增长期（DR 20-40）后引入 Google Ads + Meta 付费驱动放大。商业可行性中等，能力契合度中等（团队付费基因待补，需建 paid-acquisition SOP），客户画像满分（订阅付费 = GenGrowth 服务模式契合度满分）。**业务开展建议**：作为 Top 1 的"扩量加速器"，优先推进 paid-acquisition SOP 建设；选 1–2 个 Top 1 跑通的客户，引入 $1k–3k/月付费预算小规模测试，验证"内容承接 + 付费扩量"组合的整体 ROI。这是 GenGrowth 把"付费驱动列为下一阶段战略重点"的最优落脚点（订阅付费业务模式契合 + 与 Top 1 同客户协同）。

### Top 3：电商 + 直接交易 + 增长期 + 英文谷歌（美国）/ 巴西葡语 + 拉美（7/9）

电商主战场是平台搜索（Amazon SEO + Shopify SEO + MercadoLibre 等本地电商），团队**有合作或自建能力**。商业可行性最高（电商市场大 + 转化效率好），能力契合度中等，客户画像清晰度中等（直接交易持续付费 = 与 GenGrowth 订阅服务模式有差异但仍是"持续付费"业务，契合度中高）。**业务开展建议**：作为 Top 1 / Top 2 的客户类型风险分散——B2C 工具 / SaaS 客户增长瓶颈期，引入电商客户类型补充收入结构。优先 1 个美国 DTC 客户 + 1 个拉美电商客户，验证电商分层 SOP 方法论。需新建 `keyword-research-sop-ecommerce.md`。

> **Top 4 候选（紧随 Top 3，主 6/9 子矩阵 7/9）**：C 端小游戏 / 休闲游戏 + 付费驱动 + 美国/巴西。团队有美国 + 巴西休闲游戏发行强基因，但**休闲游戏发行公司一般有自己的投放团队**，对外部咨询服务的付费意愿中等；同时休闲游戏广告变现模式与 GenGrowth 订阅付费服务定位契合度较低。建议作为"团队基因驱动的机会型项目"——如有具体发行公司销售线索可个案推进，不进通用 SOP 优先级。

### 6.2 客户类型选择建议

**短期（0–6 个月）**：
- **主轴**：聚焦 **Top 1（B2C 工具 / 早期 SaaS + 内容驱动 + 美国）**，现有 SOP v2.3 直接跑，做 3–5 个种子样本，建立案例库

**中期（6–12 个月）**：
- 启动 **Top 2（B2C 工具 + 付费驱动）**：优先建 paid-acquisition SOP，作为 Top 1 客户的扩量协同（同客户类型双轮）
- 启动 **Top 3（电商 + 直接交易）**：建 ecommerce SOP，与团队合作资源对接，做 1–2 个电商客户案例
- 启动 Top 1 / Top 3 的巴西葡语扩展（建 portuguese SOP）

**长期（12+ 个月）**：
- 基于付费投放与电商能力沉淀，决定是否进入 Top 4 候选（休闲游戏 + 付费驱动）作为"团队基因驱动的机会型项目"
- B2B SaaS 系列保持观察，等团队接触面扩大后再判断
- YMYL / 本地服务保持暂缓，除非有新合作资源
- 媒体内容业务由于广告变现与 GenGrowth 服务模式契合度低，保持观察

### 6.3 团队需要决策的关键问题

1. **Top 1 + Top 2 同客户双轮 vs Top 1 + Top 3 跨客户类型分散？** 推荐先 Top 1 立刻起跑，paid-acquisition SOP 建设过程中 Top 1 跑通客户进入增长期再启动 Top 2；同步推进 Top 3 ecommerce SOP 建设
2. **paid-acquisition SOP（P0）**自建 vs 与第三方付费投放公司合作？团队付费基因待补，自建周期 2–3 个月，合作可立即落地但能力沉淀慢
3. **Top 3 电商方向**先做美国 DTC 还是拉美 MercadoLibre？团队合作资源具体在哪个市场更强？
4. **休闲游戏赛道（Top 4 候选）**——团队基因强但客户对外部咨询付费意愿中等，是否值得做？只要有 1 个发行公司销售线索就可以个案推进
5. **B2B SaaS 是否同意降级？**如果有具体销售线索（朋友推荐、行业关系），可个案推进而不进通用 SOP

1. 团队读评分方法论（二节，特别是 2.4 商业模式契合度过滤），就"GenGrowth 当前能力锚点 + 服务模式定位"达成共识
2. 团队逐行 review 主矩阵评分（三节），修正与内部数据不符的行（特别是 A 商业可行性列，AI 对市场规模 / LTV 的判断不一定准）
3. 修正后重新算 Top 6，进入子矩阵（四节）的展开
4. 修正 Top 3（五节）的一段话概括
5. 基于修正后的 Top 3，调整 SOP 建设优先级（六节）
6. 定稿后，本文档作为 `keyword-research-overview.md` 的"当下落地决策"附件
