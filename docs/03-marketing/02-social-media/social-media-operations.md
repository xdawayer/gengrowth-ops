## 1. 文档定位

本文件是社交媒体执行层主文档，用于承接未来几个月持续迭代的账号运营规范。

只回答：
1. 账号怎么分层。
2. 账号怎么注册、授权、配置和交接。
3. 各账号池做什么内容、按什么节奏运行。
4. 如何审批、发布、互动与复盘。
5. 内容资产与发布规范的边界。

不回答：
1. 公司层品牌战略。
2. 产品需求与页面设计。
3. 客户合同条款全文。
4. 各平台已发布内容成品归档。

补充边界：
1. 社交媒体规则、节奏、SOP 放在本文件和 `docs/03-marketing/`。
2. 实际发出的帖子、线程、评论素材、公众号正文，放在根目录 `内容资产/`。

## 2. 运营原则

1. 双账号优先，再按阈值升级：未达阈值的新产品不单独开公开账号，优先借助创始人账号做实验叙事、官方账号做沉淀。
2. 账号是资产，不是临时消耗品：所有资料、授权、历史记录必须可交接。
3. 内容先验证，再扩量：优先跑低成本题材测试，不盲目追求平台铺量。
4. AI 辅助，人工负责：AI 可以参与起草与整理，但审批与风险判断必须由人负责。
5. 社媒动作必须能回流到官网、Waitlist、Blog 或产品反馈池。

---

## 3. 账号资产分层

### 3.1 当前采用的层级

1. `L2 固定品牌层`
   - `GenGrowth` 官方品牌号
   - 创始人 / 负责人 IP 号
2. `L3 已达阈值独立产品层`
   - 当前样本：`AstrologyWiki`

### 3.2 升级规则

仅当某产品同时满足以下条件，才考虑从创始人 / 官方双账号结构升级为独立产品账号：
1. 连续 `3-4` 周存在稳定高反馈主题。
2. 已出现独立受众与独立转化动作。
3. 已有明确 Owner 承担内容、互动、复盘和安全责任。

---

## 4. 账号注册与授权流程

### 4.1 新账号准入前提

1. 明确该账号属于哪一层级。
2. 明确 Owner、协同人、审核人。
3. 明确是否需要客户授权。
4. 明确回流目标：官网、Waitlist、Blog、独立产品页或反馈表单。

### 4.2 注册流程

1. 使用统一邮箱体系、受控凭据记录和恢复信息清单。
2. 开启 2FA。
3. 记录 `platform / handle / owner / recovery email / scope / created_at`。
4. 完成头像、简介、链接、地区、品牌说明配置。

### 4.3 授权规则

1. 优先平台原生角色授权或 OAuth。
2. 禁止长期共享密码。
3. 所有授权与撤权动作必须留痕。
4. 客户账号必须遵循客户持有资产、GenGrowth 获取可撤销授权的原则。

---

## 5. 资料与 Profile 规范

每个账号必须统一配置：
1. 头像：与品牌层级一致，不使用临时图。
2. 昵称：与层级角色一致，避免同平台多号自相冲突。
3. 简介：一句话说明身份、主题和链接目标。
4. 链接：优先官网或当前验证落地页。
5. 置顶内容：说明账号定位或最近重点实验。

---

## 6. 内容运营管理

### 6.1 角色分工

1. 官方品牌号：产品更新、官网 Blog、周报摘要、案例沉淀、品牌信任。
2. 创始人 IP：判断、观点、阶段复盘、Why，以及实验首发解释。
3. 独立产品号：只服务已验证产品的独立受众与转化闭环。

### 6.2 实验内容归属

1. C 端产品的真实获客和实验，发生在对应产品渠道，不等于必须新增一个 X 公开账号。
2. 需要在 X 上公开的实验过程、失败复盘、关键截图和数据反馈，默认由创始人账号首发。
3. 官方品牌号只转发或重写已经形成结论的实验，沉淀为 Blog、Case Study、产品更新或 FAQ。
4. 只有当某个产品已经形成稳定独立受众时，才让它从“实验对象”升级为“独立账号主体”。

### 6.3 执行节奏基线

1. 创始人 IP：`1` 条 X 观点贴 + `1` 条 X 实验/复盘贴 + `1` 条 LinkedIn 长帖/摘要。
2. 官方品牌号：`1` 篇官网 Blog + `1` 条产品/案例/周报更新。
3. 已达阈值独立产品号：`2-3` 条轻量测试内容或社区互动。

### 6.4 内容类型

1. 观点型：立场、判断、框架。
2. 过程型：Build in Public、实验日志、截图。
3. 结果型：命中主题、转化变化、阶段复盘。
4. 互动型：评论、回复、问题征集、社区参与。

---

## 7. 发布与审批

1. 低风险内容：可由执行人直接发布，事后抽检。
2. 中风险内容：需负责人审批。
3. 高风险内容：涉及价格、法务、公开承诺、争议议题时必须升级审批。
4. 默认 `L1` 发布模式：AI 起草 + 人工审批。

---

## 8. 暖号、互动与风控

1. 新号必须先经历暖号期，再进入强导流。
2. 暖号期以关注、评论、转发、轻内容为主。
3. 单日动作不追求极限，避免平台判定异常。
4. 如遇限流、封号、投诉，立即停用自动化并切人工兜底。

---

## 9. 数据与复盘

每周固定复盘：
1. `Hypothesis`
2. `Action`
3. `Attribution`
4. `Decision`

最低记录字段：
1. 平台
2. 账号池
3. 内容类型
4. 主题
5. 曝光/互动
6. 点击/回流
7. 是否命中升级条件

---

## 10. 交接与撤权

1. 合同期结束或角色变更时，必须导出账号台账。
2. 交接内容包括：账号列表、授权范围、内容模板、置顶内容、最近 30 天复盘。
3. 客户账号的最终控制权始终归客户。
