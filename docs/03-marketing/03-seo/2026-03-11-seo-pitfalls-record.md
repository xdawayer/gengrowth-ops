


# 🚀 GenGrowth 项目实战故障排坑记录 (纯净版)

### Issue 1: 新文章发布超一个月未被 Google 收录

* **现象**：文章已发布且有内链，但在 GSC 检查网址时，状态显示为“不适用 (N/A)”，既没有“已发现”，也没有“已抓取”。
* **根本原因 (Root Cause)**：
1. **Sitemap 更新滞后**：静态 `sitemap.xml` 没有包含这些新文章的 URL。
2. **国际化 (i18n) 路由冲突**：Sitemap 中输出了带 `/en/` 的绝对路径，但实际页面因为隐式路由未显示 `/en/` 前缀，导致爬虫无法将地图路径与实际页面对应。


* **解决方案**：
1. 改写前端代码，部署动态生成的 Sitemap（如 Next.js 的 `sitemap.ts`）。
2. 统一前端路由标准：从 Sitemap 生成脚本中剥离 `/en/` 前缀，确保输出的 XML 链接与浏览器实际渲染的 `astrologywiki.com/wiki/...` 绝对一致。
3. 在 GSC 手动对相关 URL 提交“请求编入索引”。



### Issue 2: GSC 提示“网页会自动重定向”及“备用网页”

* **现象**：GSC 网页索引报告中，显示若干带有 `http://` 或不带 `www` 的首页 URL 未被编入索引。
* **根本原因**：服务器的强制 HTTPS 机制或 Google 的自动去重（Canonicalization）机制在正常工作。爬虫试图访问不安全/不规范的 URL 时被系统重定向，因此 GSC 记录了这些被抛弃的中间路径。
* **解决方案**：无需处理，属于健康的系统分流现象。

### Issue 3: 站点地图在 GSC 显示“无法抓取” (Couldn't Fetch)

* **现象**：提交正确的 `https://www.astrologywiki.com/sitemap.xml` 后，状态标红显示“无法抓取”。此前曾误提交过一个错误地址。
* **根本原因**：
1. 误提交的错误地址产生了视觉干扰（已通过“移除站点地图”清理）。
2. 新提交的正确地址显示“无法抓取”，通常是 GSC 的 UI 延迟（实际底层状态为 Pending 等待抓取），而非真正的技术拦截。


* **解决方案**：通过 GSC 顶部搜索框“测试实际网址”确认无拦截后，放置等待 24-48 小时，状态自动更新为“成功 (Success)”。
