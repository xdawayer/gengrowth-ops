# 关键词研究 SOP

> 适用范围：GenGrowth 种子客户增长实验 + 未来产品自动化的方法论基础。  
> 配套使用：`seed-client-growth-experiment-template.md`

---

## 导读：适用范围声明 + 场景路由

### 本 SOP 适合的场景

- **业务类型**：内容站 / 工具站 / 早期 SaaS（信息产品 / 自助型工具）
- **站点阶段**：新站到中等权重（DR 0–40），尤其是 60–90 天看初步效果的种子客户增长场景
- **目标地区**：英文谷歌生态（Ahrefs / DataForSEO / GSC 工具链）
- **获客方式**：以 SEO 为主，配合社媒"双轨"协同（Track A 内容 + Track B 社交信号）

### 本 SOP **不**适合的场景

以下场景不在本 SOP 覆盖范围内，请先到 `../keyword-research-overview.md`（路由总览）按决策维度判断：

- 老站 / DR > 50 / 需要刷新与修剪
- 电商 / 产品页与类目页分层
- 本地 SEO / 多门店服务业
- 中文站 / 百度 / 知乎 / 小红书
- 付费投放 / 社媒营销 / 合作分销

> GEO / AEO（AI 搜索引用优化）已在本文第八章嵌入初版方法论，无需切换。

## 零、竞品选择（关键词挖掘前置步骤）

> **核心原则**：对标"你能打过的人"，不是"最大的竞品"。竞品选错 = 挖到的词大量被 DR 过滤 = 无效工作。

> **本节与关键词过滤的关系（避免混淆两个不同的"30"）**
> - **本节（零）**：解决"去哪里找词"——竞品选择策略，按你的站绝对 DR 给出不同建议
> - **二节·前置过滤**：解决"找到的词哪些可执行"——DR **差距**标准，差距 > 30 → 跳过
>
> 这是两个独立判断。**关键词过滤逻辑新站老站完全相同**，都用"差距 ≤ 30"这一标准；区别只是新站通过过滤的词数量少，老站多。不存在"新站用一套过滤逻辑，老站用另一套"。

### 话题发现 vs 关键词对标：两件不同的事

使用竞品前先明确用途，高 DR 竞品和同体量竞品适合的场景完全不同：

| 用途 | 高 DR 竞品（DR 差距 > 30）| 同体量竞品（DR 差距 ≤ 30）|
|------|--------------------------|--------------------------|
| 话题发现（了解品类内容全景）| ✅ 必须用，大站覆盖最全 | 可补充 |
| 内容结构研究（格式、深度、角度）| ✅ 可用 | 可补充 |
| 关键词直接对标（生成执行词库）| ❌ 几乎全被 DR 过滤 | ✅ 主要来源 |
| Content Gap 分析 | ⚠️ 可做，但 KD 门槛需降至 ≤ 20 | ✅ 正常使用 |

**高 DR 竞品的正确用法**：提取 Top Pages 的内容类别 → 不直接用关键词 → 用类别名在 Keywords Explorer 里找低 KD 变体。

---

### 新站（你的站 DR < 20）

**优先策略：先找"弱 SERP"里排名的同类站，而不是直接选品类大站。**

1. 在 Google 搜索 5 个核心种子词，看 Top 10 里是否有 DR < 30 的站在排名
2. 有 → 这些站是真正可对标的竞品，进 Ahrefs 做关键词分析
3. 没有 → 进入「整个品类 DR 都高」的备用路径（见下方）

**新站竞品筛选标准：**

| 维度 | 标准 |
|------|------|
| DR | 比你的站高 0–20 分（DR 差距 ≤ 20） |
| 月均访问量 | 5k–50k（太小无参考价值，太大词打不过） |
| 建站时间 | 1–3 年内（靠内容增长的站，策略有效性更高） |
| 内容类型 | 与你相同或相近（内容站、工具站等） |

---

### 老站（你的站 DR > 30）

分三层使用，不同层次用途不同：

| 层次 | DR 范围 | 用途 |
|------|---------|------|
| 战术竞品 | 你的 DR ±15 | 快速胜利词主要来源，可直接抢排名 |
| 基准竞品 | 你的 DR +15 到 +30 | 3–6 个月内有机会追上的词 |
| 标杆竞品 | 你的 DR +30 以上 | 研究内容结构与选题策略，不用于 KD 对标 |

---

### 整个品类 DR 都高时（占星、健康、金融等）

头部站 DR 普遍 60–80+，找不到同体量竞品，竞品 Content Gap 框架基本失效。**放弃 Content Gap，改为工具直查：**

1. Keywords Explorer 输入 10–15 个种子词（比平时多）
2. 过滤：KD ≤ 15，月搜索量 ≥ 50
3. 优先看 Questions 和 Related terms
4. 对每个候选词，查 SERP Top 10 的 DR 分布
   - Top 10 有 2 个以上 DR < 你的站 → 可执行
   - Top 10 平均 DR > 你的站 + 30 → 暂跳过，找更弱的 SERP

**高 DR 品类的内容切入规律（以占星为例）：**

大站主力覆盖泛流量词（运势、星盘），以下类型普遍缺失且 KD < 15：

- 行星 × 宫位 / 相位组合：`mars in 8th house`、`venus conjunct saturn meaning`
- 小行星 / 小天体：Chiron、Lilith、Black Moon 相关词
- 专业术语定义词：`what is a stellium`、`out of bounds planet meaning`（featured snippet 机会，新站可抢）
- 方法论比较词：`tropical vs sidereal astrology`、`placidus vs whole sign houses`

## 一、六大关键词来源

### 来源 1：竞品关键词映射

**目标：** 找到竞品已经在排名、但你还没覆盖的词；同时通过 Top Pages 扫描了解品类内容全景。  
**工具：** Ahrefs → Site Explorer → 竞品域名

**步骤 A — Top Pages 话题扫描（所有竞品必做，约 5 分钟/个）**

> 适用于所有竞品，包括高 DR 竞品（DR 差距 > 30 时只执行此步骤，不执行步骤 B）。

1. Ahrefs → Site Explorer → 竞品域名 → Top Pages（按估算流量排序）
2. 导出 Top 50 页面
3. 标记：哪些话题 / 关键词类型带来了最多流量？（例：工具对比类、概念定义类、How-to 类）
4. 判断：这些话题我们能写出更好或更完整的内容吗？
5. 产出：内容话题清单（不是关键词列表），作为步骤 B 和 Keywords Explorer 的种子输入

**步骤 B — 关键词直接导出（仅限战术 / 基准竞品，DR 差距 ≤ 30）**

> 高 DR 竞品（DR 差距 > 30）只做步骤 A，不做步骤 B——导出的词几乎全被 DR 过滤，无执行价值。

1. Ahrefs → Site Explorer → 竞品域名 → Organic Keywords
2. 过滤条件：KD < 40，排名 1-20，月搜索量 > 100
3. 去重合并，标注来源竞品

---

### 来源 2：内容缺口分析

**目标：** 找到竞品在排名但你完全没有内容覆盖的词。  
**工具：** Ahrefs → Site Explorer → 你的域名 → Content Gap（输入竞品域名对比）  
**执行：**
1. 输入 3 个竞品，勾选"至少两个竞品在排名但你没有"
2. 过滤 KD < 40
3. 优先选择商业意图明确的词（含 "best"、"vs"、"alternative"、"how to"）

---

### 来源 3：种子词多维度拓展

**目标：** 从多个视角覆盖整个行业需求，防止遗漏整类用户。  
**工具：** Ahrefs Keywords Explorer + Google Suggest  
**执行：**
1. 用脑暴思维导图列出 3-5 个维度（例：用户角色 / 问题类型 / 工具类型 / 使用场景 / 竞品名称）
2. 每个维度列出 3-5 个种子词
3. 逐一输入 Keywords Explorer，导出"Matching terms"和"Related terms"
4. 用 Google Suggest 补充（搜索框下拉 + "People Also Ask"）

**示例维度（以 indie dev 工具站为例）：**

| 维度   | 种子词示例                                       |
| ---- | ------------------------------------------- |
| 用户角色 | indie developer, solo founder, indie hacker |
| 问题类型 | get more users, increase traffic, grow app  |
| 工具类型 | seo tool, marketing tool, growth tool       |
| 使用场景 | launch product, grow saas, app marketing    |
| 竞品名称 | ahrefs alternative, jasper alternative      |

**种子词设计原则（来自光年 SOP 方法论，适配英文 SEO）：**
- 维度要多且广，不同维度之间是并列关系，不能存在包含关系
- 恰好适配行业：不能过大（"marketing"），也不能过小（"marketing automation tool for React devs on macOS"）
- 基于真实搜索行为，不要拼凑预设场景
---

### 来源 4：社区 / 问答挖掘

**目标：** 获取目标用户的真实用语，这些词搜索工具往往低估其价值。  
**工具：** Reddit / Indie Hackers / Hacker News / Product Hunt 手工搜索  
**执行：**
1. 在 Reddit 搜索 `subreddit:indiehackers "how do I"` / `subreddit:SideProject "struggling with"`
2. 记录用户描述问题时使用的原话，尤其是具体的动词短语
3. 在 Ahrefs 验证搜索量（即使很小也值得考虑，长尾词集群价值）
4. 产出：20-50 个真实用语词

---

### 来源 5：趋势词挖掘

**目标：** 找到搜索量正在上升但竞争尚未涌入的时机词，抢先建立排名优势。  
**工具：** Google Trends + Exploding Topics  
**执行：**
1. Google Trends：输入候选词，查看近 12 个月趋势曲线
   - 上升型（近 3 月持续升）→ 标记为趋势词
   - 平稳型 → 按常规优先级处理
   - 下降型 → 降优先级或跳过
2. Exploding Topics：浏览 "Growth" 分类，找与产品相关的上升话题
3. **判断新词标准（哥飞方法论）：**
   - 搜索量近 3 月环比增长 > 30%
   - Ahrefs KD < 30（竞争还没涌入）
   - 该话题有明确的内容/工具创作空间

**时机窗口：** 趋势词通常有 2-4 个月的低竞争窗口，发现即应插队执行，不要等到下个月的计划周期。

---

### 来源 6：Social Probe 实时信号

> **与来源 4（社区挖掘）的本质区别：**
> - **来源 4**：读取他人已有讨论（Reddit 帖子、评论区、IH 提问）→ 无需账号活跃度，随时可做
> - **来源 6**：追踪自己发布内容后的互动反馈（高共鸣评论、意外爆款帖）→ 依赖 Track B 活跃运营和账号基础
>
> **早期阶段（Track B 未启动 / 账号无受众）直接跳过本来源，记录 "N/A（Track B 未启动）"。** 来源 4 已能满足社区信号需求。

**前置条件：** Track B 已启动，账号有持续内容发布和互动数据。

**目标：** 把 Track B 的高共鸣话题实时转化为 SEO 内容机会，捕捉搜索工具覆盖不到的用户真实需求。  
**工具：** 实验模板中的 `→ SEO 转化？` 字段  
**执行：**
- 每次 Social Probe 发布后，互动数异常高或出现高质量评论原话时，当天打标记
- SEO 同事每周一检查标记，通过 Ahrefs / Keywords Explorer 验证是否有对应搜索量
- **注意：** Social Probe 信号是「话题 / 角度」，不是「关键词」——需先验证搜索量、转化为具体关键词再分桶，不能直接写入关键词主表
- 验证有搜索量后插入内容日历，来源字段标注 "Social 信号"

## 二、优先级：分类优先系统

> 不用加权公式。先分桶，再在桶内排序，最后用固定规则决定跨桶执行顺序。  
> 每个桶用自己的逻辑，不做跨类型比较，消除异常值干扰。

---

### 前置关卡：过滤 + 标注

> 两种不同性质的关卡，用途不同：
> - **过滤关（第一关）**：不通过的词直接排除，不进入分类流程
> - **标注关（第二、三关）**：所有词都进系统，打标注值；标注结果供桶内排序使用，不决定词能不能进桶

**第一关（过滤）：DR 匹配**

查看目标词 Top 10 结果的平均 DR，与你的站 DR 对比。  
**差距 > 30 分 → 直接跳过，不进入分类流程。**

这一步过滤掉 KD 失真问题（KD 基于全网平均，不反映你的站实际进入可能性）。

> **新站阶段注意（你的站 DR < 20）：** 初期词库通常大量被第一关过滤（Top10 平均 DR 普遍 > 你的站 + 30）。此时：
> - 优先以 **H 列（SERP弱度）= ✅弱** 作为执行信号，而非等待 DR 差值通过
> - KD < 15 且 H 列 = ✅弱 的词，可在 **P 列（手动分桶）** 覆盖分桶结果，强制进入快速胜利桶
> - 同步扩展竞品选择范围（参见零节），优先找 DR 0–20 的同类站对标，增加过滤通过率

**第二关（标注）：SERP 弱度（快速胜利桶所有词必做，30 秒/词）**

DR 通过后，Google 搜索候选词，看 Top 10 里是否有以下弱信号：
- DR < 30 的站在排名 → 弱信号
- 论坛帖（Reddit / Quora）出现在 Top 5 → 弱信号
- 页面内容明显陈旧（3 年以上未更新）→ 弱信号

| 弱信号数量 | 判断 | 关键词主表 H 列 |
|-----------|------|--------------|
| ≥ 2 个 | SERP 弱，优先执行 | `✅弱` |
| 1 个 | 可尝试 | `⚠️中` |
| 0 个 | 强 SERP，暂缓 | `❌强` |

> H 列值是快速胜利桶内排序的第一维度——"标注"（填 H 列）和"排序"（按 H 列排序）是同一信号的数据采集和使用两个步骤，不重复，是顺序关系。

> **为什么其他桶不要求必做？**
> - **趋势词**：执行信号来自时机窗口（Trends 比值），发现即执行，不因 SERP 强度推迟
> - **战略词**：KD 20–50 的词 SERP 普遍较强，这是已知条件而非过滤依据；重要的是内容深度和持续积累
> - **长尾词**：月搜索量 < 100，KD 通常极低，SERP 天然偏弱，不需要额外验证

**第三关（标注）：AIO 风险**

以下两类词**必检**，其余可选：
- 月搜索量 ≥ 500 的词
- 词型含 `what is / meaning / definition / how does / explained` 的词

脚本可按上述词型自动在 M 列预标注 `⚠️疑似高风险`，人工用无痕窗口最终确认：出现 AI Overview 蓝色框 → `高`；未出现 → `低`。

| AIO 风险 | 内容策略建议 |
|---------|------------|
| 低 | 常规 SEO 内容 |
| 高 | 在定义解释层之上必须加一层：交互工具 / 个性化计算器 / 社区解读汇总 / 对比型切入；避免纯定义文 |

---

### 意图标注（必填元数据，分桶前完成）

> 意图不是过滤条件，是元数据——不决定词能不能进桶，但影响桶内排序和内容策略决策。

**M 列（意图_自动）脚本模式匹配规则（约 80% 准确）：**

| 意图类型 | 触发关键词型 |
|---------|------------|
| Commercial | best / top / vs / alternative / comparison / review / pricing |
| Transactional | buy / cost / cheap / discount / free trial / sign up |
| Problem-aware | fix / not working / problem / error / how to solve |
| Informational | what is / how to / guide / tutorial / why / explained |
| 待确认 | 以上均未命中 |

`待确认` 的词批量交 Claude/GPT 处理（使用四节第三层 prompt 模板）。结果粘回 Sheets，M 列自动重算桶内排序权重。

### 四个桶的分类规则

分类顺序：从上往下匹配，进入第一个符合条件的桶。

| 桶 | 分类条件 | 说明 |
| ---------- | ------------------------------------------------------- | --------------- |
| **趋势词** | Trends 3M/6M 比值 > 1.2 且 KD < 35 **且** G1 话题相关（✅）**且** G2 可承接（Y）| 发现即插队，不进固定排期 |
| **快速胜利** | KD < 20 且月搜索量 ≥ 100 | 主战场，60 天内可见效果 |
| **长尾词** | 月搜索量 < 100（不论来源） | 主战场，按主题系列矩阵批量执行 |
| **战略词** | 意图 = Commercial 或 Transactional 且 KD 20–50 **且月搜索量 ≥ 100**，OR 策略强制 = 是 | 低配执行，仅选 1–2 个门面词 |

> **趋势词相关性闸门：**
> - G1 话题相关（K 列）：脚本检查关键词是否命中预设 `TOPIC_KEYWORDS` 数组（每客户初始化时配置）；命中 → `✅相关`；未命中 → `⚠️待确认`（人工判断，否则降入快速胜利桶）
> - G2 可承接（L 列）：SEO 负责人手动填 Y/N；空值视为 N，默认不进趋势桶

> **战略词补充：** CPC 保留为展示字段（Ahrefs 导入），供参考，不作为分类条件。月搜索量 < 100 的商业意图词归入长尾词桶（矩阵执行），不进战略词桶。

> **社区来源词处理：** 来源 4 找到的词验证搜索量后按上表正常分桶；工具无法量化（N/A 或 < 10）的词不进分桶，单独记录为"内容角度参考"，用于补充集群写作角度，不做定向排名。

### 桶内排序

| 桶 | 排序维度 | 逻辑 |
| ------ | ----------------- | ------------- |
| 趋势词 | Trends 比值降序 | 趋势越陡，窗口越紧迫 |
| 快速胜利 | H 列（SERP弱度）降序 → 意图（Commercial/Problem-aware 先，Informational 后）→ 月搜索量降序 | 能赢 > 值得赢 > 量大 |
| 长尾词 | 主题相关度（1-3 手动打分）降序 | 与现有内容集群越近，优先放入同主题批次 |
| 战略词 | 主题集群相关度（1-3 手动打分）降序 → CPC 降序 | 先建集群核心，商业价值辅助排序 |

> 快速胜利桶可在 Sheets 新增辅助排序列：`=IF(H2="✅弱",3,IF(H2="⚠️中",2,1))+IF(OR(M2="Commercial",M2="Problem-aware"),1,0)`，按此列降序展示；月搜索量作第三排序键。

---

### 跨桶执行顺序（固定规则，不再计算）

1. **趋势词**：发现即插队，不等周计划，不进固定排期
2. **快速胜利**：主战场，承接约 60% 执行产能（Week 1 起）
3. **长尾词**：主战场，承接约 40% 执行产能；按主题系列批量生产，降低单篇边际成本
   - 月搜索量 50–100 且意图明确 → **Week 1 起与快速胜利桶并行执行**，不等待
   - 月搜索量 < 50 → 有余力时批量执行
4. **战略词（最低优先，全周期不超过 5 篇）**：仅选 1–2 个核心词做品牌门面；AI 起草 + 人工审稿，不占主要产能；Week 3 起，不影响快速胜利和长尾词产能

**人工覆盖：** 关键词主表「策略强制（是/否）」列，标注为「是」的词无视分桶结果，直接进快速胜利桶。用于定位词、品牌词等不应被系统跳过的词。

## 三、结果衡量标准

### 核心逻辑：预测值 vs 实际值

每个关键词在发布内容前，先用 KD 和搜索量计算预期流量，发布后对比 GSC 实际数字。偏差大的词反映问题（预测模型需调整或内容质量不足）。

### 第一步：根据 KD 预测排名位置

| KD 区间 | 新站预期排名（Week 8） | 乐观预测    |
| ----- | -------------- | ------- |
| 0–10  | P3–P10         | P1–P3   |
| 11–20 | P5–P20         | P3–P10  |
| 21–35 | P10–P30        | P5–P20  |
| 36–50 | P20–P50        | P10–P30 |

### 第二步：根据排名位置计算预期流量

Google 搜索各位置标准 CTR：

| 排名位置 | 标准 CTR |
|--------|---------|
| P1 | 28% |
| P3 | 11% |
| P5 | 6% |
| P10 | 2% |
| P20 | 0.5% |

**预期月流量 = 月搜索量 × 对应 CTR**

示例：月搜索量 500，预期排名 P5 → 预期流量 = 500 × 6% = **30 次/月**

### 第三步：建立每篇内容的预测卡

每篇内容发布时填写，Day 30 / Day 60 对比实际：

| 字段                | 内容        |
| ----------------- | --------- |
| 目标关键词             |           |
| 月搜索量              |           |
| KD                |           |
| 预期排名（Week 8）      | P___      |
| 预期月流量（Week 8）     | ___ 次     |
| **实际排名（Week 4）**  | P___      |
| **实际排名（Week 8）**  | P___      |
| **实际月流量（Week 8）** | ___ 次     |
| 预测 vs 实际偏差        | +X% / -X% |
| 偏差原因分析            |           |

### 复盘时的来源命中率统计

## 四、关键词处理流水线（自动化方案）

> 六个来源的工作量主要在**过滤**，不在收集。工具导出容易拿到 500-2000 个词，瓶颈是把它压到 30-50 个可执行词。三层流水线解决这个问题。

### 第一层：工具粗过滤（0 人工，导出时设置）

在 Ahrefs/SEMrush 导出时直接加过滤条件，原始词库缩减 80%：

| 来源 | 导出过滤条件 |
|------|-----------|
| 竞品映射 | KD ≤ 40，排名 1–30，月搜索量 ≥ 50 |
| 内容缺口 | KD ≤ 40，至少 2 个竞品在排名 |
| 种子词拓展 | KD ≤ 50，月搜索量 ≥ 50 |

过滤后进入下一层的词约 200–400 个。

### 第二层：Sheets 模板自动分桶（0 人工）

把 CSV 粘贴进预制 Sheets 模板，公式自动完成：
- DR 差距检查（超限自动标红跳过）
- 四桶分类（IF 嵌套，按 KD / 搜索量 / CPC / 意图词判断）
- 桶内自动排序（趋势词按 Trends 比值，快速胜利按搜索量，战略词按 CPC）

人工只需查看每桶前 20 名，选择执行。

**Sheets 模板生成方法：** 复制 `keyword-sheet-setup.gs` 中的 Apps Script，粘贴到任意 Google Sheet → 扩展程序 → Apps Script → 运行 `createGenGrowthKeywordSheet`，自动生成完整模板。

### 第三层：AI 批量意图判断（半自动，处理 CPC=0 的词）

社区词和 CPC=0 的词，公式无法自动判断意图。用以下 prompt 批量处理（100 词 / 次，约 30 秒）：

```
以下关键词面向 [产品类型] 的 [目标用户]。
逐行判断搜索意图（Transactional / Commercial / Problem-aware / Informational / Generic）
及是否适合创作 SEO 内容（是/否/有条件）。
只输出表格，不解释。

[粘贴关键词列表]
```

结果粘回 Sheets，分桶自动重算。

### 各来源预计处理时间

| 来源 | 处理方式 | 预计时间 |
|------|---------|---------|
| 竞品映射 | Ahrefs 过滤导出 → 粘贴 Sheets | 10 min |
| 内容缺口 | Ahrefs Content Gap 导出 → 粘贴 | 10 min |
| 种子词拓展 | Keywords Explorer 导出 → 粘贴 | 15 min |
| 社区挖掘 | 手工记录原话短语 | 20 min/周 |
| 趋势词 | Google Trends 查 5–10 个候选词 | 10 min/周 |
| Social Probe | 日常执行打标记 | 0 额外时间 |

**首次建库约 1.5 小时，之后每周增量维护约 30 分钟。**

## 五、执行节奏

### 周度节奏（每周一执行）

| 任务 | 负责人 | 时间 |
|------|-------|------|
| 检查 Social Probe `→ SEO 转化？` 标记，评估是否插队 | SEO 同事 | 15 min |
| 扫描 Google Trends / Exploding Topics 新趋势 | SEO 同事 | 15 min |
| 更新内容日历，确认本周执行词 | SEO 同事 | 15 min |
| 发布当周内容，检查 On-page SEO 清单 | SEO 同事 | 主要工作 |

### Day 30 中期复盘

**目标：** 调整后 30 天的执行计划。

复盘问题：
1. GSC 里有没有出现计划外的排名词？（发现意外机会）
2. 已发布内容中，哪几篇出现了早期排名信号（P2-P3）？
3. 6 个来源中，哪个来源的词命中率最高（上了排名或带来了流量）？
4. Social Probe 有没有发现尚未纳入计划的高价值话题？
5. 后 30 天：维持当前策略 / 调整 Tier 分配 / 增加某类来源？

**输出：** 更新后的内容日历（第 5-8 周）

### Day 60 全复盘

**目标：** 评估整体增长效果，产出可复用规律，做最终专业判断。

复盘问题：
1. **流量来源分析：** 哪些关键词实际带来了流量？哪些发布了但没有排名？
2. **来源质量分析：** 6 个来源各自的命中率是多少？
3. **意图匹配分析：** 哪种意图类型的词转化效果最好（带来实际注册/访问）？
4. **时机分析：** 趋势词的表现是否优于同难度的非趋势词？
5. **效率分析：** 每篇内容平均带来多少流量？哪种内容格式效率最高？
6. **产品判断：** 基于所有数据，这个产品的增长潜力是什么？建议继续 / 调整 / 放弃？

**输出：** 实验记录模板的 Day 60 结果报告 + 可复用规律清单

## 八、GEO/AEO 引用优化（AI 搜索引擎可见性）

> **GEO**（Generative Engine Optimization）/ **AEO**（Answer Engine Optimization）：让你的内容被 ChatGPT、Perplexity、Gemini、豆包、Kimi 等 AI 搜索引擎/聊天机器人引用，作为答案来源出现。
>
> **为什么纳入**：2026 年起，AI 搜索引用是与传统 SEO 并列的新流量入口。无论新站老站，都应在关键词与内容策略中保留这个维度。当前作为"默认意识"嵌入主流程，未来系统化后独立成 `keyword-research-sop-geo.md` 专项 SOP。

### 1. GEO 与传统 SEO 的关系

- **协同**：传统 SEO Top 5 排名的页面，是 AI 模型抓取与引用的主要来源池
- **分流**：Reddit / Quora / 社区帖子常被 AI 引用，即使传统 SEO 排名不高 —— 这是 GEO 独立的一条路径
- **互斥**：AI 直接给出答案后，用户不点击源链接 → 高 AIO 风险词在传统 SEO 上失血，但 GEO 上仍能获得品牌曝光与心智渗透

### 2. 哪类关键词更容易被 AI 引用

| 词型 | GEO 引用倾向 | 内容形态建议 |
|---|---|---|
| `what is X` / `X meaning` / `X explained` | 高 | 顶部一句话明确定义，下方展开论据 |
| `best X for Y use case` | 高 | 结构化对比表 + 明确推荐理由 |
| `X vs Y` | 高 | 对比表 + 各自适用场景 |
| `how to do X` | 中 | 编号步骤 + 每步关键提示 |
| 趋势词 / 新概念词 | 高 | AI 用户习惯用对话式查询新概念 |
| 纯品牌词 / 导航词 | 低 | 无需特别优化 |

### 3. 内容形态调整（GEO Friendly）

- **结构化答案**：顶部直接回答，再展开论据（不是"先讲故事再给结论"）
- **数字、表格、对比列表**：AI 容易抽取
- **明确的观点 + 第一手证据**：数据、案例、原话引用
- **命名实体清晰**：产品名、版本、人名、机构名 —— 这是 AI 引用时的"句柄"
- **避免内容过于泛化**：AI 倾向引用具体、可验证、有立场的内容

### 4. 监测方式

| 工具 / 方式 | 用途 | 费用 |
|---|---|---|
| 手工查询：定期用 Perplexity / ChatGPT / Gemini 搜目标词 | 看是否被引用、引用了哪些段落 | 免费 |
| Profound / OtterlyAI / AthenaHQ | 系统化追踪 AI 引用，监控品牌可见度 | 付费 |
| GA4 → Acquisition → Source/Medium | 看是否有 chat.openai.com / perplexity.ai 等 referrer | 免费 |

### 5. 与本 SOP 主流程的整合

- **关键词主表**：可在 X 列后追加 Y 列「GEO 潜力」（高/中/低），用于优先标注
- **桶内排序**：快速胜利桶中，GEO 潜力高的词优先（同条件下）
- **内容卡**：发布前回答三个问题 —— 是否有结构化答案？是否有具体数字 / 案例？命名实体是否清晰？
