# Astrologywiki 介绍资料

> 本文档用于新员工入职时理解 `Astrologywiki.com` 这个项目。
> 目标不是讲清所有产品细节，而是让新成员快速理解：它是什么、面向谁、当前在承担什么业务目标、和 GenGrowth 是什么关系、目前做到哪一步。
> 配套阅读：[业务介绍资料](./业务介绍资料.md)。

---

## 一、它是什么类型的产品

`Astrologywiki.com` 是一款面向欧美市场的现代占星 Web 产品。

它的定位不是“传统命理预测工具”，而是把西方占星学和现代心理学结合起来，帮助用户做自我探索和心理成长。可以理解为一款 **占星 + 心理学 + AI 个性化分析** 的内容与工具一体化产品。

需要区分三个名字：

- `Astrologywiki.com`：对外的产品域名和品牌入口
- `AstroMind`：产品在内部和应用内的品牌名
- `Oracle`：仓库工程代号，对应 `/users/wzb/documents/oracle`

这三者指向同一个产品，只是出现在不同语境里。

产品形态上有两层：

1. **公开内容层**（无需登录）
   - 占星百科词条
   - 占星经典书籍导读（30+ 本）
   - Synthetica 心理综合分析工具
   - Saturn Return Calculator 等免费工具
   - 主语言英文，中文为辅语言
2. **登录后产品层**（账号体系内）
   - 本命星盘解读（Swiss Ephemeris 计算）
   - 每日运势 / 行运分析
   - AI 占星问答（Ask Oracle）
   - 双人合盘（Synastry）
   - 认知行为日记（CBT Journal）
   - 周期视图（Cycles）和报告系统（Reports）

底层逻辑是：**用公开内容承接自然流量，用登录后产品承接付费转化。**

## 二、它面向谁

目标用户画像偏年轻、偏欧美、偏内省成长型。

| 维度 | 描述 |
|------|------|
| 年龄 | 18–35 岁为主 |
| 地区 | 欧美市场为主，主语言英文 |
| 兴趣 | 占星、心理学、自我成长、正念冥想 |
| 行为习惯 | 移动优先，社交媒体活跃，愿意为个性化内容付费 |
| 心理特征 | 倾向把占星当作“理解自己”的工具，而不是“算命” |

它不是要服务“想被预测人生的人”，而是服务“想更理解自己、并愿意为深度内容和个性化分析付费的人”。

这个差别会直接影响内容口径：

- 不强调“注定”“宿命”
- 强调“觉察、模式、成长”
- AI 输出强调赋能（Empowerment）而非命定（Fatalism）

## 三、它当前承担什么业务目标

在公司层面，Astrologywiki 同时承担三个目标，重要程度从高到低：

1. **作为 GenGrowth 增长方法论的真实跑通场景**
   验证“社媒先验证，SEO 再放大”这一方法论是否真的能在欧美市场上跑出有积累的自然流量与付费转化。
2. **作为一个独立可造血的产品**
   通过订阅和积分包获得真实付费收入，验证从流量到转化、再到留存的完整商业链路是否成立。
3. **作为对外讲述 GenGrowth 能力时的代表性案例**
   把抽象的“AI 增长合伙人”具体化成一个真实业务样本，便于对外说明和复用。

商业化模式上目前有两条线：

- **订阅**：$6.99/月，$41.99/年，首次订阅 50% off
- **积分包**：$4.99 / $12.49 / $19.99 / $34.99 四档（USD），同时支持 CNY
- **支付通道**：当前激活 Airwallex；Stripe、PayPal 代码保留但未启用

不同功能采取不同的免费 / 配额 / 积分扣减组合（例如 Ask 默认 3 次/周、合盘 3 次终身免费、心理维度前 3 个免费），目的是让用户低门槛体验，再围绕高价值场景转化付费。

## 四、它和 GenGrowth 的验证关系

这是新员工最容易混淆的一点，需要单独说明。

**GenGrowth 是方法与系统，Astrologywiki 是真实业务场景中的验证样本。**

它不是 GenGrowth 的“子产品”，也不是“被卖给客户的产品”。它的角色更像一块自家运营的 **样板田**：

- GenGrowth 提出方法论，Astrologywiki 用来真实跑一遍
- GenGrowth 给出的策略和工具，Astrologywiki 是第一个、也是最严苛的使用方
- Astrologywiki 跑出的反馈，会反过来修正 GenGrowth 的方法论本身

在两条增长轨道上的对应关系是：

- **社媒测品**：Astrologywiki 上的内容方向、词条选题、表达方式，先在社媒上做反馈测试
- **SEO 固盘**：被验证有效的方向，再沉淀成站内百科与经典书籍内容资产，形成长期可复利的自然流量

可以这样理解三层关系：

```
GenGrowth（方法 / 系统）
    ↓ 应用在
Astrologywiki（真实业务场景）
    ↓ 反馈到
GenGrowth（修正方法 / 沉淀案例）
```

所以日常工作中要避免两种误读：

- 把 Astrologywiki 当成“GenGrowth 顺手做的另一个站”
- 把 Astrologywiki 当成“与 GenGrowth 无关的独立创业项目”

它两者都不是，它是 GenGrowth 主线上的关键验证项目。

## 五、当前进展、重点页面类型和内容结构

下面信息基于当前仓库结构，会随业务推进持续变化，仅作为新员工的初步认知。

### 5.1 当前所处阶段

- 产品已上线、可访问、可付费
- 主域名 `astrologywiki.com`,`www` 子域 301 重定向
- 已具备完整账号体系、订阅和积分支付、AI 内容生成与缓存、E/Z 双语 SEO 静态页面输出能力
- 部署托管在 Vercel,前端 Vite + React,后端 Express Serverless,数据库 Supabase Postgres,缓存 Redis,星盘计算用 Swiss Ephemeris,AI 由 DeepSeek 提供
- 内容资产正在持续积累中,百科词条与经典书籍导读是当前 SEO 增长的主战场

### 5.2 重点页面类型

按战略重要程度可以分成三组。

**A. 公开 SEO 内容页(增长主战场,无需登录)**

| 页面类型 | 路径示例 | 角色 |
|----------|----------|------|
| Wiki Hub(首页) | `/`、`/:lang/wiki` | 站点首页即 Wiki Hub,内容优先策略 |
| Wiki 词条详情 | `/wiki/:id` | 行星 / 星座 / 宫位 / 相位等百科条目 |
| 经典书籍列表 | `/wiki/classics` | 占星经典书籍索引 |
| 经典书籍详情 | `/wiki/classics/:id` | 单本书结构化导读 |
| Synthetica | `/wiki/synthetica`(由 WikiSyntheticaPage 承接) | AI 心理综合分析工具,公开可试用 |
| Saturn Return Calculator | `/:lang/saturn-return-calculator` | 免费独立工具,主要做 SEO 获客与外链 |
| 双语静态页面族 | `public/en/**`、`public/zh/**` | 由构建脚本生成,Vercel 直接托管,服务搜索引擎索引 |

**B. 登录后核心产品页(转化与留存主战场)**

| 页面类型 | 路径 | 角色 |
|----------|------|------|
| Dashboard / 本命盘 | `/dashboard` | 用户进入产品后的主面板,展示星盘解读 |
| Forecast / 每日运势 | `/forecast` | 每日复访入口,行运分析 + 四维评分 |
| Ask Oracle / AI 占星问答 | `/oracle` | 高价值付费功能之一 |
| Synastry / 合盘 | `/us` | 双人星盘对比,共 16 个分析模块 |
| CBT Journal / 认知日记 | `/journal` | 心理学侧产品线,情绪追踪 + 认知分析 |
| Cycles / 生命周期 | `/cycles` | 月相 / 个人年 / 行运周期视图 |
| Reports / 报告 | `/reports`、`/reports/:reportId` | 月度、年度、土星回归等付费报告 |
| Settings / Credits 用量 | `/settings`、`/usage` | 账户与配额管理 |

**C. 转化与合规支撑页**

| 页面类型 | 路径 | 角色 |
|----------|------|------|
| 登录 / 注册 | `/auth` | Google、Apple、Email 验证码三种入口 |
| Onboarding | `/onboarding` | 出生信息收集,直接决定后续个性化质量 |
| Payment 结果页 | `/payment/success`、`/payment/credits-success` | 订阅与积分购买成功页 |
| 法律页面 | `/privacy`、`/terms`、`/cookies`、`/about`、`/help` | GDPR / CCPA 合规与品牌信任所需 |

### 5.3 内容结构大致逻辑

把内容侧的逻辑抽象出来,可以归纳成三层:

1. **公开内容层(SEO 抓手)**
   - 百科词条 + 经典书籍 + 免费工具
   - 主要面向英文搜索流量
   - 强调可被搜索引擎索引、可被分享、可独立阅读
2. **AI 个性化层(转化抓手)**
   - 本命盘解读、Ask Oracle、合盘、Synthetica
   - 基于用户出生信息和星盘上下文,由 AI 生成深度个性化解读
   - 大部分能力以“前几次免费 / 之后配额或积分”的组合做转化
3. **自我陪伴层(留存抓手)**
   - 每日运势、CBT 日记、周期视图、报告系统
   - 目的是让用户产生“持续回来”的理由,而不是一次性消费
   - 也是订阅价值最直接的承接面

新员工只需要先建立这种三层结构的概念,具体每一层下还有哪些子模块,后续再按需展开即可。

## 六、新员工最少需要记住的几点

如果只保留最核心的几点,关于 Astrologywiki 应优先理解的是:

1. 它是一款面向欧美 18–35 岁用户的现代占星 Web 产品,强调“理解自我”而非“算命”
2. 它同时承担 **方法论验证、真实付费产品、对外案例** 三个角色
3. 它和 GenGrowth 是 **样板田与方法论** 的关系,不是独立项目,也不是子产品
4. 增长主战场在公开 SEO 内容,转化和留存主战场在登录后 AI 个性化产品
5. 当前已上线、可付费、可观测,内容资产仍在持续积累中,是公司当前最重要的真实业务样本
