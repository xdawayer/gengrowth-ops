# SEO 运营绩效评估体系 v0.3（早期精简版）

> **适用对象**：当前 SEO 运营（1 人）。社媒/内容岗位入职后可直接复用，仅需替换主指标。
> **状态**：draft，与 SEO 同事公示并跑过一个完整季度后转 stable。
> **关联**：员工手册 §5 绩效管理；公司价值观；GenGrowth 业务能力锚点。

---

## 一、设计哲学

| 原则 | 落地动作 |
|------|----------|
| 弱化量化考核 | 用 `优秀 / 良好 / 合格 / 待改进` 四档，不做加权总分 |
| 不增加管理负担 | 上级单方评估为主，员工自评 1 页问卷；月度不打分 |
| 自动化只做 ROI 高的 | 全程仅 1 个周度脚本 + 1 张季度数据看板 |
| 评估服务于成长 | 不绑短期奖金，目的是反馈、转正/晋升、年终浮动 |
| 服务业务实际场景 | SEO 目标市场为 Google/Bing 生态（美国 + 巴西），不评估百度系能力 |

---

## 二、四维框架

| 维度   | 角色                   | 周期          | 形式          |
| ---- | -------------------- | ----------- | ----------- |
| 目标达成 | 主线（决定整体评级 60% 的判断权重） | 季度结果 + 月度过程 | 数据 + 上级判断   |
| 日常工作 | 主线（决定整体评级 40% 的判断权重） | 周度自动 + 月度回顾 | 脚本输出 + 抽审   |
| 价值观  | 一票否决项                | 季度回顾        | 上级观察 + 自评对齐 |
| 成长性  | 调节项（影响晋升/调薪节奏）       | 季度回顾        | 上级判断 + 自评   |

> "60%/40%"是判断权重，不是数字加权。两条主线决定季度整体评级。
> **价值观**严重不达标可一票否决整体评级；**成长性**不影响当季评级，只影响"是否给更大舞台/调薪/晋升"。

---

## 三、四维考核标准

### 维度 1：目标达成（主线）

#### 主目标 — C 端产品 PV 增长

> **Q2 OKR**：日均 PV 达到 **5,000–10,000**（自然搜索为主，Google/Bing 生态）

**季度评级锚点**：

| 等级 | 锚点 |
|------|------|
| 优秀 | 季末日均 PV ≥ 10,000，且自然搜索为主，健康度好 |
| 良好 | 季末日均 PV 7,000–10,000 |
| 合格 | 季末日均 PV 5,000–7,000 |
| 待改进 | 季末日均 PV < 5,000，或全靠付费/异常流量撑起 |

**健康度参考信号**（不单独评分，作为评级时的修正项）：
- 自然搜索 vs 付费 vs 直接访问的比例
- 长尾词 vs 头部词的分布
- 新页面对增量的贡献占比
- 美国 / 巴西两个核心市场的覆盖均衡性

#### 次目标 — GenGrowth 工作流协同

不打分，定性记录。季度回顾时上级列 3–5 件具体事并点评：
- 反哺到产品的 SOP / 方法论
- 沉淀的可复用资产（关键词主表、模板、Apps Script、prompt 库）
- 跨职能交付物（与产研 / 内容 / 商务的协作产出）

#### 月度（轻流程）

- 上级看一眼 GA4 趋势 + 30 分钟 1on1
- 聊：上月做了什么、下月做什么、有什么卡点
- **不打分**，只对齐方向，不进档案

---

### 维度 2：日常工作（主线）

**核心**：基于公司/业务规范的文档输出（SEO 诊断、SOP、关键词研究、增长实验记录、客户/案例审计）。

**季度评级锚点**：

| 等级 | 描述 |
|------|------|
| 优秀 | 输出量稳定、文档审计基本一次过、规范零失误、抽审内容质量高于预期 |
| 良好 | 输出量与质量稳定，偶有规范小问题但不影响阅读 |
| 合格 | 输出量达成、规范基本合规、抽审内容能用 |
| 待改进 | 输出量明显不足、或文档质量需多次返工、或频繁规范错误 |

**核心审计机制**：`seo-quality-audit` subagent

详细评审标准与 agent 行为约定见 [SEO 输出质量评审标尺 Rubric](2026-05-11-seo-output-quality-rubric.md)。

执行方式：

1. Lynne 在 Claude Code 中手动触发"评估 SEO 输出"
2. Agent 先列出待评审清单，等 Lynne 拍板
3. 按 rubric 跑 P0/P1/P2 三档评分，输出统一格式报告
4. 主文档写入 `docs/05-governance/people-ops/perf-feedback/seo/{YYYY-WW}-quality-audit.md`（SEO 本人可读不可写）
5. 不进 cron，由 Lynne 控制节奏（前期每周 1 次，后期可降为月度抽审）

**配套**：

- 规范类检查（命名/位置/层级）由 `doc-audit` subagent 兜底，质量审计不重复
- 文档"业务价值"判断与"时效性"通过月度 1on1 同步

---

### 维度 3：价值观（一票否决，不计分）

四个价值观对应 SEO 岗位的具体场景。**AI 原生心智协同已融入"进取"和"求真"**。

| 价值观 | 在 SEO 岗位上的具体含义 | 不达标的硬性信号 |
|--------|-------------------------|------------------|
| 进取 | 主动学新工具/AI/算法变化、不等催；**主动用 AI 拆解问题、沉淀 prompt 资产** | 长期被动等待安排 / 拒绝学新工具 |
| 求真 | 数据先行、不把 AI 输出当事实、PV 异常时主动归因 | 虚报数据 / 拿 AI 输出当结论 / 关键决策无依据 |
| 开放 | 接受 PV 下滑的归因反馈、引入外部 SEO 案例 | 长期拒绝反馈 / 闭门造车 |
| 共创 | 沉淀方法论给后来的社媒同事、协助产品迭代 | 拒绝沉淀 / 只做本职不补位 |

**评估方式**：季度上级用 1 页表，每项写 1–2 句具体观察（事件/文档），员工自评对齐。

**一票否决场景**：
- 求真严重违规（虚报数据、刻意误导）→ 当季绩效直接判 `待改进`，无视其他维度表现

---

### 维度 4：成长性（调节项，影响晋升/调薪节奏）

合并为两条主线。

#### 4.1 AI × SEO 复合能力

> 在本公司，"用 AI 工具的能力"和"做 SEO 的能力"是一体的，合并评估。

| 等级 | 表现 |
|------|------|
| 优秀 | 能独立设计完整增长实验；用 AI 拆解复杂任务；沉淀可复用 prompt/工作流；判断算法变化 |
| 良好 | 能用 AI 工具完成 SOP，对常见场景有判断 |
| 合格 | 能执行既定 SOP，会用主流 AI 工具 |
| 待改进 | 机械执行，遇到偏差不会调整，不主动用 AI |

**评估方式**：季度上级回顾 + 1 个具体作业（任选一周用 AI 解决一个真实业务问题，事后口头复盘 15 分钟）。

#### 4.2 思维质量（独立思考 + 框架能力）

| 等级 | 表现 |
|------|------|
| 优秀 | 能挑战既有结论；从单点经验抽象出可复用框架（变成 SOP/模板） |
| 良好 | 能基于现成框架做合理推断 |
| 合格 | 能完成任务但不抽象 |
| 待改进 | 人云亦云，不思考结构 |

**评估方式**：季度上级看其本季沉淀的 SOP/模板/复盘文档，挑 1 篇深聊 30 分钟。

## 四、自动化清单（极简）

| 工具 | 用途 | 频率 | 状态 |
|------|------|------|------|
| `seo-quality-audit` subagent | 按 rubric 评 SEO 输出内容质量 | 手动触发，前期每周 1 次 | 待落地 |
| `doc-audit` subagent | 规范/治理检查（命名/位置/层级） | 按需触发 | ✅ 已可用 |
| GA4 / GSC 后台 | 月度上级直接看 PV 趋势 | 自助 | 默认已接入 |
| 季度评估留档模板 | 季度回顾 | 每季 1 次 | 待落地 |

**不做的**：360 互评、笔试系统、飞书机器人推送、信号扫描脚本、月度自动报告。团队 ≥ 3 人再说。

## 五、季度评估流程（唯一的"正式"流程）

```
季末前 1 周
  ① 员工 1 页自评（4 维度，每维度写 3–5 条具体事 + 自我评级）
  ② 上级看 GA4 + 周度 snapshot 汇总 + 抽审 2 份文档
  ③ 上级写 1 页评估（4 维度 + 综合评级 + 下季成长建议）
  ④ 30–60 分钟 1on1 对齐
  ⑤ 留档：docs/05-governance/people-ops/{YYYY-QN}-perf-review-seo.md
```

**整体评级规则**：
- 基于"目标达成 + 日常工作"两条主线 → `优秀 / 良好 / 合格 / 待改进`
- 价值观严重不达标 → 一票否决，无视其他维度
- 成长性单独一段写"是否给更大舞台 / 调薪 / 晋升建议"

**不做的**：
- 不算加权总分
- 不做强制分布
- 不与季度奖金挂钩（季奖看公司业绩）
- 不做多人校准会

## 六、社媒/内容岗入职后的扩展

社媒同事到岗后，本体系直接复用，仅替换：

- **目标达成**主指标 → 社媒粉丝增长 / 互动率 / 引流到 C 端 PV 的贡献
- **日常工作**核心 → 内容产出 + 文档沉淀；同样跑 doc-audit
- **价值观 / 成长性** → 完全复用

团队达 3 人时再考虑加：
- 同事互评（即使 2 人互评也可以）
- AI 原生协同的横向观察
