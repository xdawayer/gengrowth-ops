# 2026 Q3 增长目标拆解

> **用途**：把 Q3 团队目标从上到下讲清楚，方便向马博洋、彭满、产研直接传递——大家知道**为什么这个目标、自己那截该做什么、怎么算达成**。
> **关联**：`2026-06-16-growth-team-roles-and-division`（SSOT，谁负责哪条线）、`SEO 运营绩效评估体系`、路径 B 选词立项标准 `docs/repo/gengrowth-ops/inbox/02-keyword-research/2026-06-26-路径B选词汇报`。
> **设计原则**：**顶层一个数字焊死，下层全部弹性可调**。因为现在很多数据模型（每访客值多少钱、转化率）还不确定，硬设精确的下层数字是假精确。顶层固定给方向和士气，下层"找增长点、随数据按月调"。

---

## 一、北极星（Q3 固定不动）

> **日流水（分成后）达到 $1,000/天，并能持续。**

- **口径**：与客户分成的产品，按**分成后**归我们的收入算。
- **成本 30%**：只是**方向性概念，不做严格核算、不作闸门**。前期投入集中、没业绩时成本占比会 >100% 是正常的；30% 是"上规模后该往哪收敛"的锚，不参与里程碑判定。
- **为什么是收入不是流量**：收入是确定、无歧义、所有人都能对齐的目标。**产品层和流量层都为这一个数字服务**——不为"跑量"跑量，为"能变现的增长"找增长点。

---

## 二、达成路径

### 产品端（供给 + 变现）

1. **稳健验证路线**：astrologywiki 持续做，**补 iOS 端**；GenGrowth 持续打磨与推广。
2. **流量路径（快速拿量）**：重点做**热词、能快速获量的产品**，有立项与筛选标准（见路径 B 选词汇报）。目前已产出 **2 个工具站产品**，未来继续拓展，并可能**形成工具聚合站、集中权重**。
3. **外部合作产品**：倾向**商业模式已基本验证的增量型产品**（web + iOS 的工具与游戏）。对外 BD 由**创始人**承接。
4. **商业化能力**：有流量之后，重点提升变现——**创始人给方向、产研负责人承接**，含**广告变现 + 订阅付费**。

### 营销端（流量 + 增长点探索）

1. **SEO**：路径已初步打通，Q3 重点**针对以上产品做增长点探索**，不是维持现状。
2. **社媒**：刚启动，**边探索边沉淀 SOP**，重点同样是**找增长点**。

## 四、流量层口径：主指标用 UV，不用 PV

**结论：流量层主指标 = qualified UV（导向优先变现产品的独立访客），不是 PV。**

理由——按"谁控制什么"拆：

```
PV       = UV × 每次访问看几页（depth）
Sessions = UV × 每人来几次（visits/user）
```

- **depth（看几页）= 产品粘性/类型**，wiki 一个用户翻十页、工具站进来算一次就走——**这是产研的事**。用 PV 考核流量团队 = 让他们为产品设计背锅，且跨产品不可比（wiki 会把工具站淹掉）。
- **visits/user（来几次）= 留存**，也是产品的事。
- **UV（来了多少个不同的人）= 拉新**，才是流量团队真正可控、且跨产品可比的产出。

| 指标 | 归谁 |
|---|---|
| **UV（拉新，导向优先变现产品）** | 流量层 = 马博洋 / 彭满 |
| PV/UV、depth、留存、转化率、RPM/ARPU、$/visitor | 产品层 = 产研负责人 / 创始人 |

> 口径细节：UV 用 GA4 用户口径，**统计窗口定死**（日 UV ≠ 月 UV，不能相加）；只算**优先变现产品**的 UV（qualified UV），防止靠给不变现页面灌垃圾流量刷数。

## 五、Q3 前段必须先跑出的输入：$/UV 基线

要把 "$1,000/天" 反推成 "该做多少 UV"，缺一个输入：**每个产品的"每 UV 值多少钱"（$/UV = 广告 RPM 换算 或 CVR×ARPU）**。现在不知道，也不该拍脑袋。所以 **Q3 前段流量团队的目标是双重的**：

1. 给优先产品**做 UV 增长 + 找增长点**；
2. 与产研一起，用早期真实数据**跑出各产品的 $/UV 基线**。

有了 $/UV，才能算：`$1,000/天 ÷ 各产品 $/UV = 需要多少 UV`，再把 UV 目标分到 SEO / 社媒。**顺序是"先探索、后设硬数"**，匹配"数据模型还不确定"的现实。

**初步 UV 锚（先给团队一个数值概念，非锁定硬指标）**：按默认 **$0.05/UV（美国用户）** 估算——

> **$1,000/天 ÷ $0.05/UV ≈ 20,000 qualified UV/天**（全部优先产品合计）。

这是增长同学的**起始靶子**。随真实 $/UV 修正：**单 UV 价值低于 $0.05 就把 UV 目标调高**（如实测 $0.025/UV → 需约 40,000 UV/天），高于则调低。不同产品差异大（工具站广告低、订阅高），先按 $0.05 混合默认起步，**季中按真实数据分产品重设**。

### Q3 流量层目标（口径 = Q3 月底达到即可，均按 /天）

| 线 | 核心指标 | 当前基线（/天） | Q3 月底目标（/天） | 倍数 |
|---|---|---|---|---|
| 总盘 | qualified UV | 28 | **20,000** | ~700× |
| SEO（马博洋） | GSC 点击 | 13 | **8,000** | ~600× |
| 社媒（彭满） | 曝光量（reach） | 0（刚启动） | **~1.6 万**（≈50 万/月，季末几天日均） | 0→ |

> **口径**：均为 **Q3 月底达到即可**（end-of-Q3 run-rate），不是全季度均值；曝光按季末最后几天日均评估。

> **现实校准（怎么正确理解 20k）**：20,000 UV/天**不是**"把现有一个站的 28 UV 做到 700 倍"，而是**一个产品组合的合计**——astrologywiki + 多个热词工具站 + 外部合作产品。**外部合作只是产品归属在外，SEO 和社媒仍由马博洋/彭满做，流量全部计入他们的产出。**
> - **量级来自"产品数 × 单产品流量"**：如 10–15 个产品各 ~1–2k UV/天 → 合计 1.5–3 万/天。增量靠**多产品铺开**（热词工具站低成本批量造、低竞争快起量），不是单站暴涨。"700×"只是"从一两个产品铺到十几个"的自然结果。
> - **可行性的三个真变量**：① Q3 能上线并起量的**产品数 × 命中率**；② 单产品流量上限；③ **$/UV（变现）——这是成败命门**：纯广告 $0.03/UV，20k 也只有 $600/天、单产品经济太薄；必须靠**订阅/更高 RPM 把 $/UV 推到 $0.1–0.2**（产研负责人变现职责），$0.1 × 20k = $2000/天，模型才宽裕。
> - **别纠结"单站能不能 700×"，那是假问题**；真问题是产品数、命中率、$/UV。
>
> **社媒 Q3 定位**：直接 UV 早期就是几百/天，**不指望它扛量**；价值是 reach 累积 + Social Probe 喂词 + 新品冷启动 + 沉淀 SOP。工具站为主的阶段 **SEO 是主力、社媒是辅助**（社媒更适合社区/可病毒化产品）。彭满 Q3 按"**打基础 + 探索 + 支持冷启动**"考核（过程 + reach 增长 + SOP），不背大 UV 数。

### 订阅类产品怎么算收入（$/UV 的一种）

> 术语先分清：**ARPPU** = 每**付费**用户收入；**ARPU** = 每**用户/访客**（含免费）收入。**MRR 和 $/UV 都用 ARPPU**，下面统一按 ARPPU = $10。

- **MRR（月经常性收入）** = 活跃付费用户 × 月 ARPPU；**日流水** = MRR ÷ 30（确认口径）或当日实收（新签 + 续费，现金口径）。
- **反推北极星**：$1,000/天 → MRR ≈ $30k → 若 ARPPU $10，需 **~3,000 活跃付费用户**。
- **$/UV（订阅）** = 访客→付费 CVR × ARPPU × 计入月数；**Q3 保守只计首月**：如 CVR 2% × $10 = **$0.20/UV**。
- **对比广告**：广告 $/UV ≈ (PV/UV) × RPM/1000 ≈ 3 × $10/1000 = **$0.03/UV**。**订阅每访客值钱约 6–7 倍**——选优先产品时看这个。

## 六、里程碑激励机制

**原则：不见真金白银不发奖金（有收入才有奖金池，自筹），发多少跟达成程度挂钩。**

- **启动线（已定）**：**达成率 ≥ 50% 且单位经济跑正**，才开始计奖金池；之下不发。
- **达成率**：`近 14 天平均日流水 ÷ $1,000`，用**"持续"口径**（防某天侥幸冲高就触发）。
- **发放曲线**：过启动线后**连续放大**——达成率越高发得越多，到 100% 给满，超过 100% 另加 overachievement kicker。避免"差一点拿零"的悬崖。
- **奖金池比例（Q3 暂不锁定）**：Q3 先**不定"从流水切多少进池"**——成本结构和后续增长空间还不确定，过早锁死不合理。团队先知道规则：**过了 50% 且赚钱，才开始算奖，越接近 $1,000 越多**；具体切多少比例，**季中/季末看真实数据再定**。

## 七、执行纪律

1. **顶层（$1,000/天）Q3 内不动**；下层所有数字（UV 目标、$/UV、转化）**按月随数据滚动调**。
2. 每月一次目标回顾：更新 $/UV 基线、重设下月 UV 方向、确认优先产品清单。
3. 优先产品清单（哪些产品值得导流）由**创始人 + 产研**定，流量团队据此投放精力，不自行选战场。

## 附录：流量指标定义与关系（备注）

> 为什么核心指标选 UV 而非 PV/新增/sessions，看完这节就清楚。

### 定义

| 指标 | 是什么 | 谁测 |
|---|---|---|
| **PV** 页面浏览量 | 页面被打开总次数，一人看 5 页 = 5 PV | GA4 |
| **UV** 独立访客 | 一段时间**不同的人**（去重）= GA4 Total Users | GA4 |
| **Session/Visit** 会话/访问 | 一次访问（30 分钟无操作或跨天算新会话）；visit = session | GA4 |
| **新/老用户** | 新 = 第一次来；老 = 来过又回 | GA4 |
| **GSC 点击** | 从 Google 搜索结果点进网站的次数 | Google（搜索端） |

### 层级与数量关系

一层套一层：`GSC 点击 →（部分）Sessions 来源`；`UV（人）─×每人几次会话→ Sessions ─×每次几页→ PV`；`UV = 新用户 + 老用户`。

- `Sessions = UV × 每人会话数`（≥1，看**留存/回访**）
- `PV = Sessions × 每次几页`（≥1，看**产品粘性/类型**）
- `GSC 点击 ≈ 自然搜索会话`（Google 端测，通常比 GA4 高 10–30%，因 GA4 埋点会漏记）

### 数字例子：同样 1,000 UV，两类产品 PV 差 3 倍

| 一天、同 UV=1,000 | 内容站 | 工具站 |
|---|---|---|
| UV | 1,000（=800 新 + 200 老） | 1,000 |
| Sessions | 1,300 | 1,050 |
| **PV** | **3,900** | **1,200** |
| GSC 点击 | ~700 | ~850 |

> 同样拉来 1,000 人，PV 差 3 倍多——差的是**产品粘性/类型**，不是流量团队干得好坏。**UV 两边都是 1,000，才是公平的"拉新"对比。**

### 为什么核心用 UV（不用 PV / 新增 / sessions）

- **不用 PV / sessions**：混进了产品粘性（depth）和留存（回访），是产品的事、流量团队不可控。
- **UV vs 新增用户**：老用户有两种——**粘性回流**（产品功劳）vs **营销回流**（新帖子/新排名把老访客拉回来，流量团队功劳），GA4 分不开。新增会**漏掉营销回流**；UV 会**多算粘性回流**。**早期站点老用户基数小、UV ≈ 新增，用 UV 更标准**；盯"新/老占比"，回访占比暴涨时再重审是否改用新增。
- **GSC 点击≈新用户**：搜索来的以新/非忠实用户为主（老用户直接输域名/收藏，不搜索），是干净的"搜索拉新"代理，SEO 线用它兜底口径。
