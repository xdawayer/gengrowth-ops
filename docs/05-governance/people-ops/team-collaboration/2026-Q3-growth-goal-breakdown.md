# 2026 Q3 增长目标拆解

> **用途**：把 Q3 团队目标从上到下讲清楚，方便向马博洋、彭满、产研直接传递——大家知道**为什么这个目标、自己那截该做什么、怎么算达成**。
> **关联**：`2026-06-16-growth-team-roles-and-division`（SSOT，谁负责哪条线）、`SEO 运营绩效评估体系`、路径 B 选词立项标准 `docs/repo/gengrowth-ops/inbox/02-keyword-research/2026-06-26-路径B选词汇报`。
> **设计原则**：**顶层一个数字焊死，下层全部弹性可调**。因为现在很多数据模型（每访客值多少钱、转化率）还不确定，硬设精确的下层数字是假精确。顶层固定给方向和士气，下层"找增长点、随数据按月调"。

---

## 一、北极星（Q3 固定不动）

> **日流水（分成后）达到 $1,000/天，并能持续。**

- **口径**：与客户分成的产品，按**分成后**归我们的收入算。
- **成本 30%**：只是**方向性概念，不做严格核算、不作闸门**。前期投入集中、没业绩时成本占比会 >100% 是正常的；30% 是"上规模后该往哪收敛"的锚，不参与里程碑判定。
- **为什么是收入不是流量**：收入是确定、无歧义、所有人都能对齐的目标。**产品层和流量层都为这一个数字服务**——不为"跑量"跑量，为"能变现的增长"找增长点。

---

## 二、达成路径

### 产品端（供给 + 变现）

1. **稳健验证路线**：astrologywiki 持续做，**补 iOS 端**；GenGrowth 持续打磨与推广。
2. **流量路径（快速拿量）**：重点做**热词、能快速获量的产品**，有立项与筛选标准（见路径 B 选词汇报）。目前已产出 **2 个工具站产品**，未来继续拓展，并可能**形成工具聚合站、集中权重**。
3. **外部合作产品**：倾向**商业模式已基本验证的增量型产品**（web + iOS 的工具与游戏）。对外 BD 由**创始人**承接。
4. **商业化能力**：有流量之后，重点提升变现——**创始人给方向、产研负责人承接**，含**广告变现 + 订阅付费**。

### 营销端（流量 + 增长点探索）

1. **SEO**：路径已初步打通，Q3 重点**针对以上产品做增长点探索**，不是维持现状。
2. **社媒**：刚启动，**边探索边沉淀 SOP**，重点同样是**找增长点**。

## 四、流量层口径：主指标用 UV，不用 PV

**结论：流量层主指标 = qualified UV（导向优先变现产品的独立访客），不是 PV。**

理由——按"谁控制什么"拆：

```
PV       = UV × 每次访问看几页（depth）
Sessions = UV × 每人来几次（visits/user）
```

- **depth（看几页）= 产品粘性/类型**，wiki 一个用户翻十页、工具站进来算一次就走——**这是产研的事**。用 PV 考核流量团队 = 让他们为产品设计背锅，且跨产品不可比（wiki 会把工具站淹掉）。
- **visits/user（来几次）= 留存**，也是产品的事。
- **UV（来了多少个不同的人）= 拉新**，才是流量团队真正可控、且跨产品可比的产出。

| 指标 | 归谁 |
|---|---|
| **UV（拉新，导向优先变现产品）** | 流量层 = 马博洋 / 彭满 |
| PV/UV、depth、留存、转化率、RPM/ARPU、$/visitor | 产品层 = 产研负责人 / 创始人 |

> 口径细节：UV 用 GA4 用户口径，**统计窗口定死**（日 UV ≠ 月 UV，不能相加）；只算**优先变现产品**的 UV（qualified UV），防止靠给不变现页面灌垃圾流量刷数。

## 五、Q3 前段必须先跑出的输入：$/UV 基线

要把 "$1,000/天" 反推成 "该做多少 UV"，缺一个输入：**每个产品的"每 UV 值多少钱"（$/UV = 广告 RPM 换算 或 CVR×ARPU）**。现在不知道，也不该拍脑袋。所以 **Q3 前段流量团队的目标是双重的**：

1. 给优先产品**做 UV 增长 + 找增长点**；
2. 与产研一起，用早期真实数据**跑出各产品的 $/UV 基线**。

有了 $/UV，才能算：`$1,000/天 ÷ 各产品 $/UV = 需要多少 UV`，再把 UV 目标分到 SEO / 社媒。**顺序是"先探索、后设硬数"**，匹配"数据模型还不确定"的现实。

> 量级示意（纯假设、待真实数据替换）：工具站广告若约 $0.02/UV、订阅产品若约 $0.20/UV，则达 $1,000/天需要的 UV 差 10 倍——所以现在定不了 UV 硬指标是正常的，先把 $/UV 跑出来。

## 六、里程碑激励机制

**原则：不见真金白银不发奖金（有收入才有奖金池，自筹），发多少跟达成程度挂钩。**

- **奖金池**：从**实际跑出来的流水**里切一块（比例由创始人定）——自带"有钱才发"的纪律，前期没业绩不掏钱发奖。
- **达成率**：`近 14 天平均日流水 ÷ $1,000`，用**"持续"口径**（防某天侥幸冲高就触发）。
- **发放曲线**：设一条**启动线**（如达成率 50% / 单位经济跑正之后才开始计池），之下不发；之上**连续放大**，到 100% 给满，超过 100% 再加 overachievement kicker。
  > 这样既保留"没钱不发"，又避免"$999 拿零"的悬崖——最可能的近似达成也能拿到对应奖励，不打击士气。

> **待创始人确认**：启动线设在哪、从流水切多少比例进池、档位怎么分、个人如何分配。机制先定，金额后填。

---

## 七、执行纪律

1. **顶层（$1,000/天）Q3 内不动**；下层所有数字（UV 目标、$/UV、转化）**按月随数据滚动调**。
2. 每月一次目标回顾：更新 $/UV 基线、重设下月 UV 方向、确认优先产品清单。
3. 优先产品清单（哪些产品值得导流）由**创始人 + 产研**定，流量团队据此投放精力，不自行选战场。
