# 增长团队角色与分工（单一事实源 / SSOT）

> **用途**：增长团队"谁负责 S1–S9 哪一段"的**单一事实源**（SSOT = Single Source of Truth，软件/数据领域通用叫法：某件事只在一处写权威版，其他地方引用而不复制）。各人岗位职责文档、绩效文档、各 SOP 在涉及分工时**一律引用本文，不复制**，避免双源漂移。
> **状态**：draft，三角色到齐并跑过一个周期后转 stable。
> **关联**：`2026-05-15-gengrowth-internal-growth-mvp-prd-v0.8`（S1–S9 流程定义）、`GenGrowth整体营销框架`（元层/对象层）、`social-media-operations`（账号分层）、`SEO 运营绩效评估体系`（绩效框架）。

## 一、三个执行角色（一句话）

| 角色 | 在岗 | 一句话职责 | 核心对象 |
|---|---|---|---|
| **SEO 运营**（马博洋） | 1 人（在岗） | 需求侧到内容侧：定位卡 → 关键词 → 集群 → 内容生产（S1–S5），并**全权负责外链（标的 + 联系 + 内容 + 把控）** | 见下方统一对象 |
| **AI 内容运营**（彭满） | 在岗 | 把内容通过自有社媒/社区扩散成真实增长渠道（S6 自有线），跑 Social Probe，沉淀社媒 SOP | 见下方统一对象 |
| **KOL 运营** | **待补** | 对接外部 KOL/KOC，由对方之口发布；管关系、谈判、确保发布与追踪（S6 借势线） | 见下方统一对象 |

> **统一核心对象（三角色相同）**：早期以 **astrologywiki C 端**为主 → 后续扩展到**合作发行的 C 端产品** + **GenGrowth 官网（B 端）**。三人都围绕这组对象工作，不各自割裂。

> 说明：**创始人**不在上表的"执行角色"内。创始人负责 S1 业务判断的参与、对外审批，以及**元层 build in public**（创始人账号 + 官方账号、透明度即营销）——把三人在对象层产出的真实实验上升为对外叙事。元层不归本表三角色维护。

## 三、S6 三条子渠道细分（三角色唯一重叠区，重点）

S6「分发与外联」长出三条并列子渠道，按"自有 / 服务 SEO / 借势"区分归属：

| 子渠道                              | 性质         | 策略归属                                          | 执行归属                                                       |
| -------------------------------- | ---------- | --------------------------------------------- | ---------------------------------------------------------- |
| **自有社媒分发 + Social Probe / 社区观察** | 我们自己的账号发声  | 彭满                                            | 彭满                                                         |
| **外链**                           | 内容为 SEO 服务 | **SEO 马博洋**（标的页、锚文本、质量门槛）                     | **SEO 马博洋**（联系、PR、换链内容均自己把控）；彭满在社区里发现的外链机会作为 lead 转交，不接手执行 |
| **KOL / KOC 外部合作发布**             | 借第三方红人之口   | **KOL 运营**（红人寻源、谈判、合作策略）；带链时 SEO 提供**可选 anchor/target 池**（非硬指令） | **KOL 运营**（管关系、确保发布、追踪，可因创作者适配/平台风控否决某 anchor）；**彭满**供品牌信息/卖点/素材包/红线 |

**三个防撞车原则**：

1. **自有 vs 借势**：彭满管"我们自己发"，KOL 运营管"别人帮我们发"，不重叠。
2. **外链整条归 SEO**：外链需要端到端 SEO 判断（标的+锚文本+联系+内容+把控），现由马博洋全权负责；彭满/KOL 不动主表，只在各自渠道发现外链机会时回流给马博洋。
3. **内容供给**：KOL 内容由红人按自己口吻创作，内容运营只供品牌卖点/素材/红线，不代写。
4. **KOL 锚文本不硬指**：带链优先级 = **创作者真实性 > 平台合规 > 转化 > SEO 锚文本精确**。SEO 给可选 anchor/target 池，不下单一 exact-match 硬指令（伤创作自然度 + 过度优化 footprint 惩罚风险）；KOL 运营可因创作者适配否决某 anchor，争议创始人裁决（稀有例外）。

**四个交界接口（谁交什么给谁，全部挂 cluster_id）**：

1. **SEO → 内容（内容供给）**：SEO 每产出一篇集群内容，内容运营负责分发放大到自有社媒；分发目标页 = 该集群 pillar。
2. **内容 → SEO（信号回流）**：内容运营把高共鸣话题 / 高互动选题整理成关键词候选回流 S3，SEO 决定是否纳入关键词主表。
3. **内容 → KOL（素材供给）**：内容运营供品牌信息 / 卖点 / 素材包 / 红线；红人按自己口吻创作，内容运营不代写。
4. **三方共享测量**：三人共用一张内容资产表，按 cluster_id 聚合排名 / 流量 / 社媒回流 / 合作来源，S7 复盘一起做。

> 一句话边界：**SEO 决定"做什么内容、外链指哪"，内容运营决定"自有内容怎么扩散、社媒侧该反过来做什么"，KOL 运营管"借谁的口发"，测量闭环三人共享。** 内容生产、关键词主表只归 SEO 写；内容 / KOL 通过"信号回流"影响，不直接动主表。

### 3.1 S6 机会分流（Triage）+ 接口 SLA + 仲裁

> 上面的归属是「谁拥有哪条线」，但真实机会不按线天然出现——一个 Reddit 高赞帖 / niche blogger / TikTok creator，**同时可能是社媒、外链 lead、KOL lead**。本节补「谁拍板归哪条线、多久回应、被拒怎么办」。**最轻形态：一条默认规则 + 共享 Sheet 一个 tab + 周例会 5 分钟，绝不上重 schema。**

**默认分流规则（覆盖约 90%，无需讨论）**：

| 发现的机会                      | 默认归     | 依据          |
| -------------------------- | ------- | ----------- |
| 付费 / 正式合作、借对方之口发           | KOL 运营  | 有自己受众、需谈判   |
| 有网站 / 页面、能给我们反链（无偿 earned） | 马博洋（外链） | earned link |
| 社区 / 帖子，用我们自己口吻互动          | 彭满（自有线） | owned voice |

**流程（落在共享 Sheet 的 `triage` tab，不是独立文件）**：

1. 谁发现谁记一行：`opportunity_id / discovered_by / 可能类型 / 转给谁 / 状态（待定·已接·已拒）/ 拒因 / cluster_id`。
2. **接口 SLA**：转给某人后 **3 个工作日内**回应（接 / 拒 + 理由），不得烂尾。
3. **仲裁**：默认规则盖不住、且双方分歧的（约 10%），创始人一行字定。**仲裁是稀有例外——默认规则兜不住 90% 就是规则有问题，不是天天找创始人**。
4. 周一 S8 例会花 5 分钟过 `triage` tab，不靠口头转交（口头必丢球）。

## 四、绩效考核（收敛：一个主考核 + 每人一个重点指标 + 其余参考）

**主考核 = 各角色带来的 qualified UV**（导向优先变现产品的独立访客，按 cluster_id / 渠道分开算）。每人一个**重点指标**作为 UV 的代理/杠杆，其余全部降为**参考过程指标，不作主评分**：

| 角色 | 主考核 | 重点指标 | 参考（过程）指标 |
|---|---|---|---|
| SEO 马博洋 | 带来的 UV | **GSC 点击量** | 外链产出量/质量、DR、排名、信号采纳 SLA、SOP 沉淀 |
| 彭满（社媒） | 带来的 UV | **曝光量（reach）**（+ 转化保障作质量门） | Social Probe 信号质量、粉丝/互动率、SOP 沉淀 |
| KOL 运营（待补） | 带来的 UV | **合作引流/转化** | 合作数量/质量、KOL SOP |

**保留两条防扯皮原则**：

- **激励解耦**：彭满不被"马博洋采纳了几个信号"考核（同级单边裁量）；信号回流是 Sheet 查表状态（候选/已纳入/驳回），作参考、不作主评分。
- **assisted 不硬塞 last-click**：外链/KOL 对排名的滞后贡献、社媒引发的品牌搜索，季度定性复盘时纳入，不逼 GA4 UTM 直接归。

## 五、绩效框架统一引用

三角色绩效**统一复用** `SEO 运营绩效评估体系` 的四维框架（目标达成 / 日常工作 / 价值观 / 成长性 + 四档 + 一票否决）。**"目标达成"主指标 = §四的"带来的 UV + 各自重点指标"**（UV 口径与理由见 `2026-Q3-growth-goal-breakdown`），不再逐项细分。

> **SOP 沉淀作参考指标**，口径是「**被复用过的 SOP**」不是「沉淀了几篇」（防凑数）——被另一个人/另一个周期真正复用过才算数。
> **价值观红线**：不刷量、不造假数据，触发一票否决。

详见各自绩效文档与 `SEO 运营绩效评估体系 §6`。

## 六、可衡量指标、数据来源与三人拆分

> **三人指标能分开算的前提 = 两把钥匙**：每个分发动作都带 `cluster_id`（挂哪个集群）+ `UTM`（哪条渠道、谁发的）。GA4 按 UTM 自动拆渠道，共享 Sheet 按 cluster_id 聚合。

> **⚠️ UTM 是观测字段，不是绩效归因的唯一钥匙（评审点 2）**：外链/KOL 对排名的提升滞后 2–8 周、最终表现为 Organic/Direct（不再带原 UTM）；KOL 曝光引发的品牌搜索 GA4 也归不到 KOL。**只看 last-click UTM 会系统性少算外链/KOL/社媒的 assisted 贡献，尤其少算彭满。** 所以绩效拆两层：
> - **direct attributable**：UTM 能直接归的，照算（下表）。
> - **assisted contribution**：外链/KOL/社媒对排名、品牌搜索、自然流量的滞后影响，用 `action_id + cluster_id + publish_date + 预期效果` 记进共享 Sheet，**季度定性复盘时纳入**——不硬塞 GA4 last-click，也不初期就建正式 ledger。

### 6.1 指标 × 数据来源对照

| 指标                             | 归属     | 数据来源                                     | 怎么取 / 拆分键                                                                                                  |
| ------------------------------ | ------ | ---------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| **自然搜索 qualified UV**、收录率、关键词排名   | 马博洋    | GSC + GA4                                | 现成后台；GA4 channel=Organic Search（**用户口径 UV，非 PV**，见 Q3 目标拆解）                                                 |
| DR、引用域名、外链产出量/质量               | 马博洋    | Ahrefs + **外链台账（待建）**                    | Ahrefs 自动 + 手动台账记每条链接相关性/质量                                                                                |
| **社媒可归因引流 UV**（可拆到平台 + 帖子 + 落地文章） | 彭满     | GA4                                      | 链接带 4 段 UTM：`source`=平台、`medium=social`、`campaign=cluster_id`、`content=action_id（帖子）`；落地文章看 GA4 着陆页（见 6.2） |
| 社媒资产（粉丝 / 曝光 / 互动率）            | 彭满     | 各平台原生分析（X / Reddit / TikTok / Pinterest） | 平台后台，每周手动汇总进周报                                                                                             |
| Social Probe 信号回流命中            | 彭满     | **共享 Sheet 回流清单列**                       | 手动记"提了几个 → 马博洋采纳几个"，周例会核                                                                                   |
| KOL 曝光                         | KOL 运营 | 红人提供截图 + 平台                              | 手动进 KOL 台账                                                                                                 |
| **KOL 引流 / 转化**                | KOL 运营 | GA4                                      | KOL 链接：`utm_source=kol-{红人}`、`utm_medium=kol`、`utm_campaign=cluster_id`（**纯 cluster_id，红人放 source**，否则 KOL 流量没法按 cluster_id 聚合）→ 关联 CTA / 工具事件 |
| KOL 合作数量 / 质量                  | KOL 运营 | **KOL 合作台账（待建）**                         | 手动记合作数 + 质量评分（受众匹配/交付质量）                                                                                   |

### 6.2 三人怎么同步、不打架

1. **唯一数据脊柱 = 那张共享 Google Sheet**（选题登记 / 内容资产表）。三人所有动作挂 cluster_id，写进同一张表，不各存一份。
2. **一个 GA4 属性 + 一套 UTM 命名约定**：
   - **必填（两项，缺了就拆不开人/平台）**：
     - `utm_source` = **具体平台**（reddit / pinterest / x / tiktok；KOL 用 `kol-{红人}`）→ 拆到"哪个平台 / 哪个红人"
     - `utm_medium` = **渠道类型**（social / backlink / kol / newsletter）→ 三人各自前缀，天然把流量拆开
   - **可选（能填则填，做深度分析用；不填不影响主考核）**：
     - `utm_campaign` = **cluster_id**（如 `CL-AW-05`）→ 拆到"哪个集群"
     - `utm_content` = **`action_id`（帖子 id，取自 6-ID）** → 拆到"哪条帖子带来的"
   > utm_content 填的是**帖子 id，不是文章 id**；"落到哪篇文章"由 GA4 **着陆页**自动记录（= `page_id`），不用塞进 UTM。主考核只需 source+medium 就能分清谁带来的 UV；campaign/content 是想深挖时才加。
3. **每周一 S8 例会 + 周报脚本 = 唯一对齐点**：脚本从 GA4 / GSC / Ahrefs + Sheet 拉数，生成一页"分渠道 / 分人"看板。平台原生数据（粉丝/曝光）和手动台账（外链/KOL/信号回流）由各人每周自报进周报，wzb / Lynne 抽查。

## 七、已知风险与触发条件（盯住，暂不改）

1. **马博洋是决策队列瓶颈（评审点 4）**：外链不是自动产线、是人肉 outreach（联系/PR/跟进/质控），叠加 S7 主 + S8 审核 + S9 主，协作接口都排队等他判断。**暂不拆**——没助理/外包落点，且与"外链需端到端 SEO 判断、整条归他"的既定决策冲突。**触发条件**：若彭满/KOL 因等马博洋判断而停工 > 1 个周期，启动"决策权（target / anchor / quality / approve）vs 执行劳动（outreach 跟进 SOP 化外包）"拆分。
2. **创始人勿成新瓶颈**：triage 仲裁、KOL 锚文本争议、元层交接都可能甩给创始人。**必须靠默认规则兜住约 90%，创始人只接稀有例外**，否则换汤不换药。
3. **元层↔对象层交接（评审补 1）**：创始人 build-in-public 靠三人喂素材，但没定谁/多久/交什么 → **周报固定一栏「本周可对外讲的实验点」**，低成本补上。
