# GenGrowth 团队邮箱协作与别名配置手册

日期：`2026-04-13`  
维护团队：`GenGrowth Ops Team`

关联文档：
1. `docs/05-governance/account-access/2026-03-12-platform-account-registration-playbook.md`

## 文档目的

本手册用于指导团队成员在个人 `Gmail` 或企业 `Gmail` 客户端中，配置并使用 `GenGrowth` 的别名邮箱（如 `social@`、`letty@`）进行收发，确保：

1. 对外身份统一。
2. 团队响应及时。
3. 回复时不暴露错误发件身份。

## 适用范围

适用于以下场景：

1. 团队成员需要通过个人 `Gmail` 或企业 `Gmail` 代发 `GenGrowth` 别名邮箱。
2. 管理员需要为团队开启“应用专用密码”能力。
3. 成员需要使用 `smtp.gmail.com` 完成别名发送配置。

## 第一阶段：管理员全局配置

此步骤仅需管理员执行一次，用于确保所有成员具备生成“应用专用密码”的权限。

### 操作步骤

1. 登录 `Google Admin Console`。
2. 进入 `安全性 > 身份验证 > 两步验证`。
3. 确保“强制执行”已开启，方法选择“任何方法”。
4. 勾选“允许用户生成应用专用密码”并保存。

## 第二阶段：生成 16 位应用专用密码

每位需要配置别名的成员，需在自己的 `Google Workspace` 账号下执行。

### 操作步骤

1. 登录 Google 账号安全性页面。
2. 确保已开启“两步验证”。
3. 登陆myaccout.google.com/passwords，或在页面顶部搜索框输入“应用专用密码”（`App Passwords`）。
4. 创建密码：
   应用名称填写 `Gmail-Alias`。
5. 点击“创建”。
6. 系统会显示一串 16 位黄色底纹代码，请立即复制。

### 重要提示

1. 该密码仅显示一次，不要在第三阶段完成前关闭窗口。
2. 复制时即使包含空格也通常不影响识别，但建议粘贴前手动去掉空格。
3. 不要使用日常登录 `Workspace` 的普通密码代替应用专用密码。

## 第三阶段：客户端绑定别名设置

在您常用的个人 `Gmail` 或企业 `Gmail` 中完成以下配置。

### 操作路径

1. 打开 `Gmail`。
2. 进入 `设置 > 查看所有设置 > 账号和导入`。
3. 在“用这个地址发送邮件”区域，点击“添加其他电子邮件地址”。

### 信息填写

1. 姓名：填写对外显示名称，例如 `Letty | GenGrowth`。
2. 电子邮件地址：填写您的别名，例如 `social@gengrowth.ai`。
3. 勾选“作为别名对待”。

### SMTP 配置

| 字段 | 配置值 |
|---|---|
| SMTP 服务器 | `smtp.gmail.com` |
| 端口 | `587` |
| 用户名 | 您的 `Workspace` 主账号全称，例如 `admin@gengrowth.ai` |
| 密码 | 刚刚生成的 16 位应用专用密码 |
| 连接方式 | `TLS` 安全连接 |

### 验证步骤

1. 点击“添加”。
2. 在收件箱中查收验证码邮件。
3. 将验证码填回弹窗，完成验证。

## 第四阶段：优化回复设置

此步骤用于避免收件人看到错误的回复身份。

### 操作步骤

1. 回到 `设置 > 账号和导入` 页面。
2. 找到“回复邮件时”设置项。
3. 勾选“从收到邮件的地址回复”（`Reply from the same address the message was sent to`）。

## 常见问题排查

### Q1：为什么提示“密码错误”？

必须使用 16 位“应用专用密码”，不能使用平时登录 `Workspace` 的普通密码。

### Q2：配置第二个别名时，可以复用同一个 16 位代码吗？

建议为每个新别名单独生成一个应用专用密码，以降低触发 Google 安全拦截的风险，也更便于后续单独撤销。

### Q3：发送邮件时，对方会看到我的真实地址吗？

按上述 `TLS` 配置完成后，对外默认显示的是您设置的别名地址，而不是您的日常登录地址。

## 执行检查清单

在实际启用前，请逐项确认：

1. 管理员已开启“两步验证”与“允许应用专用密码”。
2. 成员已开启两步验证。
3. 成员已生成并妥善保存 16 位应用专用密码。
4. 别名已在 `Gmail` 中完成 SMTP 绑定。
5. 验证码已完成确认。
6. “从收到邮件的地址回复”已开启。

## 备注

1. 若后续更换客户端、撤销设备授权或触发 Google 风控，可能需要重新生成应用专用密码。
2. 建议团队将别名用途、负责人和授权范围纳入统一台账管理，避免多人混用同一发件身份。
