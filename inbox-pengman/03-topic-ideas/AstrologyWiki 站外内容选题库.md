# AstrologyWiki 站外内容选题库

> 这是一个持续追加的灵感库，不是最终生产表。  
> 这里优先收录“适合站外内容表达”的主题、Hook、用户问题和可转入后续生产的候选选题。  
> 原则上，单条内容不必死绑站内页面；但每个主题包应知道大致承接方向。
> 当前主线是：先围绕 AstrologyWiki 站内主要功能和基础内容做站外内容，再穿插 SEO 需要引用的 Wiki 文章，最后根据高流量内容和 30-50 名关键词继续提排名。

## 1. 这个文件怎么用

- 先收灵感，再分主题包。
- 先判断适不适合平台表达，再决定要不要进入生产。
- 如果某个选题能自然对应到 [[inbox/00-inbox/topic-cluster-table.csv]] 里的 cluster，就顺手写上。
- 如果后续要进入正式生产，可以再转到 [[inbox-pengman/04-production/01-strategy-and-platform-research/0616AstrologyWiki 站外内容平台调研与首轮运营方案初稿.md]] 的实验节奏或选题记录里。
- 优先顺序建议是：

```text
站内主要功能 / 基础内容
→ SEO 需要引用的 Wiki 文章
→ 站外可表达的图文 / 视频脚本
→ 高流量内容
→ 30-50 名关键词提排名
```

### 1.1 选题记录字段

建议每条灵感都尽量补齐这些信息：

| 字段 | 说明 |
|---|---|
| Topic | 选题名 / Hook |
| Platform | YouTube / Shorts / TikTok / Pinterest / X / Reddit |
| Theme | Birth Chart / Houses / Aspects / Saturn Return / Astrology Literacy 等 |
| Angle | 教学 / 辟谣 / 图卡 / 故事 / 反思 / 调研 |
| Cluster | 如果能对应到主题集群，就写 cluster_id |
| Entry Point | 用户已知入口，比如 sun sign / zodiac sign / horoscope |
| Natural Next Step | 可自然承接的方向，不一定是链接 |
| Risk | 宿命化 / 过度预测 / 标签化 / 诊断化 / 无 |
| Status | Idea / Draft / Test / Archived |

## 4. 目前值得优先继续看的主题包

### 4.4 Saturn Return

### 4.5.1 生产候选：Harry Kane Leo Sun / England vs DR Congo

- Theme: Current event + Birth Chart / Sun Sign literacy

#### Higgsfield / CapCut 制作备注

- 用 Higgsfield 生成旁白、无文字竖屏图和少量轻动效；不要让 Higgsfield 生成可读文字。
- 第一条样片建议走 Higgsfield 网页端，方便手动上传素材、挑声音和看效果。
- 模板跑通后，可用 Codex 批量拆分镜、生成 prompt、调用 Higgsfield 并整理素材。
- 最小制作包：1 条旁白 + 5-6 张竖屏图 + 2-3 条轻动效 + CapCut 字幕 / CTA。
- CTA 推荐：`Read Harry Kane's full birth chart on AstrologyWiki`。

## 5. 初步风险提醒

- 太像预测、好运承诺、关系判词的题目，优先降速。
- 只适合做猎奇、但不适合做品牌表达的 Hook，不要直接拿来当长期主线。
- 适合 TikTok 的题，不一定适合 YouTube 轻量视频，反之亦然。
- 一条内容可以没有明确页面链接，但最好知道它是为了导流、互动、调研，还是品牌表达。

## 6. 后续补充建议

如果后面你想把这个选题库再往前推一步，可以分成三张更细的子页：

1. 入口主题库：只放 Sun Sign -> Birth Chart 相关的题。
2. 平台 Hook 库：只放 TikTok / Shorts / Pinterest 的可发布标题。
3. 生产候选库：只放已经能转入正式 production 的题。
