# Higgsfield 视频自动化工作流调研

> 本文根据目前收集到的 Higgsfield / Claude Code / CapCut 自动化视频流程资料整理。当前没有重新观看原 YouTube 视频，只基于已提供的调研内容做归纳和判断。

## 1. 这个工作流大概在做什么

这个流程本质上不是“AI 直接生成完整视频”，而是：

```text
脚本 / 配音
→ 转写出带时间戳的文稿
→ 每个时间戳生成一张对应画面
→ 自动下载并按时间戳命名
→ 在 CapCut 里按时间线拼接成短视频
```

它的核心价值是：

- 把“每几秒换一张图”的视频画面自动生成出来；
- 用时间戳解决音频和画面对齐问题；
- 减少手动找素材、做图、命名、整理文件的时间。

## 2. 拆解后的技术流程

| 步骤        | 工具                            | 输入         | 输出              | 作用           |
| --------- | ----------------------------- | ---------- | --------------- | ------------ |
| 1. 写脚本    | ChatGPT / Claude / 人工         | 主题、结构、口播文案 | 视频脚本            | 决定内容逻辑       |
| 2. 录音     | 真人录音 / AI 配音                  | 脚本         | 音频文件            | 作为视频主线       |
| 3. 转写时间戳  | Turboscribe 等                 | 音频         | 带时间戳文稿          | 告诉后续每个画面出现时间 |
| 4. 拆分画面需求 | Claude Code / ChatGPT / Codex | 时间戳文稿      | 每个时间点的画面 prompt | 把文字转成视觉描述    |
| 5. 生成图片   | Higgsfield                    | 图片 prompt  | 多张图片            | 为每个时间戳生成画面   |
| 6. 下载和命名  | Claude Code / Codex 脚本        | 图片链接、时间戳   | 本地图片文件          | 按时间戳自动整理素材   |
| 7. 剪辑合成   | CapCut                        | 音频 + 图片    | MP4 视频          | 最终成片         |

## 3. Claude Code 在里面的作用

原流程里 Claude Code 主要承担三个任务：

1. 读取带时间戳的脚本；
2. 批量调用 Higgsfield 生成图片；
3. 把生成的图片下载到本地，并按时间戳重命名。

但这些并不是 Claude 独有能力。理论上可以用：

- ChatGPT 生成结构化 JSON；
- Codex 写批量生成 / 下载 / 重命名脚本；
- Higgsfield 负责生成图片；
- CapCut 负责剪辑。

也就是说，Claude Code 更像是一个方便的自动化外壳，不是不可替代的核心。

## 4. 如果不用 Claude，能不能复现？

可以，但需要把 Claude Code 做的事拆开：

```text
ChatGPT：拆脚本、生成每个时间戳的画面 prompt
Codex：写脚本，调用 Higgsfield，下载图片，按时间戳命名
Higgsfield：生成图片 / 视觉素材
CapCut：导入音频和图片，手动或半自动对齐
```

更适合我们的低成本版本：

```text
短脚本 15–45 秒
→ 每 3–5 秒一个画面
→ 生成 5–12 张图
→ CapCut 拼成 Shorts
```

不建议一开始就做几十上百张图的长视频自动化。

## 5. 对 AstrologyWiki 的可行性判断

### 5.1 可行性：中高

Higgsfield 适合尝试做 AstrologyWiki 的短视频视觉素材，尤其是：

- 占星基础知识科普；
- “一个概念 + 一张图”的解释型 Shorts；
- 自我认知 / journal prompt 类短视频；
- 热点内容里的辅助画面；
- 简笔画、图解、抽象视觉、星空 / 星盘氛围图。

但它未必适合直接生成最终视频。更现实的定位是：

> Higgsfield 生成画面素材，CapCut 完成最终视频。

### 5.2 难易程度：中等

如果只是手动用 Higgsfield 生成几张图，再放进 CapCut，难度不高。

如果要复现视频里的自动化流程，难度会提高，主要难点在：

- 是否能顺利安装和登录 Higgsfield CLI / MCP；
- 是否能稳定批量生成；
- 图片链接能否被脚本下载；
- 时间戳和图片 prompt 是否能一一对应；
- CapCut 最后仍需要人工检查节奏和字幕。

因此建议分阶段：

| 阶段 | 做法 | 难度 |
|---|---|---|
| 阶段 1 | 手动生成 5–10 张图，CapCut 拼接 | 低 |
| 阶段 2 | ChatGPT 生成时间戳 + prompt 表 | 低到中 |
| 阶段 3 | Codex 写下载、重命名脚本 | 中 |
| 阶段 4 | 完整自动批量调用 Higgsfield | 中到高 |

### 5.3 效率判断：适合短视频测试，不适合一开始重度自动化

对于 15–45 秒短视频，如果只需要 5–12 张画面，Higgsfield 可能比手动画图 / 找素材更快。

但如果要搭建完整自动化，前期会有配置成本。当前更建议：

- 先不用追求全自动；
- 先用 Higgsfield 做视觉素材测试；
- 观察画面质量、生成速度、费用和可控性；
- 如果连续 10–20 条短视频都能复用，再考虑 Codex 自动化。

## 6. 对当前内容方向的适配

| 内容类型 | 是否适合 Higgsfield | 说明 |
|---|---|---|
| 占星知识科普 | 适合 | 可生成简笔画、图解、符号化画面。 |
| 占星 + 心理学 / 自我认知 | 适合 | 可做抽象情绪、journal、人物思考画面。 |
| 高 SEO 内容改写 | 适合部分内容 | 适合把概念变成视觉卡片，不适合复杂长文直接视频化。 |
| 热点内容 | 谨慎适合 | 可做辅助画面，但名人、赛事素材要注意版权和形象风险。 |
| 星盘精确讲解 | 不完全适合 | 星盘截图和符号最好用真实页面 / Canva / CapCut 控制，避免 AI 画错。 |

## 7. 推荐试验方案

先做一个小实验，不要马上上完整自动化。

### 实验 A：基础科普短视频

主题示例：

- Why Your Sun Sign Is Not Your Whole Chart
- What Is a Birth Chart?
- Patterns, Not Predictions

制作方式：

```text
15–30 秒脚本
→ 拆成 5 个画面
→ Higgsfield 生成 5 张图
→ Canva / CapCut 加字幕
→ YouTube Shorts + TikTok + X 发布
```

### 实验 B：占星 + 自我认知短视频

主题示例：

- Your Moon Sign Is About Emotional Needs
- Astrology Is a Reflection Tool
- A Birth Chart Is Not a Destiny Verdict

适合用更抽象、简笔画、journal 风格画面。

### 实验 C：热点内容辅助画面

主题示例：

- celebrity zodiac sign explained
- sports moment + astrology literacy

注意：热点视频里不要依赖 AI 生成名人真实肖像，优先用符号化画面和文字卡片。

## 8. 初步结论

Higgsfield 值得尝试，但当前不建议把它定义为“视频主工具”。更准确的定位是：

> Higgsfield = 视觉素材生成工具；CapCut / VEED = 成片工具；ChatGPT / Codex = 脚本和自动化工具。

当前建议优先测试：

1. 它生成的画面是否符合 AstrologyWiki 的品牌风格；
2. 生成速度和费用是否能接受；
3. 是否比 Canva 手工图卡更省时间；
4. 画面错误率是否可控；
5. 能否稳定用于 15–45 秒短视频。

如果测试结果好，再考虑搭建“时间戳 → prompt → 批量生成 → 自动命名 → CapCut”的半自动流程。

## **9. 第一次实测（2026-06）**

为了验证 Danny Why 的工作流是否能够在 **ChatGPT + Codex + Higgsfield** 环境下复现，本次进行了最小流程测试。

测试内容包括：

- 安装并配置 Node.js、Higgsfield CLI、Higgsfield Skills；
- 登录 Higgsfield Free Plan（10 Credits）；
- 使用一段约 20 秒、6 个时间戳的测试脚本；
- Codex 自动生成 `manifest.json` 和图片任务清单；
- 调用 Higgsfield CLI 生成并下载前 2 张图片。

测试结果：

- ✅ 整个「时间戳 → Prompt → 图片下载」流程成功跑通；
- ✅ Codex 可以替代 Claude Code 完成图片生成、下载和任务管理；
- ✅ 使用 `text2image_soul_v2` 模型，每张图片约消耗 **0.12 Credits**，2 张图片共消耗 **0.24 Credits**，成本较低；
- ⚠️ 默认 Prompt 容易生成错误文字，需要明确要求 **No text**；
- ⚠️ Danny Why 的 MS Paint 风格未必适合 AstrologyWiki，后续计划测试更符合品牌风格的插画方案。

### **结论**

本流程验证成功，可行性较高。对于 15–45 秒、5–10 张图片的短视频来说，整体成本较低，自动化程度较高，适合作为 AstrologyWiki 短视频的生产方案。当前最大的优化方向是**图片风格一致性、Prompt 质量以及 CapCut 自动拼接流程**，暂不建议直接扩展到长视频或大规模批量生成。

## 10. 成本估算：简笔画 / 图解素材路线

> 以下是本次调研记录的价格和积分估算，后续实际付款前仍需以 Higgsfield 官网当时的套餐和账单为准。

简笔画自动化路线的核心优势是：**它主要生成图片，而不是直接生成高精视频**。因此成本会明显低于真人数字人 / C-Dance 视频。

### 10.1 单张图片成本

调研估算：

- 极简文生图 / MS Paint 风格简笔画：约 **0.1–0.5 credits / 张**；
- 如果按加购价 `$49 / 1000 credits` 粗算，单张约 **$0.005–$0.02**；
- 之前最小流程实测中，`text2image_soul_v2` 每张约 **0.12 credits**，但正式预算建议按 **0.5 credits / 张** 保守预留。

### 10.2 单条视频成本

以 5 分钟视频为例：

```text
5 分钟 = 300 秒
每 3 秒换 1 张图
约 100 张图
100 张 × 0.5 credits = 50 credits
```

估算结果：

| 视频长度 | 图片数量假设 | 积分消耗 | 粗略费用 |
|---|---:|---:|---:|
| 30 秒 | 10 张 | 约 5 credits | 约 $0.25 |
| 1 分钟 | 20 张 | 约 10 credits | 约 $0.5 |
| 5 分钟 | 100 张 | 约 50 credits | 约 $2–$3 |

如果购买约 `$49 / 1000 credits` 的额度，理论上可以支撑约 **20 条 5 分钟简笔画 / faceless 视频**。

### 10.3 成本结论

这条路线适合当前阶段原因：

- 成本低；
- 废片率低，因为画风本来就是极简、粗糙、非真实；
- 生成失败的风险比真人脸、口型、手部动作低很多；
- 适合科普、故事、解释型内容的批量测试。

因此，Higgsfield 简笔画 / 图解素材路线可以继续作为 AstrologyWiki 短视频素材测试方向；但仍建议先做短视频小样，不直接扩展成长视频矩阵。
