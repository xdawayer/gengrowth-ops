# AstrologyWiki YouTube 调研与运营方案交接摘要

> 用于新对话快速接续。新对话应先读本文件，再读方案初稿；不要重新推翻已确认的方向。

## 1. 当前核心口径

AstrologyWiki 站外内容运营的核心目标不是涨粉，也不是把 YouTube / 社媒账号本身做大，而是：

> 通过 YouTube、Shorts、Pinterest、X 等站外内容，为 astrologywiki.com 的公开工具页、Wiki 页面和重点 SEO 页面带来直接或间接访问，支持网站 PV 增长。

因此所有内容、平台和实验都要绑定：

```text
SEO 主题 / 关键词 → 视频/帖子选题 → 目标页面 → CTA → 站外指标 → 站内同步指标
```

## 2. 用户的现实制作约束

首轮视频必须是低成本 MVP：

- 不真人出镜；
- 不做复杂动画；
- 不做 20–30 分钟完整大课；
- 用户视频制作经验有限；
- 项目目标有时间限制，需要尽快验证“站外 → 站内 PV / 工具启动”路径。

因此方案已调整为：

1. 优先做知识卡片型 Shorts；
2. 优先做 AstrologyWiki 页面/工具录屏型 Shorts；
3. 可做 3–6 分钟轻量视频，用静态图卡 + 星盘局部高亮 + 页面录屏 + 配音/字幕完成；
4. 复杂动画、高制作长课、真人 IP 口播暂缓。

## 3. 已确认的平台优先级

| 平台 | 当前判断 |
|---|---|
| YouTube 轻量视频 | P0，但首轮只做 3–6 分钟轻量视频，绑定目标页面和 CTA |
| YouTube Shorts | P0 配套，优先知识卡片、公式卡、工具录屏 |
| Pinterest | P1/P2，潜在 PV 路径比 Reddit 更直接，需继续调研 |
| X | P1，适合话题测试和页面分发，账号归属待确认 |
| Reddit | 用户研究 P0 / 分发 P3；调研雷达，不是扩音器 |
| TikTok | P2，暂不做首轮主平台；只观察 Hook |

## 4. Reddit 已调整过的口径

Reddit 不作为首轮发布/宣传/导流主平台。

定位：

> Research radar, not distribution channel.  
> 调研雷达，不是扩音器。

用途：收集真实问题、社区语言、FAQ、反感点和高频误解。只有在规则允许、回答完整、链接确有帮助时，才低频使用链接。

## 5. Birth Chart YouTube 长视频调研结论

已调研 3 个样本：

1. The Astrology Podcast — 学院派、专业克制、硬核录屏；内容可信，但太长太硬核。
2. The Gem Goddess — 亲切、个人 IP、free will / autopilot 口径强；导流 PDF 值得借鉴，但不适合照搬 30 分钟口播。
3. Mastering the Zodiac — 结构最清晰，用“剥洋葱”解盘顺序，视觉标准化，长线爆款；但使用 Sidereal Astrology，AstrologyWiki 应改为 Tropical Astrology。

当前结论：Birth Chart 应进入 YouTube P0 首批内容，但首轮不做 8–10 分钟理想版，而做 3–6 分钟轻量版本。

推荐方向：

- 标题可参考：`How to Read Your Birth Chart Without Feeling Overwhelmed`
- 备选：`How to Read a Birth Chart in 10 Minutes: Patterns, Not Predictions`
- 默认占星系统：Tropical Astrology
- 形式：静态图卡 + 星盘局部高亮 + AstrologyWiki 页面/工具录屏 + 配音/字幕
- CTA：Primary 指向 Birth Chart Calculator / Dashboard；Secondary 指向 Birth Chart Guide；Optional 才是 Checklist

## 6. Birth Chart Shorts 调研结论

Shorts 不是压缩版系统视频，而是单点认知切片。

适合 AstrologyWiki 的 Shorts 类型：

| 类型 | 说明 |
|---|---|
| 反常识 / 辟谣型 | 如 “Your sun sign is not your whole chart” |
| 焦虑缓解型 | 如 empty houses / hard aspects 不等于坏事 |
| 解盘公式型 | Planet = what, Sign = how, House = where |
| 知识卡片型 | 7–15 秒速查表，高收藏率，适合 Wiki 工具属性 |
| AI 工具工作流型 | AI-assisted chart reflection，但要避免 AI 算命和读图错误 |

需要避免：

- 单一配置判词；
- 宿命化“隐藏钥匙”；
- 心理诊断化表达；
- “灵魂外挂”“人格分裂”等高刺激但不安全的说法。
