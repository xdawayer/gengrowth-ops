# AstrologyWiki 站外内容运营方案、工具与下一步讨论记录

## 1. 本轮讨论背景

本轮围绕 [[inbox-pengman/00-inbox/0616AstrologyWiki 站外内容平台调研与首轮运营方案初稿.md]] 做了多轮调整，目标是把原本较长、偏调研底稿的文档，整理成更容易执行和沟通的方案稿。

核心方向：

- 站外内容运营不是为了单纯做大社媒账号，而是服务 astrologywiki.com 的 PV 增长；
- 但内容创作不能被目标页面“死绑”，否则会压缩平台表达和创作空间；
- 更合理的方式是：**主题包有承接方向，单条内容根据平台语境决定是否导流**。

## 2. 已对初稿做过的主要调整

### 2.1 重构文档结构

将原文从“调研底稿 + 方案 + 样本库 + SOP”混合状态，调整为：

1. 下一步行动摘要；

4. SEO 协同与首轮主题；
5. 平台优先级与内容形态；
6. YouTube / Shorts 首轮方案；

8. 首轮 2 周实验计划；
9. CTA、数据与协作机制；
10. 内容创作工具与 AI 辅助方案；
11. 风险与待确认问题；
12. 附录：YouTube 样本、Shorts Hook、调研模板、平台补充调研。

### 2.2 前置“下一步行动摘要”

在文档最前面新增/强化了：

### 2.3 调整目标页面与内容的关系

用户指出：目标页面和内容如果死绑，可能影响创作。

因此文档口径已从：

> 每条内容必须能回到一个站内页面或工具。

调整为：

> 不要求每条内容死绑页面；但每个主题包最好知道可回到哪里。

并新增了内容类型区分：

| 类型 | 主要目标 | 是否需要明确 CTA |
|---|---|---|
| 导流型 | 引导访问工具页 / Wiki / 指南 | 建议有 |
| 互动型 | 测试 Hook、评论、收藏、分享 | 不一定 |
| 调研型 | 收集用户问题和真实表达 | 不需要硬导流 |
| 品牌型 | 建立 AstrologyWiki 的内容边界和价值观 | 可无链接 |

当前判断：**不是每条内容都背导流 KPI，而是每个主题包有增长方向；单条内容可以完成导流、互动、调研或品牌表达中的不同任务。**

## 3. 平台策略讨论结论

### 3.1 YouTube / YouTube Shorts

- YouTube 轻量视频和 YouTube Shorts 仍是首轮 P0；
- 首轮不做真人出镜、不做复杂动画、不做 20–30 分钟长课；
- 推荐用图卡、录屏、字幕、配音、简单转场完成；
- Shorts 用于测试 Hook、收藏、评论、主页访问、轻量视频跳转和品牌搜索。

### 3.2 TikTok

用户指出：TikTok 可以复用 YouTube Shorts 内容。

已将文档口径调整为：

> TikTok 不作为独立主平台，但可以复用 YouTube Shorts。

具体含义：

- 不为 TikTok 单独开制作线；
- YouTube Shorts 完成后，可同步或轻改后发布到 TikTok；
- 用来观察 Hook、完播、收藏、互动和主页访问；
- TikTok 暂不承担首轮核心导流任务。

### 3.3 Pinterest

- Pinterest 适合常青知识卡片、速查表、图解、journal prompts；
- 可以作为 P1/P2 小规模测试；
- 需要确认外链路径、账号发布机制和 Pin → 页面访问路径；
- 不建议一开始大量生产素材。

### 3.4 X

- X 适合作为观点测试、轻量分发和用户问题雷达；
- 不建议首轮押注单条链接帖带来明显 PV；
- 账号归属需确认。

### 3.5 Reddit

- Reddit 是用户研究 P0，不是分发主渠道；
- 用来收集真实问题、FAQ、社区语言和反感点；
- 不建议硬导流。

## 4. 工具与平台讨论结论

### 4.1 基础工具栈

首轮推荐：

```text
Canva
→ 图卡、Pinterest Pin、YouTube Shorts 画面、封面

CapCut / 剪映
→ Shorts 剪辑、自动字幕、配音、简单转场、导出

Figma（可选）
→ 长期视觉模板、组件库、品牌规范

Obsidian / 表格
→ 选题、脚本、CTA、发布记录、复盘
```

暂缓：

- Premiere Pro；
- After Effects；
- 复杂动画；
- 重型 AI 视频工作流。

理由：当前瓶颈不是专业剪辑能力，而是选题、脚本、模板、平台反馈和复盘。

### 4.2 AI 辅助工具

推荐作为提效工具，而不是全自动内容生产工具。

| 用途 | 工具 | 首轮定位 |
|---|---|---|
| Hook / 脚本 / 图卡文案 | ChatGPT / Claude / Gemini | 把 SEO 主题转成平台表达 |
| 字幕 / 配音 / 简单 AI 剪辑 | CapCut AI / 剪映 AI | 与现有剪辑流程结合 |
| AI 视频 B-roll | Seedance / 可灵 / Runway / Pika | 只生成 3–8 秒背景、开头视觉或转场素材 |
| AI 语音 | ElevenLabs / CapCut TTS | 英文旁白备选 |
| 批量模板视频 | Remotion | 第二阶段再用，先手工验证内容规律 |

### 4.3 Seedance / 可灵判断

- 可以用于少量 B-roll、动态背景、星空、月相、抽象视觉；
- 不建议用它们生成整条解释型视频；
- 不要为了“电影感”牺牲知识清晰度；
- 首轮更重要的是图卡清楚、脚本准确、CTA 自然。

### 4.4 Remotion 判断

Remotion 可以帮上忙，但适合第二阶段。

适合：

- 知识卡片型 Shorts；
- Myth vs Truth 系列；
- journal prompts；
- FAQ；
- 星盘术语解释；
- 从 CSV / JSON 批量生成模板化视频草稿。

不适合：

- 首轮创意探索；
- 每条都要细调的自由剪辑；
- 非技术人员快速上手的主流程。

推荐路径：

```text
阶段 1：Canva + CapCut 手工做 10–20 条，找有效内容格式
阶段 2：Figma / Canva 固化视觉模板
阶段 3：Remotion + CSV / JSON 批量生成 Shorts 草稿，再由 CapCut 精修
```

## 5. 当前首轮生产流建议

```text
1 个 SEO 主题 / 目标页面 / 承接方向
→ AI 辅助生成 Hook、图卡文案、Shorts 脚本
→ 人工检查内容边界和 CTA
→ Canva 做 3–5 张图卡
→ CapCut / 剪映剪成 1–2 条 Shorts
→ YouTube Shorts 发布
→ TikTok 复用或轻改
→ Pinterest / X 复用图卡
→ Reddit 只观察问题，不硬导流
→ 记录平台指标、用户反馈和可能的站内承接信号
```

## 6. 需要继续确认的问题

1. YouTube / TikTok / Pinterest / X 是否新建 AstrologyWiki 独立账号？由谁持有？
2. 首轮哪些主题已有比较自然的站内承接页面、工具或 SEO 集群？
3. 是否已有品牌视觉素材：Logo、字体、颜色、图标、截图规范？
4. 是否允许使用 Canva Pro / CapCut Pro；如果需要，谁审批？
5. 站内数据由谁同步：PV、美国流量、referral、CTA 点击、工具启动、注册？
6. 发布前是否需要审核？审核人是谁？

## 7. 当前核心共识

> 首轮不是要把所有平台都做起来，而是用低成本工具和少量内容，验证哪些主题、表达和内容形态能为 AstrologyWiki 带来用户反馈、品牌认知和可能的站内访问。

> 内容不需要被目标页面死绑；但主题包应该有清楚的承接方向，这样既保留创作自由，也不让站外运营和 SEO / PV 目标脱节。

## 8. TikTok 补充讨论（2026-06-16）

本轮进一步明确 TikTok 的首轮定位。

用户判断：TikTok 可以做，但当前不适合单独开新的制作线路。主要原因是独立制作成本较高，而 YouTube Shorts 的内容可以直接复用或轻改后发布到 TikTok。

已知观察：

- TikTok 上搜索占星相关内容，流量和内容供给都较高；
- `birth chart` 等关键词下可以看到较多高互动内容；
- 常见内容形态包括真人出镜讲解占星知识，时长大约 15 秒到 3 分钟以内；
- 也有低成本的卡片静止画面、动态背景、字幕、音乐组合类视频；
- 高刺激内容如 lucky placements、wealth、relationship、attraction 等可能有流量，但需要谨慎处理，避免宿命化、好运承诺和标签化。

当前共识：

> TikTok 可以做，但首轮作为 YouTube Shorts 的复用发布平台，不单独开制作线。重点是低成本测试 Hook、播放、完播、收藏、评论、主页访问、品牌搜索和用户问题。

如果 TikTok 复用内容表现明显优于 YouTube Shorts，再考虑将 TikTok 从 P2 复用位升级为 P1 短视频实验平台。
