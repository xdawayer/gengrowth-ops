# 配图提示词：我们怎么用 Obsidian、GitHub、终端和 skills，搭企业级主工作流

## 统一风格基线

```text
深石墨灰或哑光黑背景，高级手绘战略图 / 高质感信息图风格，带轻微粉笔或白板笔质感，但绝不要廉价黑板报感。
整体像 founder 在解释一套真实系统，理性、克制、可信。
配色以米白、低饱和青蓝、淡青绿、柔和橙黄为主，高对比但不过度艳俗。
画面有信息图感，但不拥挤，不做复杂 dashboard，不做赛博朋克，不做卡通幼稚风。
尽量少文字，复杂标签后期手动添加。
正文插图尺寸：16:9
封面图比例：2.35:1
```

## 正文插图 1：整体架构总览图

建议插入位置：
第一节“三个层次：从 prompt 到 context 到 harness”之后。

Prompt：

```text
请生成一张用于微信公众号正文插图的横版高质感手绘战略图，主题是“小团队企业级主工作流的整体架构”。

画面采用清晰的系统架构构图，整体为深石墨灰背景，高级手绘信息图风格，带轻微粉笔或白板笔质感，但不要廉价黑板报。

核心视觉：
在画面中心放置一个“AI大模型/推理与生成引擎”的抽象核心节点，作为整套系统的能力内核。它不是孤立助手，而是连接上下文、版本、执行和流程的中心引擎。

围绕中心节点分布四个结构化模块：
左侧：知识库层，表现为本地文档树、笔记卡片、引用关系、文档网络
右上：版本层，表现为版本节点、提交链路、分支、系统记录
右下：执行层，表现为终端窗口、文件读写、生成与改写动作
下方或外圈：流程边界层，表现为可复用流程块、模块化任务、skills 调用边界

同时在画面上方或外圈，用更轻的方式表现三层方法论：
prompt engineering、context engineering、harness engineering
但不要依赖复杂文字表达，可以更多用层级关系、边框和结构带来暗示。

整体气质：
理性、克制、可信、像 founder 在解释一套真实系统
不要品牌 logo
不要复杂说明文字
不要做成 PPT
不要做成普通流程图
不要拥挤
保留适度留白
```

## 正文插图 2：知识库与文档分层图

建议插入位置：
第二节“上下文不只要有位置，还要有分层”之后。

Prompt：

```text
请生成一张用于微信公众号正文插图的横版高质感手绘信息图，主题是“知识库不只要有位置，还要有分层”。

整体风格延续高级手绘战略图语境：深石墨灰背景，米白、青蓝、淡青绿、柔和橙黄线条与模块，结构清晰，克制，不拥挤，不廉价。

画面核心是一个分层知识库结构，像一个清晰的文档系统或信息架：
第一层：正式文档
第二层：过程记录
第三层：外部输入
第四层：对外资产

不要依赖大量文字，可以用四层抽屉、卡片堆栈、文件架、分层模块来表现“不同类型内容各归其位”。

在画面一侧加入一个抽象的 AI 读取窗口或上下文视窗，表现 AI 从这个分层结构中读取上下文。
要突出一个概念：
“上下文不是越多越好，而是越清楚越好。”

视觉感觉要传达：
有位置
有边界
有层次
不混乱
适合企业级主工作流

不要 logo
不要复杂中文大段文字
不要卡通人物大头
不要做成技术 dashboard
```

## 正文插图 3：PRD 流程图

建议插入位置：
第六节“一条 PRD 链路，看这套系统怎么运转”中，案例开始前或案例后。

Prompt：

```text
请生成一张用于微信公众号正文插图的横版高质感手绘流程图，主题是“从对话到系统：一份 PRD 如何进入企业级主工作流”。

风格延续同一套公众号视觉系统：
深石墨灰背景，高级手绘战略图风格，结构化、清晰、克制，有适度信息图感，但不要像 PPT 或普通流程图。

画面采用从左到右的流程结构：

最左侧：
自然语言讨论和需求澄清，表现为聊天气泡、需求卡片、背景信息输入

中间第一段：
PRD 草稿生成，表现为正式文档卡片从对话中被提炼出来

中间第二段：
知识库归位 + 版本链路，表现为文档进入文档树、再进入版本节点和记录链路

中间第三段：
评审与确认关口，表现为一个明显的收敛节点、检查节点、确认节点，传达“不是所有内容都自动往下流”

最右侧：
衍生为 SPEC、执行文档、任务块，进入稳定系统

画面必须突出两个感觉：
1. 从聊天到正式系统的转换
2. 权限和确认在中间逐渐收紧

不要复杂文字
不要 logo
不要人物写实
不要廉价流程图
不要满屏箭头
整体要像一张高质量的产品方法论文插图
```

## 封面图 Prompt

```text
请生成一张适合微信公众号封面的横版头图，比例 2.35:1，主题是“从分散对话到企业级主工作流”。

整体视觉语境延续微信公众号前几篇文章的风格：
深石墨灰或哑光黑背景，高级手绘战略图 / 高质感信息图风格，带轻微粉笔或白板笔质感，但绝不要廉价黑板报感。
整体像 founder 在解释一套真实系统，理性、克制、可信，同时具备公众号封面的传播性。

构图要求：
主视觉集中在画面中左区域，右侧预留较大干净空间，用于后期手动添加标题。
画面不要过满，要有留白，但不能空。

核心画面：
左侧起点是分散的对话、零散信息、笔记卡片、需求输入，表现“聊天、草稿、碎片化上下文”
中间是一个清晰的 AI 引擎 / 系统核心节点，连接各类信息流
右侧逐渐转化为结构化系统：文档树、版本节点、终端执行、流程模块、评审关口、稳定的协同链路

画面要体现：
对话不是终点
系统才是承接
AI 不只是一个助手，而是进入了一套正式工作流

可加入一个非常简洁的操作者剪影或抽象角色，增强记忆点，但不要写实人物，不要大头像
可加入少量箭头、模块卡片、节点连接，但不要做成复杂流程图

色彩要求：
米白、低饱和青蓝、淡青绿、柔和橙黄为主
高对比但克制
不要赛博朋克
不要紫色霓虹
不要低质海报红黄大字感

禁止：
不要 logo
不要品牌元素
不要大段文字
不要复杂仪表盘
不要廉价黑板报
不要卡通幼稚风
不要培训海报感

目标效果：
这是一张关于 AI 工作流架构的深度文章封面，不是工具教程，也不是普通产品宣传图。
让人一眼感觉到：这篇文章在讲，如何把 AI 从聊天助手，变成能进入正式业务系统的工作节点。
```
