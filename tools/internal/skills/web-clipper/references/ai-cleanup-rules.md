# AI 纠错规则

验证通过后，对剪藏文档执行最终纠错。**仅修正错误，不改变原文语义。**

## 必须修正

### 1. 无意义字符
- 零宽空格 `\u200b`、零宽连接符 `\u200d`
- 装饰性分隔符（如 `♪(*^^)o∀ - 到底了`、`===END===`、重复的 `---` 分隔线）
- 仅含表情符号的空行或无实际内容的段落
- 连续多余空行（超过 2 个空行合并为 1 个）

### 2. 错字与语法
- 明显的错别字（如「己经」→「已经」、「在得」→「再得」）
- 中英文混排标点错误（中文语境用了英文逗号/句号，反之亦然）
- 缺失的标点（句末无标点的正文段落）
- 英文拼写错误（如 `teh` → `the`）

### 3. Markdown 格式修正
- 未闭合的粗体 `**`、斜体 `*` 标记 → 闭合或移除
- 空的链接 `[text]()` → 移除链接标记，保留文本
- 空的图片引用 `![]()` → 移除
- 表格中的 `<br>` → 保留（表格内换行需要）
- 标题前后缺少空行 → 补充空行
- 列表项前后不一致的缩进 → 统一为 2 空格缩进

### 4. 排版规范
- 中英文之间加空格（如 `使用Python` → `使用 Python`）
- 数字与中文之间加空格（如 `共86张图片` → `共 86 张图片`）
- 连续多个空格 → 合并为 1 个
- 行尾多余空格 → 移除

## 禁止修改

| 类型 | 说明 |
|---|---|
| **原文语义** | 不改写、不润色、不换说法 |
| **专业术语** | 技术名词、产品名、品牌名保持原样 |
| **数据与数字** | 统计数据、百分比、金额不能改 |
| **引用内容** | 引号内的引用文字不能改 |
| **表格数据** | 表格中的内容不能改 |
| **图片引用** | `![alt](path)` 不能改路径 |
| **链接 URL** | 不能改链接目标地址 |
| **Frontmatter** | 不修改 YAML frontmatter 内容 |
| **代码块** | ``` 内的代码不能改 |
| **原文风格** | 不统一作者的个人表达风格 |

## 操作方式

1. 通读全文，标记需修正的位置
2. 逐个修正，每次修正只改最小范围
3. 修正后确认：改动是否仅限于纠错？是否改变了语义？
4. 如果不确定是否为错误，**不改**
