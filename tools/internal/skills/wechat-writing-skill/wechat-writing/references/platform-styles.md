# 平台原生基线 — Step 9 非公众号 lane 必读

> 公众号的风格基线在 `style-guide.md`（声音1/2 + 范文 + 禁区）。本文件是 **X / LinkedIn 的原生基线**，加长尾平台的通用适配原则、各平台图必要性。
>
> **铁律：原生 ≠ 把公众号长文翻过去。** 每个平台从 Step 1 的共享可信度 brief **重新选** 主 claim / 证据密度 / 反共识角度 / CTA / 读者痛点（见 SKILL.md Step 9）。同一内核，不同载体。
>
> **写谁的口吻先看 background：** X/LinkedIn 的"创始人判断/一手案例"必须可溯源到 `background-sources.md` 的 **founder 账号定位** 段，不能凭空编。声音仍是 GenGrowth 那两种（声音1 wzb 踩坑活人感 / 声音2 Lynne 创始人框架），由 **平台 + 题型** 双驱动，不是所有 X 都一个腔。

## X（创始人账号）

X 不是把文章压缩，是**把一个判断拆成可独立转发的链路**。读者在信息流里刷到，3 秒决定看不看下去。

**形态选择（先定单帖还是 thread）：**
- **单帖**：一个尖锐判断 + 一个支撑，能在 280 字（中文约 140）内说完 → 就发单帖，别硬凑 thread。
- **thread（3-7 条）**：判断需要递进论证、或有 3+ 个并列点时用。**别超过 7 条**，超了说明该回公众号长文。

**首帖（hook）= 全条命门：**
- 第一句就是**可独立转发的完整判断**（不是"🧵关于X我有些想法"这种空 hook）。
- 给反差/具体数字/反共识断言，让人想点开看为什么。
- 例（声音2）："做了一年企业 AI 落地，最大的认知转变：企业从不为'更聪明的 Agent'多付一分钱。"

**每帖信息密度：**
- 一帖一个点，点透即止。**别在一帖里塞两个论点**。
- 短句优先，能断行就断行（X 的换行成本低、提升可读）。
- 中文为主；**专有名词/工具名/人名保留英文原文**（Claude Code、Harness、Replit）。面向海外读者的硬核技术判断可整条英文，但创始人账号定位若以中文 builder 为主，默认中文。

**链接 & CTA：**
- thread **末条**才放链接（引到公众号原文/产品），首帖放链接会压触达。
- CTA 轻：末条用"完整拆解👇（链接）"或"展开讲过，评论区放了链接"，不喊"快关注"。
- 可设**追更钩子**："这套三层框架我们自己在用，下条讲 Sandbox 怎么划边界。"

**禁区：** 空 hook 开场（"a thread 🧵"）、为凑 thread 注水、每帖都放链接、营销腔感叹号堆叠、把公众号金句原样搬（X 要更口语、更冲）。

## LinkedIn（创始人账号）

LinkedIn 是**职业语境里的判断**，读者是 founder / operator / 投资人 / 潜在客户。比 X 长、比公众号短，重"专业可信"不重"情绪共鸣"。

**先定两件事（看 background 的账号定位）：**
- **给谁看 + 什么语言**：如果创始人 LinkedIn 主要面向**海外** founder/operator → **英文**，且**弱化中国平台语境**（不默认读者懂"公众号/小红书"，必要时一句话解释或换成 newsletter/blog 等通用说法）。面向国内职业人脉 → 中文。**拿不准就在 Step 1.5 问一句。**
- **基础社交图谱**：如果账号还没基础关注，LinkedIn lane 的 ROI 存疑——这种情况 Step 1.5 默认不推荐 LinkedIn，除非创始人明确要养号。

**结构（短文+判断，不是长文）：**
- 开头 1-2 句给**职业语境里的判断**（不是故事开场）：观察到的行业信号 + GenGrowth 的立场。
- 中段：1-2 个**带证据/案例**的支撑（这里最吃 background 的一手案例——没有就别编，降级成更轻的观点贴）。
- 结尾：一句可被同行接话的判断 + 评论区放原文链接的 CTA。
- 段落短、空行多（LinkedIn 移动端阅读）。

**语气：** 克制、专业、有判断但不说教。少 emoji（最多分点用 1-2 个）。**不用公众号那种煽动金句和加粗判断句轰炸**——LinkedIn 上显得用力。

**hashtag & CTA：** 末尾 3-5 个精准 hashtag（#AIAgent #ToB 之类，别堆）。链接放评论区或末尾"原文见评论"。

**禁区：** 把公众号长文整篇贴上来、中国平台黑话不解释（海外向时）、情绪化吐槽、励志鸡汤收尾、hashtag 堆十几个。

## 长尾平台（即刻 / 知乎 / 视频号口播 / 技术博客 等）— 通用适配原则

核心 3 平台之外的，**不写专门基线**，按下面原则现场适配（选题点名了才出）：

1. **长度随平台**：即刻短（一两段观点）、知乎中长（问题导向+结构化回答）、技术博客长（可接近公众号但更技术、可带代码）、视频号是**口播脚本**（口语、有节奏、可断句成提词器）。
2. **语气随平台**：即刻随性、知乎"专业答主"、技术博客硬核、视频号像对着镜头说话。
3. **CTA 随平台机制**：知乎引导收藏/关注、即刻引导互动、博客末尾留订阅、视频号引导主页。
4. **都从共享 brief 起草**，不翻公众号；拿不准这个平台怎么写就在 Step 1.5 问创始人一句它的定位。
5. **产品案例拆解 / 知识库长文** 这类是**内容形态提示不是平台**——按它指向的真实平台（公众号/技术博客）处理。

## 各平台配图必要性（省创始人时间，不该出的别出 prompt）

| 平台 | 默认图 | 说明 |
|---|---|---|
| 公众号 | 封面必出 + ≤2 插图 | 见 `image-style.md`，深石墨灰手绘无文字 |
| X | **默认不出图** | 文字 thread 为主；除非有现成截图/数据图能增信，才出 1 张 |
| LinkedIn | 可选简图/截图 | 职业向，简单示意图或产品截图即可，不需要公众号那种封面 |
| 小红书（legacy） | 图卡必出 | 委托 `baoyu-xhs-images` 出小红书风图卡 |
| 微博（legacy） | 可选 | 配 1 张或纯文 |
| 长尾 | 按平台 | 视频号无静态图、技术博客可配架构图、知乎可配示意图 |

配图一律遵守 SKILL.md 红线：**只出 prompt，创始人手动生图**，不在流程里自动调 baoyu 生图。
