# 配图风格基线 — GenGrowth成长笔记

> Step 6 配图环节读本文件。**核心原则：本 skill 只产配图 prompt，不在流程里同步生图。**
>
> 为什么不在关键路径同步生图：历史经验里，生图最耗时——卡在"生成→不满意→改 prompt→再生"的同步循环上，
> 且每次重新描述风格导致不收敛。这跟"一周一篇 / 创始人耗时 ≤1h"的目标冲突。
> 所以这里把生图踢出关键路径：先一次性产出到位的 prompt，生图定稿后异步委托 baoyu
> （默认 nano banana + GPT Image 双模型各出一版选优，模型/命令见 SKILL.md Step 6）。

---

## 三条硬约束

1. **数量 ≤3 张**：1 张封面 + 最多 2 张正文插图。不够撑就少放，不要凑数。
2. **一致性优先**：所有图共享同一视觉语言（见下），保证一个号下的图像辨识度。
3. **prompt 一次到位**：构图、元素、配色、尺寸、"无文字"全部写死在 prompt 里，不靠来回调。

---

## 统一视觉语言

- **背景**：深石墨灰（dark graphite），所有图统一。
- **风格**：高级手绘战略图风格（hand-drawn strategic diagram），米白线条为主。
- **配色点缀**：低饱和青蓝 / 淡青绿 / 橙黄 / 米白，各用于区分不同元素，背景始终深石墨灰。
- **文字**：**一律无任何文字、不写任何字符**（公众号配图加文字易显廉价，且不利复用）。
- **焦点**：发光汇聚点用琥珀金 / 暖白，在深色背景上形成强视觉焦点。

## 尺寸规范

| 用途 | 比例 | 像素 |
|------|------|------|
| 封面（头图） | 2.35:1 | 2560×1088（公众号头图最低 900×383 也可） |
| 正文插图 | 16:9 | 1920×1080 |
| 次图 / 缩略图 | 1:1 | 200×200 |

## 封面构图模板（可复用）

左侧散乱输入（约 30%）→ 中央发光汇聚焦点（约 40%）→ 右侧有序输出图标（约 30%）。
表达"从散乱无序，进入一个被系统接住/收敛的瞬间"。具体元素按文章主题替换，但三段式构图和"汇聚漩涡"焦点保持稳定。

## 正文插图模板（可复用）

水平排列的手绘圆角矩形 + 带箭头连线 + 关键节点上方手绘问号气泡（暗示每层回答上一层引出的问题）。
适合表达"分层 / 递进 / 链路"类框架。每个矩形内放一个功能图标（文档叠加=知识库、Y 形分支=版本、`>_`=终端、齿轮=流程边界）。

## prompt 产出格式

每张图给出一段中文描述式 prompt（生图模型对中文长描述识别足够），包含：构图 + 各区域元素 + 配色 + 尺寸 + "整体无任何文字"。
参考仓库已沉淀的两份范例（直接 `@` 引用以保持一致）：

- `内容资产/微信公众号/prompts/配图提示词-承接系统.md`
- `内容资产/微信公众号/prompts/配图提示词-企业级工作流.md`

产出后标注每张图的**插入位置**（如"放在第五节和第六节之间"）和**尺寸**。

## 生图：委托 baoyu skills（不要手搓生图循环）

真要生成图片时，**不要自己拼生图工具的同步循环**（那正是历史上最耗时的坑）。委托给已装好的宝玉 skill，它们自带一致性维度系统、一次到位：

- **封面** → `baoyu-cover-image`。它有 5 个维度（type / palette / rendering / text / mood）。
  GenGrowth 预设映射：`aspect=2.35:1`、`palette=深石墨灰系`、`rendering=手绘`、`text=none`、`mood=克制/战略感`。
- **正文插图** → `baoyu-article-illustrator`。它按 Type×Style 分析文章并配图。
  GenGrowth 预设：`Type=framework/flowchart`、`Style=手绘战略图`、深石墨灰背景、无文字、≤2 张。
- **框架图 / 流程图 / 对比图**（结构性内容，如递归论证、六字段交接单）→ `baoyu-infographic`，比通用生图更稳、返工少。
- **直接点模型 / 双模型出图** → `baoyu-imagine`（nano banana = `google` + `gemini-3.1-flash-image-preview`；GPT Image = `openai` + `gpt-image-1.5`）。

**本文件（GenGrowth 视觉语言）是喂给 baoyu skill 的"风格预设"**——调用时把上面的预设维度连同本文件的视觉语言一起传过去，保证一个号下图像一致。封面默认双模型各出一版选优。

**默认仍把生图放在文章定稿之后、作为可选的异步步骤**，不卡在正文创作的关键路径上，守住"省时"目标。
