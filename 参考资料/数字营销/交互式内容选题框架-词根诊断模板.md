# 交互式内容选题框架 — 词根诊断模板

> **用途**：给「交互式游戏 / 科普可视化 / 在线测试」这类 neal.fun 式网页内容做选题。
> 每挖到一个候选**词根**，按本框架填一遍，输出"做 / 不做 / 怎么做"的裁决。
> **核心原则**：`KD 可打 + 能形成词族矩阵` ＞ `volume 大`。SERP 竞争结构（谁在排）比绝对搜索量更决定优先级。

---

## 〇、为什么这类内容要单独一套框架

普通 SEO 选题盯"搜索量 × 难度"。但交互式内容的可搜索需求**只收敛成 5 个意图族**，且用户搜的是"带具体好奇心/动作的意图"，不是泛词。先把词根归到下表，再往下诊断：

| 意图族 | 用户在搜 | 热度来源 | 交互形态 | 典型词根 |
|---|---|---|---|---|
| **尺度好奇** | 有多大/多深/多小 | 常青、零商业竞争 | 滚动可视化 | `how big is the universe`、`size of ___` |
| **财富模拟** | 花掉某人的钱 | 病毒 + 稳定搜索 | 点击模拟器 | `spend ___ money` |
| **技能/反应测试** | 测我能不能/多快 | 高频、可重复、易出精选摘要 | 小游戏 | `reaction time test`、`cps test` |
| **What-if 模拟** | 如果 X 发生会怎样 | 灾难/科幻好奇 + 蹭热点 | 参数模拟 | `asteroid impact simulator` |
| **无聊发现** | 无聊时玩什么 | 巨量但极卷 | 聚合入口 | `games to play when bored` |

## 二、已诊断词根（实例库，持续追加）

### `reaction time test` — 🟢 优先级 #1（SEO 主导）
- **SERP**：头词被 **Human Benchmark 近乎垄断** + 寄生克隆；但 `average reaction time by age` 长尾是**碎片化小站**（reaction-time-test.io、kordu.tools、measurehuman.com…）→ 缺口在长尾。
- **盲区**：HB 只给测试、**不做按年龄/场景的个性化解读**。
- **长尾树**：按年龄（by age / for 20 year old）、按场景（gaming / fps / valorant / drivers）、按类型（audio / visual）、按阈值（what is a good / is 200ms good）。
- **形态**：测试 + 年龄/性别输入 → "你比 92% 的 20 岁人群快"个性化结果 + 配数据长文抢精选摘要。
- **裁决**：常青 + SERP 碎片化可打 + 交互差异化明确，做"数据文 + 测试页"内链矩阵。

### `spend ___ money` — 🟡 优先级 #2（社交先行）
- **SERP**：已是 **EMD 克隆农场红海**（spendbillmoney.com、spendelonmoney.org、spendthebillions.com… 十几个站抢同批词），neal.fun/spend 也只是其一。泛词 SEO 防御力极弱。
- **能赢的角度**：①**蹭新实体抢速度**（新晋富豪/网红出现时第一时间做 `spend [新实体] money` + 实时 Forbes 净值）；②**财商/教育深度**（wealth inequality simulator、how much is a billion）。
- **形态**：模拟器 + 实时净值 + 可分享小票；社交首发 → 搜索沉淀。
- **裁决**：纯 SEO 进场是和农场卷，不划算；只在"新实体速度"或"内容深度"上做。

## 三、扩词 + 验热度 SOP

**扩词**：词根 × 修饰词
- 疑问前缀：`how big / how deep / how fast / how much / what if`
- 动作后缀：`test / simulator / calculator / generator / quiz / game`
- 实体替换：`spend ___ money`、`size of ___`

**验热度（免费→付费优先级）**
1. **Google Trends** — 常青 vs 脉冲（脉冲要趁势）
2. **Google SERP 自带信号** — PAA、Related searches、有没有 interactive/featured snippet（有 = 这类页能赢）
3. **AnswerThePublic / Keyword Planner** — 批量出量级
4. **Ahrefs / Semrush** — Volume + KD，盯 **KD<30 且 volume>1k**
5. **Reddit / Quora 搜词根** — 验真实需求（反复有人问 = 真需求）

**筛选铁律（4 条全过才做）**
- [ ] 意图能被**一个交互页面**直接满足（不是要长文）
- [ ] **常青** 或 有可预期的周期性热点
- [ ] **KD 可打**（避开被 Human Benchmark / coolmathgames / typing.com 锁死的头部）
- [ ] 能形成**词族矩阵**（一个词根带 10+ 长尾，做内链）

## 四、neal.fun 的具体落地启示

> 把 neal.fun 拆解出的方法论，落到"我们自己做交互式内容 + SEO"的可执行动作。详见 [[../产品分析/neal-fun-分析报告]]。

### 落地启示 1：产品即营销——为"可截图的奇观时刻"而设计
- **neal.fun 怎么做的**：每个作品都内置一个"会想发给朋友"的瞬间（荒诞密码规则、无限合成、众人开车）。零投放，靠产品自传播。
- **我们落地**：选题阶段就问一句——"**这个结果页用户会截图吗？**"。
  - 测试类 → 必须有**个性化可分享结果卡**（"你比 X% 的人快"）。
  - 模拟类 → 必须有**可截图小票/成绩单**。
  - 没有分享钩子的选题，社交飞轮起不来，砍掉。

### 落地启示 2：内容光谱——低操作的"科普可视化"反而是更好的 SEO/病毒资产
- **neal.fun 怎么做的**：作品分两桶——深度游戏（Infinite Craft）制造**流量尖峰**；低操作可视化（The Deep Sea / Size of Space）是**人人看懂、随手分享**的常青资产。
- **我们落地**：别只盯"做个好玩的游戏"。**尺度好奇/数字直观化这类"会动的信息图"，开发成本低、SEO 意图明确（how big / how much）、常青**——应作为打底的 SEO 资产，游戏类作为引爆点。两者配比：可视化打底 7 : 游戏引爆 3。

### 落地启示 3：用 LLM 把"内容生产成本"降为边际可变成本
- **neal.fun 怎么做的**：Infinite Craft 用 Llama 生成无限元素，单人产出"无限内容"游戏，绕过单人产能瓶颈。
- **我们落地**：选题时优先考虑**能用 LLM/数据 API 生成无限长尾页**的形态——
  - `spend [任意实体] money` → 接 Forbes/维基数据，**程序化批量生成**每个名人一页（programmatic SEO）。
  - `size of [任意物体]` → 数据驱动，一个模板裂变上千页。
  - 这是把"交互式内容"和"程序化 SEO"结合的关键杠杆。

### 落地启示 4：流量地理 > 流量总量
- **neal.fun 怎么做的**：美国流量占比 ~36.5% 是其高 CPM 广告变现的命脉。
- **我们落地**：选词根时**优先英语区高 CPM 意图**（北美用户搜的 test/simulator），同样的 UV，变现差 3–5 倍。词根筛选加一条："这个词的主要搜索国是不是 Tier-1？"

### 落地启示 5：警惕"头词垄断"，专挑垄断者的盲区
- **neal.fun 怎么做的**：不和大平台（CrazyGames）拼资源，靠**别人想不到的玩法**切独特需求。
- **我们落地**：SERP 诊断里若头词被单一强站锁死（Human Benchmark / typing.com），**不正面刚**——找它"只给工具、不给个性化解读"的盲区，用"交互 + 数据解读"组合从长尾包抄。

### 落地启示 6：增长靠"持续产出爆款"，但要降 bus factor
- **neal.fun 的风险**：流量脉冲式，高度依赖创始人个人产能（bus factor=1）。
- **我们落地（反向吸取）**：用**内容日历 + 程序化模板 + LLM 产能**，让"出爆款"从"靠灵感"变成"可复用的流程"，避免重蹈 neal.fun 的单点依赖。中型可视化作品填补爆款间歇，维持搜索基本盘。

---

## 五、一句话方法论

> **找的不是"volume 大的词"，是"意图清晰 + 交互能直接满足 + SERP 有缺口 + 能裂变词族 + 结果可分享"的词根。** neal.fun 证明：当内容自带可分享性，最简单的展示广告就是最高 ROI 变现；我们要做的，是把这套"为传播而设计 + 程序化裂变长尾"复制到自己的选题流程里。
