# X AI 独立开发者调研报告

> 报告日期：2026-04-07 | 调研目的：通用了解 / 对标研究  
> 研究对象：`@yihui_indie`、`@AI_Jasonyu`，以及 3 个更具代表性的 X 独立开发者样本

## 一、执行摘要

- 这批开发者的共同点不是“会做 AI”，而是把 **内容、分发、收费、产品迭代** 串成了闭环。X 在这里不是单纯的曝光渠道，而是产品验证、销售转化、个人品牌和招聘合作的总入口。
- `@yihui_indie` 更像“**教育产品 + 创作者品牌 + 自研工具**”的组合体，现金流核心明显偏向课程，产品侧还在向更强的全球化 SaaS 迈进。
- `@AI_Jasonyu` 更像“**多产品矩阵 + 出海流量方法论 + 订阅变现情报**”的组合体，重心不是单一爆款，而是持续上新、借趋势拿流量，再把流量导向多个付费点。
- 更具代表性的国际样本里，`@levelsio` 代表“公开构建 + 高争议内容 + 多现金流产品组合”，`@marc_louvion` 代表“面向开发者卖加速器/模板/课程”，`@damengchen` 代表“AI SaaS + 明确收入叙事 + 持续 build in public”。
- “杭州 97 年小伙 5 个月上 120 个 App”目前只能确认到媒体二次报道，**没有找到可可靠对应的 X 账号**；这一案例更适合作为趋势信号，而不适合作为已验证的 X 样本。

## 二、样本与方法

### 2.1 信息来源分层

| 级别 | 来源类型 | 用法 |
| --- | --- | --- |
| A | 官方网站 / 定价页 / App Store | 判断产品、收费方式、目标用户 |
| B | X 个人简介与公开帖子镜像 | 判断自我定位、内容风格、近期话题 |
| C | Link in bio / 帮助中心 / 第三方资料 | 补充产品矩阵、价格与运营细节 |
| D | 新闻转载与二次报道 | 只作线索，不单独当作事实结论 |

### 2.2 判断口径

- “产品/商业模式”尽量基于官方页面。
- “营收、用户量、粉丝数”如果来自个人简介或第三方镜像，统一视为 **自报或第三方抓取**。
- “内容风格”基于近期开帖主题、置顶式自我介绍、产品链接结构做归纳。

## 三、核心样本分析

### 3.1 熠辉 Indie `@yihui_indie`

#### 3.1.1 基本画像

| 维度 | 结论 |
| --- | --- |
| 账号定位 | AI Coding + AI Anime + B 站 / YouTube / Builder |
| 公开粉丝 | X 约 5.1 万，镜像抓取存在波动 |
| 主要产品 | `happyaicoding.com` AI 编程课；`yidooo.com` AI 动漫/短视频创作平台 |
| 核心受众 | 中文 AI 编程新手、想做出海产品的普通用户、ACGN/短剧创作者 |
| 变现重心 | 课程订阅更稳定，工具产品承担增长与全球化上限 |

#### 3.1.2 产品与业务结构

**产品 1：AI 编程实战课程**

- 官方站显示课程卖点是“零基础 + Cursor + Claude Code + 实战项目”。
- 公开价格是 **599 元/年**，站内展示 **2300+ 学员**、**50+ 实战项目**。
- 课程覆盖网站、小程序、插件、App、Agent，以及支付、部署、图像理解、AI 文生图等模块。
- 学员案例页显示其产品范围已经不只是 demo，而是偏“能上架、能收款、能跑分发”的实战项目。

**产品 2：YidooO / AI 动漫工具**

- 官方站定位为 ACGN 创作者的一体化 AI 动画平台，功能覆盖角色生成、分镜、图生视频、音视频同步。
- 变现模式是 **订阅 + Points 点数消耗 + 企业定制**。
- 官网公开到的高端版本是 `Ultimate` 定制方案，说明其目标不只做 C 端试玩，也在尝试 B 端/工作室客单价。

#### 3.1.3 商业模式判断

**[事实]**

- 课程是明确的订阅现金流产品，定价透明，受众明确，内容更新频率高。
- 动漫工具是更典型的 AI SaaS，具备点数、订阅、企业服务三层收费可能。

**[判断]**

- 熠辉当前最稳的收入来源大概率仍是 **个人品牌驱动的知识付费**，因为课程天然和内容分发强绑定，且能靠直播、社群、答疑提高转化。
- 工具产品更像第二增长曲线：如果跑通全球化和 ACGN 创作者场景，收入上限可能高于课程，但竞争和模型依赖也更强。

#### 3.1.4 X 内容与风格

近期开帖主题可以归为 4 类：

1. **课程销售型内容**  
   例如直播预告、涨价提醒、B 站榜单、购买提醒、学员群导流。
2. **趋势判断型内容**  
   高频讨论 `Vibe Coding`、`Seedance 2.0`、API 接入、出海机会。
3. **社群经营型内容**  
   强调答疑、交流群、直播连麦，明显在经营高互动转化漏斗。
4. **产品转向型内容**  
   他公开提到“课程是确定的收益，风险较低；今年会花更多心思去做产品”，说明本人也清楚课程和产品是两套不同增长逻辑。

**风格判断：**

- 不是纯技术号，更像 **“面向小白的 AI 创业教练 + 正在做产品的 builder”**。
- 语气偏直接、带销售感，但不走纯鸡血路线，强调“能学会、能做出来、能赚钱”。
- 内容重心非常贴近中文用户当下焦虑：不会写代码、怕错过 AI 红利、想出海但不知道从哪开始。

#### 3.1.5 结论

熠辉的强项在于把“**知识付费、社群运营、项目实战、个人品牌**”打通；弱点在于其课程收入和个人服务能力绑定较深，产品业务还需要真正跑出英文市场和持续留存。

### 3.2 鱼总聊 AI `@AI_Jasonyu`

#### 3.2.1 基本画像

| 维度 | 结论 |
| --- | --- |
| 账号定位 | AI / 出海 / SaaS / App / eSIM 实操干货 |
| 公开粉丝 | 镜像抓取在约 2.1 万到 3.6 万之间波动 |
| 自报经历 | 8 年产品经验，20+ 款出海产品，500 万+ 用户，单品 $200K+ 营收 |
| 代表产品 | PaywallPro、Portix、Ghibli Style、AI Headshot |
| 变现特征 | 多产品组合，B2B 与 B2C 并行，偏全自然流量叙事 |

#### 3.2.2 产品与业务结构

**产品 1：PaywallPro**

- 官网定位是 **订阅优化与竞品分析平台**，核心卖点是 iOS 订阅应用的 paywall 截图、历史版本、A/B 变体、数据和 AI 洞察。
- 官网展示“研究 2,000+ iOS 应用订阅模型和数据”，并提供免费版、Pro、Team。
- 帮助中心写明它面向开发者、PM、设计师和运营，且按月/年订阅收费。

**产品 2：Portix**

- App Store 显示其为 iPhone 端 AI 视频/图片工具，覆盖文字生成视频、AI 滤镜、换脸、模板等。
- 变现方式是 **应用内订阅**，页面公开到的价格包括周订阅和年订阅。
- 产品天然适合做多语言出海与 ASO。

**产品 3：Ghibli Style / AI Headshot**

- `ghiblistyle.co` 是典型热点型 AI 图像工具，公开价格有免费、Pro、Premium。
- `ai-headshot-generator.org` 的公司说明页显示这是订阅制 SaaS，使用 Stripe，主要面向北美和欧洲。

#### 3.2.3 商业模式判断

**[事实]**

- Jason 的产品组合同时包含：B2B 情报工具、B2C AI 图像/视频工具、移动端 App。
- 其 link in bio 和 X 简介都围绕“出海、AI、App、增长”来包装个人品牌。

**[判断]**

- 他更像 **“产品工作室 + 内容分发号 + 方法论销售入口”**，而不是单个 SaaS 创始人。
- PaywallPro 是最容易沉淀壁垒的资产，因为其核心是持续更新的数据资产；Portix / Ghibli / Headshot 类工具更接近趋势型流量产品。
- 如果“全自然流量”成立，他最强能力不只是做产品，而是 **抓热点、抓关键词、抓转化场景**。

#### 3.2.4 X 内容与风格

近期开帖主题主要有：

1. **AI 工作流与自动化认知升级**  
   强调 AI 已从“辅助”进入“可直接上线”的工作结构重写阶段。
2. **可操作的增长干货**  
   包括 OpenClaw、抓词、SEO、数据采集、付费墙优化、工具推荐。
3. **出海配套信息差**  
   包括 eSIM、手机号、账号注册、出海运营杂项。
4. **个人创业叙事**  
   包括健康、熬夜、职业切换、赚钱窗口期等。

**风格判断：**

- 比熠辉更“增长导向”，更像 **操盘手/产品经理型创作者**。
- 内容结构常见“强钩子 + 方法拆解 + 立即可做”的信息产品写法。
- 他的 X 不是纯 build in public，而是 **build + distribute + sell** 同步进行。

#### 3.2.5 结论

Jason 的强项是 **多产品矩阵、流量敏感度、订阅与变现方法论**；弱点是产品线较散，长期品牌资产是否能沉淀到少数强护城河产品，还需要时间验证。

### 3.3 “杭州 97 年小伙，5 个月上 120+ App”可信度核验

#### 3.3.1 能确认的部分

- 多家媒体在 2026 年 2 月转载了这条新闻，叙事高度一致：
  - 杭州 1997 年出生
  - 5 个月上线 120 多个 App
  - 90% 有付费用户
  - 核心方法是用 AI 扫描需求后批量生成 App

#### 3.3.2 不能确认的部分

- 没有找到可稳定对应的 **X 账号、个人网站、产品集合页**。
- 报道中姓名写作“张三”，高度像匿名化处理。
- 个别二次稿提到代表产品“康心伴”，但没有找到足够强的第一手链路。

#### 3.3.3 结论

这个案例可以作为“**AI 批量做 niche app 正在进入大众媒体叙事**”的信号，但**不适合作为已核实的 X 独立开发者样本**。如果后续你要做人物库，建议把它放进“待核实名单”。

## 四、补充的更具代表性的开发者

### 4.1 补充标准

- 在 X 上有清晰个人品牌
- 产品与营收路径公开度较高
- 其模式对“AI + 独立开发 + build in public”具有代表性

### 4.2 样本列表

| 开发者 | 代表产品 | 商业模式 | X 风格 | 为什么更代表性 |
| --- | --- | --- | --- | --- |
| `@levelsio` | PhotoAI、Nomad List、Remote OK、Interior AI | 多产品组合；订阅 + 招聘发布 + 社区会员 + AI 工具 | 高争议、高传播、强个人观点 | 是 build in public 的经典模板，且产品、分发、话题制造能力都成熟 |
| `@marc_louvion` | ShipFast、CodeFast、Launchviral | 一次性买断模板/课程 + 小工具矩阵 | 面向 builder 的教育型内容 | 代表“卖铲子给独立开发者”的 AI 时代新路径 |
| `@damengchen` | PDF.ai、Testimonial.to、PDFgen | 标准 SaaS 订阅 + API + AI 功能扩展 | 收入公开、叙事清晰、持续复盘 | 代表“单人/小团队可做出真正长期 SaaS 资产” |

### 4.3 简析

#### `@levelsio`

- SuperX 抓取到的简介中，Pieter 直接把多个产品月收入挂在 bio 上。
- `PhotoAI` 是标准 AI 订阅产品；`Remote OK` 则是典型平台型业务，招聘发布单次收费。
- 他的 X 风格高度个人化，甚至带争议，但这正是他的分发引擎之一。
- 代表意义：**产品可以很多，但真正的底层能力是制造注意力、抓住长期关键词、并把流量导回自己的资产。**

#### `@marc_louvion`

- Marc 的个人站本身就是一张公开营收面板，产品集中在 `ShipFast`、`CodeFast` 等面向 builder 的工具与课程。
- `ShipFast` 公开是一次性买断的 SaaS boilerplate；`CodeFast` 则是 AI 时代的创业者编程课。
- 这一路径非常适合“独立开发者服务独立开发者”，利润率高，交付标准化。
- 代表意义：**卖给 builder 的加速器，比自己反复追热点做 AI wrapper 更稳。**

#### `@damengchen`

- Damon 在 X 上长期公开 `Testimonial.to` 与 `PDF.ai` 的 ARR 进展，形成强 build in public 认知。
- `Testimonial.to` 是稳定的 SaaS 订阅生意；`PDF.ai` 是 AI 场景爆发后放大的第二增长曲线，并延伸出 API。
- 他的内容风格不是卖情绪，而是卖“真实增长截图 + 过程复盘 + 关键决策”。
- 代表意义：**真正有代表性的 AI 独立开发者，不只是会追热点，更会把热点沉淀成可续费的长期资产。**

## 六、关键结论

### 6.1 哪两类模式最值得关注

1. **“卖结果给终端用户”**  
   如 AI 头图、动漫化、视频化、移动端特效工具。这类增长快，但生命周期短，更依赖热点与投放/SEO/ASO。
2. **“卖加速器给开发者/团队”**  
   如课程、模板、paywall intelligence、AI client、API。这类天花板未必最高，但更容易形成复购、口碑和较高毛利。

### 6.2 X 上真正有效的内容，不是泛 AI，而是三件事

- **结果**：收入、榜单、案例、上线速度
- **方法**：怎么做、怎么卖、怎么分发
- **身份**：我是谁、我凭什么、我正在做什么

### 6.3 对 GenGrowth 的直接启发

- 如果要把 X 当主战场，内容不能只讲“AI 很强”，必须讲 **“我怎么把 AI 变成增长结果”**。
- 比起泛泛点评工具，更值得做的是 **“可执行工作流 + 真实案例 + 自家产品嵌入”**。
- 最容易沉淀成资产的方向，不是跟风做一个又一个 AI wrapper，而是做 **开发者/增长团队的情报、模板、自动化和验证工具**。
