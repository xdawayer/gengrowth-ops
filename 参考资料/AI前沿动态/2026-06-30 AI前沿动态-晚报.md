# 🏛️ 深度研究 ｜ 联邦把它列为「国家安全供应链风险」，加州却把它请进所有政府部门

> **一句话定性**：这不是一笔普通的政府打折单。在五角大楼把 Anthropic 定性为「国家安全供应链风险」、把它挤出联邦体系仅几个月后，全美最大州经济体、也是 AI 监管的领头羊——加州——反手把 Claude 设成**全州政府第一个、唯一一个**面向所有机构开放的 AI 生产力工具。这是一场关于「**治理姿态正在成为市场分层依据**」的现场实验：同一套「负责任 AI」的立场，在华盛顿是负债，在萨克拉门托是入场券。

**来源**：加州州长办公室官方新闻稿《Governor Newsom announces a first-of-its-kind partnership…》，2026-06-29 发布；TechCrunch、CBS Sacramento、Fox Business、CNN、ACLU/EFF 等多方交叉核实。

## 一、为什么这一条值得单独深读

过去半年我们推的多是「模型跃迁」「出口管制」「人才战」「百亿并购」这种硬格局事件。一笔「州政府用 Claude 打五折」乍看权重不高。但它真正的分量不在折扣，而在**它发生的时间点和它折射的结构性裂缝**：

- 2026 年 2 月底，特朗普指示联邦机构停用 Anthropic；国防部长 Hegseth 把 Anthropic 定性为「**对国家安全的供应链风险**（Supply-Chain Risk to National Security）」，理由是 Anthropic 拒绝让自家技术用于**大规模监控**和**全自主武器**。国防部转而签了 OpenAI。
- 四个月后，**加州**——美国第一大州、全球第五大经济体、且是 SB 53 等前沿 AI 立法的策源地——签下**首个同类**的全州协议，让 Claude 成为加州技术厅（CDT）新建采购门户里**第一个对所有州级机构开放**的 AI 工具。加州 CIO 被问到联邦那顶「供应链风险」帽子时，回答是：谈判中「**压根没被提起**（just didn't come up）」。

也就是说：**同一家公司、同一套安全立场，在联邦被武器化为「风险」，在州一级却被当成「正是我们要的负责任供应商」。** 这条新闻的真正信息量，是美国公共部门 AI 市场正沿「价值观/治理路线」**联邦–州两极分裂**，而供应商的治理姿态从「合规成本」升级成了「市场准入的分层变量」。这对任何想把 agent 系统卖进受监管/公共部门的团队，是一级信号。

## 二、核心事实（分层）

**A. 协议条款（硬数据）**
- **5 折**：所有州级机构以**对折价**获取 Claude；同等折扣**同时向地方政府开放**——加州的市、县皆可享。
- 附带：**免费的员工培训** + Anthropic 工程师提供的 **GenAI 技术支持与工作流设计输入**。
- 渠道机制：通过加州技术厅（CDT）**新上线的「全州 IT 共享服务门户」SITeS**（Statewide Information Technology Shared Services）发放——官方定位是「把 AI 工具集中到一处、围绕关键业务场景给出**透明定价**」。Claude 是该门户里**第一个面向全部州级机构开放的 AI 生产力工具**。
- **未披露**：官方稿与 TechCrunch 均**未给出合同总金额、覆盖员工数、具体机构数量**——这是个值得记下的透明度缺口（详见质疑部分）。

**B. 已经在跑的场景（不是 PPT，是生产）**
- **CDT + CalOES（加州应急管理办公室）**：用 **Claude Security + Claude Code** 做**网络防御**——对**州政府代码**做扫描、定级（triage）、打补丁。这是本协议里最该被划重点的一条：**agentic coding 已被信任去碰关键公共基础设施的真实代码库。**
- **加州车管局（DMV）**：改善客服、压缩等待时间。
- **加州医疗服务厅（DHCS）**：全美**最大的 Medicaid（医补）机构**，用 Claude 跑内部工作流以更好服务受助人。
- **Engaged California**：用于「协商式民主」平台的内容协助。
- **Poppy**：内置常见州务查询的预置 prompt 工具。

**C. 政策背景（加州一侧的制度铺垫）**
- 2026-03-30，Newsom 签 **行政令 N-5-26**：给 AI 供应商设更严的护栏——**算法偏见、数据隐私、透明度与问责**门槛，机构要用 AI 须先过线。
- 2026-03，另有行政令**加速**政府采用 AI，但同时要求「更强安全标准」。
- 2026-05-21，行政令要求就 **AI 对劳动力市场的冲击**做正式研究。
- 更早的背景：Newsom 对 AI 监管立场一向「两面」——2024 年否决过争议的 SB 1047 前沿 AI 安全法案，却又支持随后的透明度路线。这次他给协议的定调是「**用技术的加州方式：负责任、透明、为人服务**」。

**D. 关键原话**
- **Newsom**：「AI 不应取代政府工作中属于人的部分；它应让我们的工作者更快行动、更有效解决问题、为加州人交付更好结果。」
- **Kate Jensen（Anthropic 美洲区负责人）**：「从一开始，负责任地、为人服务地构建 AI 就是我们的路径——这次合作正是把它落地。」
- **Nick Maduros（加州政府运营署署长）**：「要做到这点，我们得确保团队用上最好的现代工具，包括 Claude。」

## 三、机制拆解：折扣只是钩子，真正的战场是「采购轨道 + 治理路线」

把这件事拆成三层因果，能看清它为什么是战略事件而非促销：

1. **折扣层（钩子）**：5 折 + 免费培训 + 工程师驻场，本质是**旗舰参考客户的获客补贴**。拿下「全美最大州政府」当样板，对 Anthropic 后续向其它州、向受监管行业复制的销售杠杆，远超这笔单的直接收入。

2. **轨道层（锁定）**：真正的护城河是 **SITeS 门户**。一旦 Claude 成为这条「全州共享服务采购轨道」上的默认/首发选项，后来者要么挤不进这条轨道，要么得在已被 Claude 定义的业务场景模板里竞争。**谁定义了采购轨道，谁就拿到了默认值。**（这与我们此前在 Cursor/Grok 深度研究里总结的「数据层→默认值层→定价层」温水煮青蛙是同一机制，只是这里的「默认值层」是政府采购门户。）

3. **路线层（分层准入）**：最关键的一层。加州 EO N-5-26 把「偏见审计、隐私、透明度」设成供应商**硬门槛**——而这恰好是 Anthropic「负责任 AI」立场天然能过的锁。**让 Anthropic 在联邦被惩罚的那套姿态（拒绝大规模监控/自主武器、强调护栏），正是打开加州这把锁的钥匙。** 于是出现了一个结构性结论：**美国公共部门不是一个市场，而是按治理价值观分裂的两个（甚至多个）市场；供应商的「立场」成了决定它能进哪个市场的产品级变量。**

## 四、正反观点

**支持方（这是 Anthropic 的一次漂亮卡位）**
- **市场对冲成立**：被联邦冻结后，立刻在「州 + 受监管行业」这条平行赛道拿下标杆，证明「负责任 AI」不是纯成本，而是能换来特定高价值买家的差异化资产。
- **信任阈值被跨过**：CalOES 让 Claude Code 去扫描/修补**真实州政府代码**——agentic coding 进入关键公共基础设施，这是品类级的信任里程碑，对所有做读码/改码 agent 的团队都是利好背书。
- **可复制模板**：SITeS 这种「共享服务采购门户」一旦跑通，是可以被其它州照抄的分发范式；先发者吃默认值红利。

**质疑方（风险与水分）**
- **透明度缺口**：合同金额、覆盖范围、机构数全部未披露。「5 折」相对什么基准价？没有总额，公众无法判断这笔交易的真实规模与性价比，也无从评估单一供应商集中度。
- **单一供应商锁定**：把全州的采购默认值押给一家私企，本身就是治理风险——这与 Newsom EO 想防的「问责/透明」有内在张力。把关键代码的扫描与打补丁交给一个外部模型，**「监督者悖论」**同样适用（谁来审 AI 改的政府代码？）。
- **劳动力与监管者的角色冲突**：Newsom 自己 5 月才下令研究 AI 对就业的冲击，转头就把 AI 推进所有政府部门；公共部门工会对「AI 进政府服务」历来警惕。更微妙的是**监管者与被监管者的距离**——加州是 AI 立法重镇，州长高调与一家前沿实验室「结盟」，监管捕获之嫌难免（与我们此前在 Daybreak/Trusted Access 里点过的「政府关系即护城河、也即单点依赖」是同一隐忧，只是换到州一级）。
- **PR > 实质？**：「用技术的加州方式」是漂亮叙事；但生产场景目前是客服、内部工作流、代码扫描这类**低风险**用途，离「重塑政府服务」尚远。

## 五、深度解读：对行业 + 对「做多 agent 系统的我们」的具体含义

**对行业格局**
- **公共部门正成为 AI 的「新企业市场」，但它沿价值观裂成两半。** 联邦（国防/安全口）按「能否用于监控/武器、是否服从行政意志」筛供应商，OpenAI 占位；州与民事口按「偏见/隐私/透明度」筛供应商，Anthropic 卡位。**一个工具在五角大楼被禁、在萨克拉门托被捧，会成为常态。** 这条裂缝会持续放大，并外溢到欧盟、各国地方政府。
- **治理姿态从「成本中心」变成「市场分层变量」。** 实验室的安全/价值观立场不再只是合规开销，而是决定它能进哪块公共市场的战略选择。这反过来会**激励实验室主动「选边」**做市场细分。

**对做 agent 系统的我们（落到可执行判断）**
1. **公共/受监管部门是真实的增量市场窗口，但入口是「治理产品化」。** 想进这条赛道，偏见审计、数据隐私、可审计性、透明定价不能是事后补丁，得做成**一等产品表面**（直接对标 EO N-5-26 这类供应商门槛）。把「合规与可解释」当功能卖，而不是当负担扛。
2. **盯住「共享服务采购门户」这类轨道，而不是单个机构订单。** SITeS 模式会被复制——能上轨道 = 拿到默认分发。对 to-G/to-B 团队，研究并卡位这些采购门户的接入资格，价值高于零散 PoC。
3. **agentic coding 进政府关键代码 = 信任阈值已过，但门槛同步抬高。** CalOES 用 Claude Code 扫/补州代码，意味着读码改码 agent 在最保守的买家那里也站住了——但也意味着**可审计、危险原语 allowlist、可回滚、循环熔断**从「加分项」变「准入项」（与我们反复强调的 control 范式一致）。把这些做成基础设施，是进这个市场的硬通货。
4. **治理姿态当成「可配置变量」，做供应商/路线冗余。** 同一个 agent 系统，可能需要在「联邦安全口能接受的配置」与「州民事口能接受的配置」之间切换；底层模型供应商也要可热切换（在加州能用 Claude，不代表在另一些场景不会被换）。**多模型 + 多治理配置 + 多采购轨道**的冗余，是公共部门生存项。
5. **监管者关系是护城河也是新单点依赖。** 与某州/某监管机构深度绑定能换分发，但政治风向、选举周期、监管捕获质疑都会反噬——别把业务连续性押在单一政府关系上。

## 六、接下来盯什么（信号）

- **第二个州**：是否有别的州照抄 SITeS 式门户、把 Claude（或竞品）设为默认？复制速度 = 这套打法的护城河深度。
- **合同细节披露**：金额/范围若后续公开，可校准这笔交易的真实规模与单一供应商集中度风险。
- **DOD「供应链风险」定性的法律走向**：ACLU/CDT 已就此向法院施压（主张不应因 Anthropic 倡导 AI 护栏而惩罚它）；若该定性被推翻或固化，会决定「联邦–州分裂」是临时还是结构性。
- **竞品反击**：OpenAI/Google 是否针对州与民事口推出对标的「治理友好」打包与采购通道。
- **生产场景升级**：CalOES 的代码扫描/打补丁若出现任何事故或扩面，是 agentic coding 进关键基础设施成败的早期读数。
- **劳动力反弹**：加州公共部门工会对「AI 进政府」的反应，以及 Newsom 5/21 劳动力研究的结论。
