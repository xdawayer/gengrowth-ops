# Claude Science：前沿实验室的护城河，正从「模型」搬到「工作流」

**一句话定性：** Anthropic 这次不发新模型，而是把 60+ 科研数据库、算力调度和「可复现」打包进一个跑在现有 Claude（含 Opus 4.8）上的科研操作台——这是把 Claude Code 的打法平移到生命科学，也是三大实验室集体上攻「价值垂直」、对冲中端模型商品化的最清晰信号。对做 agent 系统的团队来说，它把一个残酷的事实摆上台面：**护城河正在从模型权重，迁移到工作流 / harness / 上下文集成这一层——而这正是平台方开始亲自下场抢的那一层。**

## 为什么值得深读 / 背景

过去 72 小时 Anthropic 连出三张牌，看似无关，其实是同一盘棋的三条腿：

1. **Sonnet 5**（6/30，已推）——把中端能力做到「够好又更便宜」，代表**商品化/效率**这条战线；
2. **Fable 5 / Mythos 5 出口管制解除 + 国家安全门控**（6/30，已推）——顶级能力被政府闸门锁定，代表**能力门控**这条战线；
3. **Claude Science**（6/30，本篇）——不靠新模型，靠工作流把 Claude 装进科研全流程，代表**价值垂直化**这条战线。

三者合起来回答了一个所有前沿实验室都躲不掉的问题：**当模型能力在中端快速趋同、价格逐年腰斩，钱从哪里赚、护城河挖在哪里？** Claude Science 给出的答案不是「造一个更强的生物模型」，而是「占领某个行业的操作层（operating layer）」。TechCrunch 的定性很直接：Anthropic 想让 Claude Science 之于科研，成为 Claude Code 之于软件开发那样的**默认工作台**。

这对「做多 agent 系统」的我们不是一条行业八卦，而是一次战略校准：它验证了 harness/工作流是真护城河，同时也警告了平台方会亲自杀进这一层。

## 核心事实（硬数据分层）

**定位（最关键的一条）**
- Claude Science **明确不是新模型、也不是「生物学专用的更强模型」**。它跑在**今天人人可用的现有 Claude 模型上（含 Claude Opus 4.8）**，无特殊访问、无能力门控。差异化 100% 来自工作流与集成，而非权重。

**集成范围**
- 预置 **60+ 精选 skills 与 connectors**，覆盖基因组学、单细胞、蛋白质组学、结构生物学、化学信息学。
- 直连一批领域权威数据库：**UniProt、PDB、Ensembl、Reactome、ClinVar、ChEMBL、GEO** 等。

**可复现性（差异化核心）**
- 每生成一张图（如 3D 蛋白结构、化学结构式），都同时给出：**产出该图的确切代码与运行环境 + 一段自然语言说明其生成过程 + 完整的消息历史**。这是把「AI 出的结果不可审计」这个科研硬伤直接堵上。

**算力调度**
- 统一管理**本地笔记本、经 SSH 的 HPC 集群、以及经 Modal 的按需 GPU**——把「在数据库/流水线/工具间反复横跳」的苦活收进一个环境。

**商业化 / 分发**
- Beta 面向 **Pro / Max / Team / Enterprise**（Team/Enterprise 需管理员启用）——**走宽订阅铺量**，不是窄门企业定制。
- 配套 **AI for Science 资助计划**：最多 **50 个项目**，每个 **$30,000 额度**；申请开放至 **7/15**，7/31 前通知，项目周期 **9/1–12/1**。
- 战略纵深：建立在 6 月 **~$4 亿收购 Coefficient Bio** 和 **挖来 AlphaFold 之父、诺奖得主 John Jumper** 之上；据 Endpoints 报道，Anthropic **自己也在启动内部药物发现项目**（埋下与客户竞合的伏笔）。

## 机制 / 技术拆解：它到底「垂直」在哪

关键要看清：Claude Science 的价值**不在模型层，在三个「胶水」层**——而这三层恰恰是 agent 工程的主战场：

1. **上下文集成层**：把 60+ 领域数据库变成模型随手可调的工具（UniProt/PDB/ChEMBL…）。对科学家而言，「模型知道去哪查、怎么查、拿回来怎么用」比「模型参数更大」更能省时间。Forbes 一位评测者称它花 **$26** 就把自己所在领域的研究图谱梳理了一遍。

2. **执行/算力编排层**：本地 ↔ HPC(SSH) ↔ 云 GPU(Modal) 的透明切换，让 agent 能真正「跑起来」而不只是「说出来」。这是从 chatbot 到 **可执行科研 agent** 的分水岭。

3. **可复现/可审计层**：代码+环境+消息历史三件套绑定每个产物。这是把 agent 输出**变成科学证据**的必要条件，也是它相对「套壳」竞品最难被复制的部分——因为它需要把复现基建做深。

一句话：Claude Science 证明了**「同一个模型 + 更好的 harness」可以是一个独立产品**。这正是我们这类团队赖以立足的命题。

## 正反观点

**支持方（这是真护城河）**
- 科研的痛点从来不是「模型不够聪明」，而是工具碎片化、结果不可复现、算力接不上。Claude Science 精准打这三点。
- 走宽订阅（Pro 就能用）+ 资助计划，是典型的**开发者生态/自下而上渗透**打法，复制了 Claude Code 的成功路径。
- 复现三件套 + 深度数据库集成需要长期投入，**不是一个周末能套壳出来的**，具备真实的积累型壁垒。

**质疑方 / 风险（别被叙事带跑）**
- **「同一个模型自我核查」的根本缺陷**：没有独立真相源，AI 幻觉在科研场景是致命的——错误的蛋白结构/通路会污染下游研究。
- **AI-for-science 的兑现远落后于宣传**：多项研究指出，当代 LLM 在科学上「缺乏发散与否定的想象力」，不会主动提出零假设，最大的实际应用竟是「写垃圾论文」——PLOS、Frontiers 已因 AI 灌水停收部分投稿。工作台再顺手，也不解决「洞见」问题。
- **竞合暗雷**：Anthropic 一边把 Claude Science 卖给药企，一边自己下场做药物发现（Coefficient Bio + Jumper）。**你的科研数据喂进一个同时是你潜在竞争对手的平台**——这是生命科学客户的真实顾虑。
- **「工作流可被更强模型抹平」的老问题**：今天靠 harness 领先，明天对手模型原生具备工具编排/长程执行时，工作流红利会不会被吃掉？（这正是 TechCrunch 没敢深挖、但对我们最相关的一问。）

**三家的战略分岔（值得单独记住）**
- **Anthropic**：走宽 + 工作流（Claude Science，人人可订阅）。
- **OpenAI**：走窄 + 门控（GPT-Rosalind，企业定制/受限预览）。
- **Google DeepMind**：靠**别人没有的自研专有模型**（AlphaFold 系）。
> 同一个「AI-for-science」赛道，三种完全不同的护城河假设——法律、金融、工程等其他垂直大概率会重演这套分岔。

## 深度解读：对行业 + 对「做多 agent 系统的我们」的具体含义

**对行业：** 前沿实验室的竞争重心正从「谁的模型基准更高」转向「谁能占领某个行业的操作层」。模型正在成为**可替换的商品**，而工作流/集成/可复现正在成为**可收费的产品**。Claude Science 是这个转向的样板间——它公开承认「护城河不在权重」，等于给整个行业发了信号：**上攻价值垂直，是对冲中端商品化的主路径。**

**对我们（可执行判断，不是套话）：**

1. **命题被验证，也被抢跑。**「同一个模型 + 更好的 harness = 独立产品」这条我们赖以立足的假设，被 Anthropic 亲自背书了。好消息：说明我们做的事有护城河。坏消息：**平台方会亲自杀进最肥的垂直**。选赛道时要问一句——「这个垂直，Anthropic/OpenAI 会不会两年内自己做？」科研、法务、编码这种大而通用的，平台方志在必得；要挖能守住的护城河，得往**平台方看不上或够不着**的方向走：更细分的行业、更脏的专有数据、更强的分发关系、更深的合规/工作流嵌入。

2. **把「可复现/可审计」当一等公民，而不是事后补丁。** Claude Science 最难被套壳的部分不是数据库接得多，而是「代码+环境+消息历史」绑定产物。**在 agent 输出即证据的场景（金融、医疗、法务），可审计性本身就是护城河。**建议把「每个 agent 动作可回放、可归因、可复现」提到架构一等约束。

3. **警惕「同一个模型自我核查」的陷阱。** 这是 Claude Science 的公开软肋，也是我们的通病。**验证器必须尽量独立于生成器**——用不同模型/规则引擎/外部真相源交叉核验，别让 agent 既当运动员又当裁判。（这也呼应我们此前定的「把静默降级当一等错误码」的思路。）

4. **竞合意识写进架构。** 当你把业务数据喂进平台方的垂直产品，你就在训练潜在对手。**保留把工作流从任一 provider 上迁走的能力**（多模型热切换 + 自持编排/复现层），别让最有价值的上下文集成被锁死在单一平台的私有工作台里。

5. **分发打法可抄。** Pro 订阅即用 + 资助计划（$30k×50）是低摩擦、自下而上的生态渗透。做垂直 agent 产品时，**先让个人开发者/研究者零门槛用起来**，再自上而下谈企业，往往比一开始就啃 Enterprise 更快。

## 接下来盯什么（信号）

- **资助计划成果**（9/1–12/1 项目周期结束后）：50 个项目里若跑出**可复现、经同行认可的真实发现**，则「工作流即护城河」证伪了「AI-for-science 泡沫」论；若只出漂亮 demo，则质疑方胜。这是最硬的验证点。
- **Anthropic 自研药物管线** vs. 客户竞合的公开摩擦：是否有药企因数据顾虑却步。
- **OpenAI / Google 的对位反应**：GPT-Rosalind 是否也转向「宽订阅工作台」，或 Google 是否把 AlphaFold 系包装成可订阅产品——三家战略会不会收敛。
- **同一套「操作层」打法向法律/金融/工程垂直的复制**：谁先出「Claude Law / Claude Finance」级的工作台。
- **Project Glasswing 7 月总结报告**（预告 7 月出，尚未落地）：同属「Claude 做深垂直（安全）」主线，可与本篇互为佐证。
