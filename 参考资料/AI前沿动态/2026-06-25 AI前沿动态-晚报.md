# 🕵️ 深度研究 ｜ Anthropic 指控阿里巴巴打了「史上最大蒸馏战」：前沿能力的边境，正在被武装化

> **一句话定性**：这不是又一起普通的「中国实验室抄作业」八卦。它是头部前沿实验室**第一次把竞品蒸馏行为，用可量化的账户/调用数据打包成一份递交白宫与参议院的政策诉求**——把「谁能从谁的模型里抽走能力」从工程问题，正式升级为国家安全与产业政策问题。对所有把前沿模型当地基的团队（包括我们）来说，它宣告了一件事：**通过 API 暴露出来的高级能力，本身就是一种可被工业化窃取、也正在被监管化的资产。**

**来源**：CNBC《Anthropic accuses Alibaba of campaign to 'brazenly' and 'illicitly' extract AI capabilities》（2026-06-24）；Bloomberg / The Next Web / Benzinga / Japan Times 同日跟进；批评视角见 Global Times（援引前商汤智能产业研究院院长田丰）。
🔗 原文：https://www.cnbc.com/2026/06/24/anthropic-alibaba-distillation-campaign.html

## 一、为什么这一篇值得单独深读

过去半年，AI 圈的「蒸馏」指控大多停留在传闻级（2026 年 2 月 Anthropic 曾点名 DeepSeek / MiniMax / Moonshot）。这次不同的有三点：

1. **有硬数字、有时间窗、有规模对比**——不再是「疑似」，而是一份带账户数、调用数、起止日期的取证式叙述。
2. **指控对象升级**——从中国初创公司，升级到**阿里巴巴**这一巨头及其 Qwen 实验室；单这一起的规模就超过 2 月三家之和。
3. **动作升级**——Anthropic 没有走法庭，而是**直接致信白宫与多位参议员**，把它包装成「在政府已经警告之后仍公然为之」的国家安全事件，并附上具体立法/反垄断诉求。

换句话说，这是一份**伪装成新闻的政策文件**。它要解决的不是「阿里有没有抄」，而是「美国政府要不要把蒸馏定性为受管制行为」。这条线和我们过去两周持续跟踪的主线（[[2026-06-20 Fable5 出口管制下线]]、[[OpenAI Daybreak Trusted Access]]、出口管制博弈）是同一张大网——**前沿能力的获取与流动，正在被国家力量重新划界。**

## 二、核心事实（按可信度分层）

**A. 取证数据（Anthropic 致信中的硬指控）**
- **约 25,000 个欺诈账户**，在 **2026-04-22 至 06-05** 期间，对 Claude 发起 **2,880 万次（28.8M）交互**。
- 瞄准的是 Claude **最值钱的两项能力：软件工程（software engineering）与 agentic reasoning（智能体推理）**——也正是当前商业价值最高的两个前沿。
- 操作者被指「与阿里巴巴及其 Qwen 实验室关联」，通过批量假账户**绕过 Anthropic 明令禁止中国境内部署/访问的地理封锁**。
- Benzinga 口径：致信对象包括参议员 **Elizabeth Warren、Tim Scott**，并称约 **2,880 万次 Claude 对话被「收割（harvested）」**。

**B. 规模对比（说明「为什么这次是质变」）**
- 2026 年 2 月那一轮：DeepSeek / MiniMax / Moonshot 三家合计约 **24,000 个假账户、1,600 万+次交互**。
- **阿里这一起（28.8M）单独就超过了上述三家之和**——故称「史上最大」。

**C. Anthropic 的具体诉求（这才是真正的「正文」）**
- 向国会/白宫提出三条：① **澄清反垄断指引**，让各实验室可以就蒸馏行为**互相通报/共享信息**而不触红线；② **支持对先进 AI 芯片的出口管制**；③ 对使用蒸馏的公司**设定处罚**。
- 时间叙事被刻意强调：这波操作发生在 **Kratsios 备忘录之后**、「在政府已发出警告的情况下公然为之」——把它钉成「明知故犯的国安事件」。
- 配套立法已在路上：参议员 **Bill Hagerty、Andy Kim** 拟在国防法案中加修正案；众议员 **Bill Huizenga、Sydney Kamlager-Dove** 也在推相关法案。

**D. 对方反应**
- 阿里巴巴**「不予置评（no comment）」**，对 CNBC 的问询亦未及时回应——注意：截至发稿**没有正式否认**，但也**没有承认**。

## 三、机制拆解：什么是「蒸馏」，为什么 API 防不住

**蒸馏（distillation）** 的技术本义：用一个更强的「教师模型」大量产出问答/推理轨迹，再拿这些高质量输出去训练一个更小、更便宜的「学生模型」，让学生以极低成本逼近教师的行为。它在学术与工业界是**完全标准、合法且普及**的模型压缩手段——这正是争议的核心。

把它「武器化」的关键在于：
- **教师能力是通过 API 免费/低价暴露的**。你只要能持续大量地问，就能把教师在 SWE、agentic reasoning 上的「解题分布」搬走。模型越强、越 agentic，单次交互携带的「可蒸馏知识密度」越高——**前沿能力越强，越值得被蒸，也越容易被蒸。**
- **真正昂贵的是前沿研发（数据、算力、对齐），蒸馏几乎零边际成本地搭便车。** 这就是 Anthropic 说的「siphon（虹吸）」——你花几十亿训出的能力，对方用 API 调用费就抽走一大块。
- **地理封锁靠账户层防御，而账户可以被批量伪造。** 25,000 个假账户绕过「禁止中国访问」，说明**基于注册/IP 的准入控制在工业级对手面前形同虚设**——这点对所有做 agent 准入/配额的团队都是直接警示。

**为什么「证明」很难（也是质疑方的最大抓手）**：蒸馏发生在对方的训练管线里，你**看不到**。Anthropic 能拿出的只是「己方这侧的异常调用模式」（账户聚类、调用量、目标能力指向），而**无法直接证明这些数据最终进了 Qwen 的训练集**。指控的证据强度，本质上停在「高度可疑的使用行为」，而非「被蒸馏的铁证」。

## 四、关键原话 / 引述

- Anthropic（致信措辞，经 CNBC/Bloomberg 转述）：阿里相关操作者「**brazenly（明目张胆）**」且「**illicitly（非法）**」地提取 Claude 能力，且发生在「**Kratsios 备忘录之后、政府警告之下**」。
- 阿里巴巴：「**no comment**」。
- 批评方——田丰（前商汤智能产业研究院院长，via Global Times）：
  - 「**蒸馏是 AI 行业广泛采用的模型压缩技术**」，把它框定为非法是对正常技术实践的曲解；
  - 指控反映的是「**技术霸权焦虑**」，是 AI 竞争中的「**踢掉梯子（kick away the ladder）**」战术；
  - 「中国公司是在合规框架下、通过合法数据源与算法优化进步的」；
  - 反将一军：Anthropic 自己刚以 **15 亿美元**和解了作家们关于未授权训练数据的诉讼，「**通过污名化竞争对手，转移外界对其自身数据滥用历史的注意力**」。

## 五、正反观点

**支持 Anthropic 一方（这是真问题）：**
- 数据规模与目标指向（专挑 SWE + agentic）确实不像普通用户行为，更像有组织的数据采集。
- 25,000 假账户系统性绕过地理封锁，无论最终用途如何，**本身已是明确的服务条款与出口意图规避**。
- 如果前沿能力可以被零成本虹吸，那「投入数十亿做前沿研发」的商业模型就被掏空——这是真实的产业激励问题，值得政策介入。

**质疑 / 风险一方（同样站得住）：**
- **举证天花板**：无法证明数据进了 Qwen 训练集；指控止于「可疑使用」。
- **双重标准**：Anthropic 自身有 15 亿美元未授权数据和解在前，道德高地不稳。
- **监管捕获之嫌**：诉求落点是「出口管制 + 反垄断豁免（好让实验室抱团通报）+ 对竞品罚款」——**每一条都直接利好在位的美国前沿实验室**。把商业竞争包装成国安议题，可能是在用政策筑墙。
- **技术常识**：蒸馏是行业通用技术，「合法蒸馏」与「恶意侵权」的边界至今没有公认标准；在没有这条线之前，指控更像叙事博弈。

**我的判断**：两边都对一半。**使用行为层面的滥用大概率真实存在**（假账户绕封锁很难洗白）；但**「蒸馏窃取能力」的因果链证据不足**，且 Anthropic 的诉求结构明显服务于自身竞争与 IPO 前的叙事。这是一起「真问题 + 被战略性放大并政策化」的复合事件。

## 六、深度解读：对行业 + 对「做多 agent 系统的我们」的具体含义

**对行业格局：**
1. **「能力边境」正式成为新战场。** 继 Fable 5 被出口管制下线（厂商被政府强制下线）之后，这一起是**厂商主动把竞品准入推向政府管制**。两件事方向相反，但指向同一个未来：**前沿能力的跨境流动将被许可化、名单化、可处罚化。** API 不再是中立的水电，而是受地缘政治约束的「受控物项」。
2. **中美前沿差距叙事被反复重写。** 如果连阿里都要靠大规模蒸馏来追 SWE/agentic 能力，说明（按 Anthropic 的叙事）真正的前沿仍有代差；但中方叙事（蒸馏是常规技术、开源协作才是正道）则指向「差距在快速收窄且方式合法」。**真相大概率是：基座差距仍在，但通过蒸馏 + 开源，应用层能力差距远小于基座差距。**
3. **「蒸馏防御」会成为前沿实验室的新护城河投入项**——指纹水印、调用异常检测、账户图谱、输出投毒——这些都会涨价并转嫁到 API 定价里。

**对做 agent 系统的我们（落到可执行判断）：**
1. **准入控制要按「工业级对手」设计，而不是按「普通滥用」设计。** 25,000 假账户绕过地理封锁，说明**单靠注册/IP/邮箱验证完全不够**。我们自己产品若有 API/agent 入口，需要：行为指纹 + 调用量异常检测 + 账户聚类，把「高密度系统性抽取」当成一类专门威胁来防。
2. **把「上游 API 条款风险」写进供应商风险登记册。** Anthropic 正在推动「对蒸馏设罚 + 实验室间互相通报」。这意味着**未来 API 提供商对「你拿输出去做什么」的审查会变严**——我们如果用 Claude/GPT 输出做任何形式的微调、合成数据、蒸馏到自有小模型，要提前核对 ToS 与新监管，避免踩线被封号。这条和我们一直强调的**多模型冗余/可热切换**直接挂钩：任何单一上游的封号或政策变更，都不能让我们停摆。
3. **「能力获取的合规性」本身要做成资产。** 当蒸馏边界被政策化，**能证明「我的能力来自合法授权链路」会变成 to-B 销售的信任凭证**（尤其面向受监管行业）。反过来，链路不清的能力获取会变成隐性负债。建议现在就把我们 agent 的「能力来源（哪个模型、什么授权、是否含合成/蒸馏数据）」盘点存档。
4. **开源权重的战略价值再被强化，但要看清两面。** 一方面，开源/自托管仍是对抗「上游政策性断供」的唯一真保险（呼应 [[Fable5 出口管制]] 与 SpaceX/Reflection 开源算力交易）。另一方面，**如果开源生态被指为「蒸馏赃物」的温床，开源模型本身可能成为下一个监管目标**——「用开源模型」未来也可能附带合规审查。把这点写进风险登记册。

## 七、接下来盯什么（信号）

- **阿里是否正式回应**（否认 / 反诉 / 沉默）——沉默越久，越说明在权衡而非清白。
- **白宫/商务部是否把蒸馏纳入正式管制语言**（Hagerty-Kim 国防法案修正案、Huizenga 法案进展）——一旦成法，API 输出的跨境使用就有了执法依据。
- **「实验室间反垄断豁免互相通报」是否落地**——若成真，等于前沿实验室结成「能力外流情报联盟」，对所有下游是重大格局变化。
- **是否出现技术性的「蒸馏取证标准」**（水印/指纹被法庭或监管采信）——这会决定指控能否从「可疑」升级为「可罚」。
- **是否波及其他中国实验室或扩大到 GPT/Gemini**（OpenAI、Google 会不会跟进类似指控）——若三家齐发，等于行业集体把「能力边境」制度化。
- 关联追踪项：[[GPT-5.6 临近发布]]（预测市场已滑向 7 月）、Fable 5 复线进展（第 13 天仍未复线）。
