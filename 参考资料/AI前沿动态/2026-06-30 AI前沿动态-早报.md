# 效率拐点：当 agent 的单位经济学反转

**一句话定性：** 一家旧金山 agent 创业公司把 100% 流量从 Claude 切到中国开源模型 DeepSeek、推理成本暴降约 90%——这不是孤例，而是「tokenmaxxing（无脑堆 token 冲榜）」时代结束、agent 行业进入「单位经济学第一」阶段的标志性事件，对所有「做多 agent 系统」的团队都是一记直球。

## 为什么值得深读 / 背景

过去一年 agent 工程的隐性默认是「token 越烧越好」：榜单、内部 leaderboard、「用满 AI」的 KPI，催生了所谓 **tokenmaxxing**——开发者拼命往工具里灌 token，不太在意单位产出。代价现在集中爆发：

- **Uber** 用 4 个月（到 4 月）烧光了 **2026 全年 AI 预算**，5000 名工程师人均月成本 **$500–$2000**，COO 公开质疑「到底值不值」；6 月起 Uber 上线分级管控，起步档 **$1,500/月/人**（Fortune 5/26）。
- **Sam Altman**（5 月）承认 AI 成本对客户已成 **"a huge issue"**。
- agentic 系统天然高频、长链路、多轮调用，token 消耗相对聊天是数量级跃升——**单位成本第一次成为决定 agent 产品能否活下去的变量**，而不再是事后优化项。

Lindy 这篇一手复盘，把这个抽象趋势变成了可被复制的工程决策，所以值得当作本周最具战略含义的一条来深挖。

---

## 核心事实（硬数据分层）

**Lindy 迁移（一手）**
- 团队规模：**25 人**创业公司；迁移前 **AI 成本已超过人力成本**——CEO Flo Crivello 称「unsustainable」。
- 候选评测：横向测了 **GLM5.1、Kimi K2.5、Kimi K2.6、DeepSeek v4 Flash**。
- 最终选型：**DeepSeek v4 Flash**，托管在 **Atlas Cloud（美国公司、美国基础设施）**——即用中国开源「权重」、但不直连中国厂商。
- 迁移结果：在迁移流量上 **推理成本下降约 90%**；将在数月内省下「millions」。
- 路由策略：把大部分 managed-agent 流量（含 Claude/Sonnet 路径与残余 Gemini 路径）切到 DeepSeek；**Sonnet 仅在用户显式选择、或需要更高智能的路径时保留**。
- 方法论原话：「**Offline evals are necessary. They are not enough.**」——他们叠加了离线评测、provider 实测、prompt 优化、内部灰度、在线评测、留存监控，才全量上线。

**行业层（独立来源交叉）**
- **DeepSeek 在 Vercel 平台上的 token 使用份额从 <1% 跃升到 17%（5 月）**，但营收份额仍约 1%（Rest of World 引 Vercel 数据）。
- OpenRouter 上最受欢迎的四个模型来自 **DeepSeek、腾讯、MiniMax、小米**，均属最具成本效率的一档。
- 个人开发者对比：Claude 编码会话约 **$10/小时**，DeepSeek 等价方案 **<$0.50/小时**；有开发者把月支出从 $500（Claude/ChatGPT）压到 $200（MiniMax、Kimi、小米 MiMo）。

## 关键原话 / 引述

- Flo Crivello（Lindy CEO）：「**The cost curve crashed to the ground.**」「**It's a matter of survival for the business.**」——并明确表示：**如果 Anthropic 降价，他愿意切回 Claude**。
- Sam Altman：AI 成本已成 **"a huge issue"**。

这两句把本轮叙事钉死了：**迁移的驱动力是价格，不是质量偏好**。这意味着在位实验室手里仍握着「降价就能召回客户」的牌，但也说明中端 agent 工作负载的客户忠诚度极低。

## 正反观点

**支持/迁移方逻辑**
- 成本对小团队是生死线（AI 成本 > 人力成本时无可争辩）；90% 降幅直接改写单位经济学。
- 用美国云（Atlas Cloud）托管开源权重，规避了「数据出境给中国厂商」的一部分顾虑。
- 智能分级路由保留了高端模型在难任务上的能力，不是「一刀切降智」。

**质疑/风险方**
- **质量仍落后美国前沿**：中国开源模型「handle routine tasks adequately」，但难任务、长程可靠性上仍有差距——全量切换对高价值场景有风险。
- **地缘 / 合规 / 数据**：美国议员已对 **Airbnb、Anysphere（Cursor 母公司）使用中国开源模型** 发起调查；大型受监管企业因数据安全与审查顾虑仍不敢用。
- **可逆性即脆弱性**：Lindy 自己说「Anthropic 降价就回切」——这种迁移是议价杠杆，不是终局；它揭示的是中端市场的商品化，而非中国模型的胜利。
- **变现悖论**：中国厂商即便拿下 17% 用量，营收份额仍仅 1%——用户经美国云间接调用，厂商建立不起客户关系，这反过来削弱其长期投入能力。

## 深度解读：对行业 + 对「做多 agent 系统的我们」的具体含义

**对行业。** 这是 agent 商业模式的第一次「成本现实检验」。前沿实验室的护城河正在沿能力梯度分层：**顶端（最难任务、前沿推理）仍有定价权，中端（常规、高频、可外包给便宜模型的任务）正在被开源权重商品化。** GPT-5.6 已经出了 Luna（$1/$6）这种下探档、Sol 计划上 Cerebras 冲 750 tok/s——这都是对「效率拐点」的正面应战。降价战与「价值垂直化」（如 Anthropic 6/30 的 AI for Science）会是在位者的两条对冲路径。

**对我们（做多 agent 系统的团队）——落到可执行判断：**

1. **把「单位经济学」提到架构第一性原则。** 每条 agent 链路要能算清 token 成本/任务，并设预算护栏（参考 Uber 的分级管控）。tokenmaxxing 时代的「能跑就行」必须退役，**每任务成本（cost-per-task）成为和延迟、成功率并列的一等 KPI**。

2. **多模型智能路由从「优化项」变成「标配」。** 常规/可降级任务跑便宜模型，难任务/高价值路径才上前沿模型——Lindy 的「DeepSeek 兜底 + Sonnet 按需」就是模板。没有这层路由，单位成本无法压缩，也拿不到对供应商的议价权。

3. **把 provider 抽象成可热切换的商品层 + 建立 eval 闸门。** Lindy 的「offline + online eval gate」是迁移能成立的前提。**热切换能力 = 议价权 + 抗断供保险**（与我们 seen 里出口管制把美国前沿模型变成「受监管波动资产」的主线直接呼应：一边监管把上游推向不确定，一边成本把中国开源推进生产，多模型多阵营冗余既省钱又抗风险）。

4. **中国开源权重进入「严肃生产候选池」，但合规要做成一等公民。** 把「模型来源 / 托管地 / 数据流向 / 客户可接受度 / 上游 ToS」写进供应商登记册（与 deemed-export、上游 API 蒸馏条款同一治理框架）。to-B 场景尤其要预判客户与监管对「中国模型」的红线——Airbnb/Anysphere 被调查就是前车之鉴。用美国云托管开源权重是当前的折中解。

5. **别把当前选型当终局。** 迁移由价格驱动、且可逆。给自己保留「在位实验室降价/出新便宜档时快速回切」的能力，避免为省一时成本牺牲质量上限或踩合规雷。

## 接下来盯什么（信号）

- **在位者降价/下探档**：Anthropic 是否推更便宜 Sonnet/Haiku 档；OpenAI Luna 与 Cerebras 高速推理的落地价格与可用性。
- **更多美国生产团队公开迁移案例** + 议员对「美企用中国开源模型」调查的升级或立法动作。
- **中国开源新版基准是否逼近前沿**：DeepSeek v4、GLM5.x、MiniMax、Kimi 下一版在难任务/agentic 基准上的表现。
- **Anthropic 6/30「AI for Science」** 是否以高价值垂直（生物/制药/科研）对冲中端商品化压力——价值垂直化能否撑住定价权的早期读数。
- **DeepSeek 等的变现曲线**：用量 17% / 营收 1% 的剪刀差是否收敛，决定中国厂商能否持续投入。
